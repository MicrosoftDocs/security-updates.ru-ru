Советы по безопасности (Microsoft) (2905247)
============================================

Небезопасная конфигурация сайта ASP.NET делает возможным несанкционированное получение прав
-------------------------------------------------------------------------------------------

Дата публикации: 10 декабря 2013 г. | Обновлено: 9 сентября 2014 г.

**Версия:** 2.0

Общие сведения
--------------

### Аннотация

Майкрософт объявляет о доступности обновления для Microsoft ASP.NET, чтобы устранить уязвимость в состоянии просмотра ASP.NET, возникающую, если в настройках конфигурации отключить проверку Machine Authentication Code (MAC). Данная уязвимость делает возможным несанкционированное повышение привилегий. Ей подвержены Microsoft .NET Framework 1.1 с пакетом обновления 1 (SP1), Microsoft .NET Framework 2.0 с пакетом обновления 2 (SP2), Microsoft .NET Framework 3.5, Microsoft .NET Framework 3.5.1, Microsoft .NET Framework 4 и Microsoft .NET Framework 4.5/4.5.1.

Уязвимым для атак становится любой сайт ASP.NET, для которого в настройках конфигурации отключена проверка MAC состояния просмотра. Злоумышленник, успешно воспользовавшийся данной уязвимостью, может с помощью специально созданного HTTP-содержимого внедрить код для выполнения в контексте служебной учетной записи на сервере ASP.NET. Специалистам Майкрософт известно об опубликованных общих сведениях, позволяющих использовать данную уязвимость, но им неизвестно о каких-либо активных атаках.

**Смягчающие факторы**:

-   На сайтах ASP.NET проверка MAC состояния просмотра по умолчанию включена.

**Рекомендация**. Майкрософт рекомендует пользователям выполнить предлагаемое действие, чтобы убедиться в том, что проверка MAC состояния просмотра ASP.NET остается включенной на сайтах ASP.NET. Более подробные сведения см. в разделе **Предлагаемые действия** данного выпуска советов по безопасности.

Пользователям, которые уже установили какое-либо из перечисленных выше обновлений, следует повторно применить данные обновления, чтобы защититься от уязвимости, описанной в этом выпуске советов. В отношении остального уязвимого программного обеспечения, не включенного в данный список, успешно обновившим свои системы пользователям никаких действий выполнять не требуется.
  
**Какова область действия данных советов?**   
Назначение этого выпуска советов по безопасности — уведомить пользователей о том, что корпорация Майкрософт публикует обновление, которое позволяет администраторам настроить свои серверы ASP.NET так, чтобы проверка MAC состояния просмотра всегда оставалась включенной, а также предоставляет общие рекомендации по включению проверки MAC состояния просмотра на серверах IIS.
  
**Что такое состояние просмотра?   
**Состояние просмотра — это функция ASP.NET, позволяющая разработчикам веб-сайтов поддерживать состояние страниц и сохранять данные в веб-формах для запросов POST или обновлений и изменений страниц. Состояние просмотра широко используется разработчиками ASP.NET, поэтому оно распространено на сайтах ASP.NET. Синтаксический анализ состояния просмотра выполняется всегда, даже если свойству EnableViewState задано значение False. Дополнительные сведения см. в статье [Общие сведения о состоянии просмотра ASP.NET](http://msdn.microsoft.com/library/ms972976.aspx).
  
**Позволяет ли отключение состояния просмотра снизить опасность этой уязвимости?   
**Нет. Синтаксический анализ состояния просмотра всегда выполняется сервером ASP.NET, даже если свойству EnableViewState задано значение False, независимо от того, задано это свойство или нет в web.config, директиве @Page или теге ASP.NET. Злоумышленник может внедрить свойство состояния просмотра в запрос POST клиента, обойдя настройку EnableViewState.
  
**Что такое проверка MAC состояния просмотра?   
**Проверка MAC (Machine Authentication Code) состояния просмотра — это функция, которая заставляет ASP.NET создавать хэш данных состояния просмотра при создании страниц. Затем этот хэш используется для сравнения с состоянием просмотра при обратной передаче, позволяя серверу проверить, было ли состояние просмотра подделано. Эта технология позволяет избежать недопустимого изменения данных при обратной передаче, снижая опасность описанной в этом выпуске советов уязвимости.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**   
В большинстве случаев злоумышленник, успешно воспользовавшийся этой уязвимостью, может повысить привилегии до уровня служебной учетной записи на уязвимом сайте ASP.NET (с неправильно настроенной проверкой MAC состояния просмотра).
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**   
Злоумышленник, не прошедший проверку подлинности, может отправить специально созданное HTTP-содержимое целевому серверу, что может позволить злоумышленнику на сервере выполнить код в контексте служебной учетной записи на сайте ASP.NET.
  
**Как действует обновление?   
**Обновление устраняет данную уязвимость — проверка MAC состояния просмотра всегда включена и исключена возможность ее отключения на сервере.
  
**Какие дополнительные действия должны предпринять пользователи после установки данного обновления?   
**Характер данного исправления требует, чтобы некоторые пользователи, в частности, те, которые используют ASP.NET в веб-фермах, предприняли дополнительные действия, чтобы обеспечить согласованную доступность своих сайтов ASP.NET. Дополнительные действия по настройке см. в разделе **Предлагаемые действия** ниже.
  
**Как определить, какая версия Microsoft .NET Framework установлена на моем компьютере?**  
Пользователь может установить и запустить несколько версий .NET Framework на компьютере, причем установка версий допускается в любом порядке. Существует несколько способов определения того, какие версии .NET Framework установлены на компьютере. Дополнительные сведения см. в [статье 318785 базы знаний Майкрософт](http://support.microsoft.com/kb/318785).****
  
**Чем отличаются .NET Framework 4 и клиентский профиль .NET Framework 4?**  
Предлагаются два профиля для распространяемых пакетов .NET Framework версии 4: .NET Framework 4 и клиентский профиль .NET Framework 4. Клиентский профиль .NET Framework 4 является подмножеством профиля .NET Framework 4, оптимизированным для клиентских приложений. В нем содержатся функции для большинства клиентских приложений, включая функции Windows Presentation Foundation (WPF), Windows Forms, Windows Communication Foundation (WCF) и ClickOnce. Это позволяет ускорить развертывание и уменьшить размер установочных пакетов для приложений, предназначенных для клиентского профиля .NET Framework 4. Дополнительные сведения см. в статье MSDN [Клиентский профиль .NET Framework](http://msdn.microsoft.com/en-us/library/cc656912.aspx).****
  
**У меня установлена среда .NET Framework 3.0 с пакетом обновления 2 (SP2); эта версия не значится в списке уязвимого ПО в этом бюллетене. Нужно ли мне устанавливать это обновление? **  
В этом бюллетене описана уязвимость, затрагивающая слои объектов .NET Framework 2.0. В установщике .NET Framework 3.0 с пакетом обновления 2 (SP2) запрограммирована установка .NET Framework 2.0 с пакетом обновления 2 (SP2), поэтому при установке первого продукта устанавливается и второй указанный продукт. Поэтому пользователи, у которых установлена среда .NET Framework 3.0 с пакетом обновления 2 (SP2), должны установить обновления безопасности для .NET Framework 2.0 с пакетом обновления 2 (SP2).
  
**У меня установлена среда .NET Framework 3.5 с пакетом обновления 1 (SP1). Нужно ли мне устанавливать какие-либо обновления?**   
В этом бюллетене описана уязвимость, затрагивающая слои объектов .NET Framework 2.0. В установщике .NET Framework 3.5 с пакетом обновления 1 (SP1) запрограммирована установка .NET Framework 2.0 с пакетом обновления 2 (SP2) и .NET Framework 3.0 с пакетом обновления 2 (SP2). Поэтому пользователи, у которых установлена среда .NET Framework 3.5 с пакетом обновления 1 (SP1), должны установить обновления безопасности для .NET Framework 2.0 с пакетом обновления 2 (SP2).
  
 
  
Предлагаемые действия  
---------------------
  
<span id="sectionToggle3"></span>
-   **Применение этого обновления для уязвимых версий Microsoft .NET Framework **
  
    У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](https://support.microsoft.com/kb/294871). Пользователи, у которых не включена функция автоматического обновления, могут ее включить, следуя процедуре, описанной в разделе [Включение или отключение автоматического обновления](http://go.microsoft.com/fwlink/?linkid=398470).
  
    Администраторам корпоративных установок и конечным пользователям, которые планируют установить это обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это при первой возможности, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747). Данные обновления также можно загрузить по гиперссылкам на Центр загрузки Майкрософт в таблице "Подвержены уязвимости" этого выпуска советов. Сведения об установке данных обновлений вручную см. в [статье 2905247 базы знаний Майкрософт](http://support.microsoft.com/kb/2905247).
  
-   **Администраторам и для корпоративных установок со сценариями веб-ферм   
    **Майкрософт рекомендует следовать указаниям, доступным в [статье 2915218 базы знаний Майкрософт](http://support.microsoft.com/kb/2915218).
