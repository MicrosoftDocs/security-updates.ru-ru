Security Bulletin

Бюллетень по безопасности (Майкрософт) MS07-061 - Критическое
=============================================================

Уязвимость обработки URI в системе Windows делает возможным удаленное выполнение кода (943460)
----------------------------------------------------------------------------------------------

Дата публикации: 13 ноября 2007 г. | Дата обновления: 16 января 2008 г.

**Версия:** 1.2

### Общие сведения

#### Аннотация

Данное обновление устраняет уязвимость, о которой сообщалось в открытых источниках. В способе, используемом оболочкой Windows для обработки специально созданных URI существует уязвимость, делающая возможным удаленное выполнение кода. Если оболочка Windows проверяла URI недостаточно тщательно, злоумышленник может воспользоваться этой уязвимостью и выполнить произвольный код. Корпорацией Майкрософт найдены способы использования этой уязвимости в системах с обозревателем Internet Explorer 7. Однако уязвимость существует и в файле Shell32.dll системы Windows, который входит в состав всех поддерживаемых выпусков Windows XP и Windows Server 2003.

Данное обновление для системы безопасности является критическим для всех поддерживаемых выпусков Windows XP и Windows Server 2003. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости**.

Это обновление для системы безопасности устраняет уязвимость, изменяя способ обработки неправильных URI оболочкой Windows. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

Это обновление для системы безопасности устраняет уязвимость, описанную впервые в [советах корпорации Майкрософт по безопасности (943521)](http://technet.microsoft.com/security/advisory/943521). Дополнительные сведения об уязвимости см. в подразделе вопросов и ответов нижеследующего раздела "Сведения об уязвимости".

**Рекомендация.** Майкрософт рекомендует пользователям установить обновление немедленно.

**Известные проблемы.** [В статье 9434601 базы знаний Майкрософт](http://support.microsoft.com/kb/943460) описаны известные на данный момент проблемы, с которыми можно столкнуться при установке этого обновления для системы безопасности. В этой статье также приводятся решения, рекомендованные для устранения этих проблем.

Уязвимость при обработке URI — CVE-2007-3896
--------------------------------------------

<span></span>
В способе, используемом оболочкой Windows для обработки специально созданных URI существует уязвимость, делающая возможным удаленное выполнение кода. Злоумышленник может воспользоваться ею, включив специально созданный URI в приложение или вложение, что делает возможным удаленное выполнение кода.

В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896).

#### Факторы, снижающие опасность уязвимости при обработке URI, — CVE-2007-3896

К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.

-   Система Windows 2000 с пакетом обновления 4 (SP4) не подвержена этой уязвимости.
-   Система Windows Vista не подвержена этой уязвимости.
-   Система Windows Vista x64 Edition не подвержена этой уязвимости.
-   Корпорацией Майкрософт не найден способ воспользоваться данной уязвимостью в любой операционной системе Windows с обозревателем Internet Explorer 6.
-   Злоумышленник, успешно воспользовавшийся данной уязвимостью, может получить те же самые права, которые имеет локальный пользователь. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.
-   При атаке через электронную почту пользователи, которые читают почту в формате обычного текста, менее подвержены уязвимости. Наибольшая вероятность использования этой уязвимости возникает, если пользователь откроет вложение или щелкнет ссылку, ведущую на вредоносный веб-узел.

#### Методы обхода уязвимости при обработке URI — CVE-2007-3896

Корпорацией Майкрософт не найдены методы обхода этой уязвимости.

#### Вопросы и ответы об уязвимости при обработке URI — CVE-2007-3896

**Какова область воздействия данной уязвимости?**  
Эта уязвимость связана с запуском программного кода в удаленном режиме. Злоумышленник, успешно воспользовавшийся данной уязвимостью, может получить те же права, которыми обладает локальный пользователь.

Если пользователь вошел в систему с правами администратора, то злоумышленник, которому удалось воспользоваться уязвимостью, может установить полный контроль над системой. Таким образом, злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

**В чем причина уязвимости?** В недостаточной обработке неправильных URI оболочкой Windows.

**Что такое URI?**
Универсальный код ресурса (URI) — это строка символов, используемая для распознавания ресурсов в Интернете или в сети и управления ими. URL-адрес является типичным примером URI, ссылающимся на такой ресурс, как веб-узел. Дополнительные сведения об URI см. в документе [RFC 2396](http://www.ietf.org/rfc/rfc2396.txt) (на английском языке).

**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**  
Убедив пользователя открыть вложение в сообщении электронной почты или перейти по ссылке на вредоносный веб-узел, злоумышленник может запустить произвольный код в контексте вошедшего в систему пользователя. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

**Каким образом злоумышленник может воспользоваться этой уязвимостью?**  
Сначала он должен создать специальный URI и отправить его в приложение уязвимой системы, которое попытается открыть источник, на который дана ссылка в URI. Этой уязвимости подвержены приложения, которые используют URI в качестве входных данных из ненадежных источников, таких как вложения в сообщениях электронной почты, документы или данные из сети, и принимают их за безопасные. При определенных обстоятельствах обработка специально созданных URI позволяет выполнить произвольный код. Чтобы воспользоваться уязвимостью, злоумышленнику придется убедить пользователя запустить вредоносное вложение или приложение либо посетить веб-узел. Злоумышленник может создать специальный веб-узел, предназначенный для использования данной уязвимости через обозреватель Internet Explorer, а затем убедить пользователя посетить этот веб-узел. Уязвимостью можно также воспользоваться через веб-узлы, которые подверглись атаке, и веб-узлы, которые принимают или размещают пользовательские сведения или рекламу. Такие веб-узлы могут содержать специально созданные URI, использующие данную уязвимость. Специально созданные URI можно также передать в уязвимые системы посредством рекламных объявлений или иными способами.

**Какие системы в первую очередь подвергаются риску?**  
Уязвимы поддерживаемые выпуски Windows XP и Windows Server 2003 с установленным обозревателем Internet Explorer 7.

**Как действует обновление? **
Оно устраняет уязвимость, изменяя способ обработки недопустимых URI оболочкой Windows.

**В моей системе не установлен обозреватель Windows Internet Explorer 7. Почему я получаю это обновление? **
Уязвимость существует в файле Shell32.dll системы Windows, который входит во все поддерживаемые выпуски Windows XP и Windows Server 2003. Корпорацией Майкрософт не найдено способов использования этой уязвимости во всех операционных системах с обозревателем Internet Explorer 6. Это обновление доступно в качестве меры глубокой обороны для всех пользователей поддерживаемых выпусков Windows XP и Windows Server 2003 независимо от установленной версии Internet Explorer.

**Я использую систему Windows** **Vista** , **подвержена ли она этой уязвимости? **
Нет. Система Windows Vista не подвержена данной уязвимости. Windows Internet Explorer 7 входит в состав Windows Vista, но оболочка Windows этой системы не подвержена данной уязвимости.

**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности? **
Да. На момент выпуска этого бюллетеня по безопасности об уязвимости было уже объявлено в открытых источниках. Ей был присвоен классификационный номер [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896), и она впервые была описана в [советах корпорации Майкрософт по безопасности (943521)](http://technet.microsoft.com/security/advisory/943521).

**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками? **
Да. На момент выпуска данного бюллетеня по безопасности корпорация Майкрософт получала сведения об использовании этой уязвимости.

**Помогает ли установка этого обновления для системы безопасности защититься от атак с применением кода, опубликованного в открытых источниках и использующего данную уязвимость? **
Да. Данное обновление безопасности устраняет используемую уязвимость. Уязвимости, устраняемой данным обновлением, присвоен классификационный номер [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896).

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленной версии системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание.** В зависимости от версии операционной системы или установленных программ некоторые из файлов, перечисленных в таблице сведений о файлах, могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, устанавливаемого на компьютере, сравнив ее с версией, приведенной в соответствующей таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленной версии системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание.** В зависимости от версии операционной системы или установленных программ некоторые из файлов, перечисленных в таблице сведений о файлах, могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, устанавливаемого на компьютере, сравнив ее с версией, приведенной в соответствующей таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
