Security Bulletin

Бюллетень по безопасности (Майкрософт) MS07-031 - Критическое
=============================================================

Уязвимость в пакете безопасности для безопасного канала Schannel в системе Windows делает возможным удаленное выполнение кода (935840)
--------------------------------------------------------------------------------------------------------------------------------------

Дата публикации: 12 июня 2007 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Данное критическое обновление для системы безопасности устраняет уязвимость в пакете безопасности для безопасного канала (Schannel) в системе Windows, о которой сообщалось в частном порядке. Пакет безопасности для безопасного канала Schannel реализует стандартные интернет-протоколы проверки подлинности SSL и TLS. Данная уязвимость делает возможным удаленное выполнение кода при просмотре с помощью обозревателя специально созданной веб-страницы или при использовании приложением протоколов SSL и TLS. Однако попытки использовать ее приведут, скорее всего, к завершению работы обозревателя или приложения. Система не будет соединяться с веб-узлами и ресурсами по протоколу SSL или TLS, пока не будет перезагружена.

Данное обновление для системы безопасности является критически важным для поддерживаемых выпусков системы Windows XP, важным для выпусков системы Windows 2003 и имеет среднюю важность для выпусков системы Windows 2000. Дополнительные сведения см. в подразделах **Подвержены уязвимости, Не подвержены уязвимости** данного раздела.

Обновление для системы безопасности устраняет уязвимость, изменяя способ анализа клиентом данных о серверных ключах, отправляемых сервером. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.** Майкрософт рекомендует пользователям установить обновление немедленно.

**Замена обновлений.** Данное обновление не заменяет предыдущие обновления для системы безопасности.

Уязвимость в пакете безопасности для безопасного канала Schannel в системе Windows — CVE-2007-2218  
--------------------------------------------------------------------------------------------------
  
<span></span>
В методе, используемом безопасным каналом Schannel на клиентском компьютере для проверки отправляемых сервером цифровых подписей, существует уязвимость, делающая возможным удаленное выполнение кода. Злоумышленник может воспользоваться ею через обозреватель, создав специальный веб-узел и убедив пользователя посетить его. В любом случае злоумышленник не может заставить пользователей посетить эти веб-узлы. Злоумышленник может создать специальный веб-узел для использования данной уязвимости через обозреватель Internet Explorer, а затем убедить пользователя посетить его.
  
Стандартное описание данной уязвимости приводится в перечне Common Vulnerabilities and Exposures по адресу [CVE-2007-2218](http://cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-2218)
  
#### Факторы, снижающие опасность уязвимости в пакете безопасности для безопасного канала Schannel в системе Windows, — CVE-2007-2218
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   В случае атаки через Интернет злоумышленник должен владеть веб-узлом, который содержит веб-страницу, предназначенную для использования данной уязвимости. Кроме того, веб-узлы, которые принимают или размещают пользовательские сведения или рекламу, могут иметь специальное содержимое, использующее данную уязвимость. В любом случае злоумышленник не может заставить пользователей посетить эти веб-узлы. Он старается склонить их к этому, убеждая, как правило, щелкнуть ссылку, ведущую на веб-узел, в сообщении электронной почты или программы обмена мгновенными сообщениями.  
-   Попытки воспользоваться данной уязвимостью, приведут скорее к состоянию отказа в обслуживании, вызванному неожиданной перезагрузкой уязвимой системы, чем к удаленному выполнению кода. В системе Windows 2000 состояние типа "отказ в обслуживании" ограничено существующим и новыми SSL- и TLS-соединениями; система при этом остается работоспособной. В системе Windows Server 2003 успешная атака может привести к перезагрузке системы.
  
#### Методы обхода уязвимости в пакете безопасности для безопасного канала Schannel в системе Windows — CVE-2007-2218
  
Корпорацией Майкрософт не найдены методы обхода этой уязвимости.
  
#### Вопросы и ответы об уязвимости в пакете безопасности для безопасного канала Schannel в системе Windows — CVE-2007-2218
  
**Какова область воздействия данной уязвимости?**  
В системе Windows XP эта уязвимость делает возможным удаленное выполнение кода. Воспользовавшись этой уязвимостью, злоумышленник может в удаленном режиме захватить полный контроль над системой, Таким образом, злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями. Попытки использовать эту уязвимость приведут, скорее всего, к отказу в обслуживании, а не к удаленному выполнению кода.
  
В системе Windows 2000 ее использование может привести к невозможности принимать SSL- и TLS-соединения. Система не будет соединяться с веб-узлами и ресурсами по протоколу SSL или TLS, пока не будет перезагружена.
  
В системе Windows 2003 использование этой уязвимости может привести к автоматической перезагрузке.
  
Данная уязвимость не позволяет злоумышленнику выполнить код или повысить уровень прав в системах Windows 2000 и Windows 2003.
  
**В чем причина уязвимости?**  
Безопасный канал Schannel выполняет неполную проверку специально сконструированных цифровых подписей, отправляемых сервером во время SSL-подтверждения.
  
**Что такое безопасный канал Schannel?**  
Пакет безопасности для безопасного канала (Schannel) — это поставщик службы безопасности (SSP), реализующий стандартные интернет-протоколы проверки подлинности SSL и TLS. Безопасный канал Schannel является частью пакета безопасности, предоставляющего службу проверки подлинности для обеспечения безопасного соединения между клиентом и сервером. Дополнительные сведения о безопасном канале Schannel см. на [веб-узле MSDN.](http://msdn2.microsoft.com/en-us/library/ms678421.aspx)
  
**Что может сделать злоумышленник, воспользовавшись этой уязвимостью?**  
Воспользовавшись этой уязвимостью в одном из выпусков системы Windows XP, злоумышленник может получить полный контроль над системой. Однако попытки использовать уязвимость приведут, скорее всего, к завершению работы обозревателя или приложения, использующего протоколы SSL и TLS. Система не будет соединяться с веб-узлами и ресурсами по протоколу SSL или TLS, пока не будет перезагружена. Во всех выпусках систем Windows 2000 и Windows 2003 попытки использовать эту уязвимость приведут к атаке типа "отказ в обслуживании".
  
**Каким образом злоумышленник может воспользоваться уязвимостью?**  
Злоумышленник может воспользоваться этой уязвимостью через обозреватель, создав специальный веб-узел и убедив пользователя посетить его. Обработка таких специально сконструированных цифровых подписей, отправляемых сервером, приводит к завершению работы приложения, использующего данный API, а также отключает протокол SSL до выполнения перезагрузки. В любом случае злоумышленник не может заставить пользователей посетить эти веб-узлы. Он старается склонить пользователей к этому, обычно убеждая их щелкнуть ссылку, ведущую на его веб-узел, в сообщении электронной почты или в запросе службы мгновенных сообщений.
  
**Какие системы в первую очередь подвергаются риску?**  
Вредоносное действие возможно только при посещении пользователем веб-узлов после входа в систему. Поэтому все системы, в которых часто используются обозреватель или приложения, работающие с протоколами SSL и TLS (например, рабочие станции или серверы терминалов), подвержены наибольшему риску воздействия этой уязвимости.
  
**Как действует обновление?**  
Обновление устраняет уязвимость, изменяя способ выполнения клиентом анализа данных о серверных ключах, отправляемых сервером.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**  
Нет. Корпорация Майкрософт получила информацию об этой уязвимости из достоверных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**  
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

#### Сведения о развертывании

**Установка обновления**

При установке этого обновления системы безопасности необходимо проверить, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления корпорации Майкрософт.

Если один из файлов RTMQFE, SP1QFE, SP2QFE ранее был обновлен другим исправлением, установщик скопирует на компьютер указанные файлы. В противном случае будут скопированы файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

-   **Проверка версий файлов**

    Эти действия зависят от установленной версии и выпуска системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.

    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.

**Установка обновления**

При установке этого обновления системы безопасности необходимо проверить, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления корпорации Майкрософт.

    Эти действия зависят от установленной версии и выпуска системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание.** В зависимости от версии операционной системы или установленных программ некоторые из файлов, перечисленных в таблице сведений о файлах, могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
