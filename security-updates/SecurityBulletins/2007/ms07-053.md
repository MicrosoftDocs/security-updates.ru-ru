Security Bulletin

Бюллетень по безопасности (Майкрософт) MS07-053 - Важное
========================================================

Уязвимость в службах Windows для UNIX делает возможным несанкционированное получение прав (939778)
--------------------------------------------------------------------------------------------------

Дата публикации: 11 сентября 2007 г. | Дата обновления: 19 сентября 2007 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это существенное обновление для системы безопасности устраняет одну уязвимость, о которой сообщалось в открытых источниках. В службах Windows для UNIX 3.0, службах Windows для UNIX 3.5 и подсистеме для UNIX-приложений существует уязвимость. Запуск определенных двоичных setuid-файлов может позволить злоумышленнику несанкционированно получить права.

Это существенное обновление для системы безопасности поддерживаемых выпусков Windows 2000, Windows Server 2003, служб Windows для UNIX 3.0, служб Windows для UNIX 3.5 и подсистемы для UNIX-приложений (компонента Windows Server 2003 и Windows Vista). Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** данного раздела.

Дополнительные сведения об этих уязвимостях см. в подразделе "Вопросы и ответы" для каждой отдельной уязвимости в разделе **Сведения об уязвимостях**.

**Рекомендация.** Корпорация Майкрософт рекомендует пользователям установить данное обновление для системы безопасности.

**Известные проблемы.** Нет.

Уязвимость в службах Windows для UNIX делает возможным несанкционированное получение прав — CVE-2007-3036  
---------------------------------------------------------------------------------------------------------
  
<span></span>
В службах Windows для UNIX 3.0, службах Windows для UNIX 3.5 и подсистеме для UNIX-приложений существует уязвимость. Запуск определенных двоичных setuid-файлов может позволить злоумышленнику несанкционированно получить права. Воспользовавшийся ею злоумышленник может несанкционированно получить права в системе пользователя.
  
В перечне Common Vulnerabilities and Exposures данной уязвимости присвоен номер [CVE-2007-3036](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3036).
  
#### Факторы, снижающие опасность уязвимости в службах Windows для UNIX, делающей возможным несанкционированное получение прав, — CVE-2007-3036
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Службы Windows для UNIX 3.0 и службы Windows для UNIX 3.5 не входят в стандартную конфигурацию Windows 2000 с пакетом обновления 4 (SP4), Windows XP с пакетом обновления 2 (SP2), Windows Server 2003 с пакетом обновления 1 (SP1) и Windows Server 2003 с пакетом обновления 2 (SP2). Службы Windows для UNIX 3.0 и службы Windows для UNIX 3.5 могут быть установлены в системах Windows 2000 с пакетом обновления 4 (SP4), Windows XP с пакетом обновления 2 (SP2), Windows Server 2003 с пакетом обновления 1 (SP1) и Windows Server 2003 с пакетом обновления 2 (SP2). Подсистема для UNIX-приложений не включена по умолчанию в системах Windows Vista и Windows Server 2003. Подсистема для UNIX-приложений является дополнительным компонентом систем Windows Vista и Windows Server 2003.
  
#### Методы обхода уязвимости в службах Windows для UNIX, делающей возможным несанкционированное получение прав, — CVE-2007-3036
  
Корпорацией Майкрософт не найдены методы обхода этой уязвимости.
  
#### Вопросы и ответы об уязвимости в службах Windows для UNIX, делающей возможным несанкционированное получение прав, — CVE-2007-3036
  
**Какова область воздействия данной уязвимости? **  
В службах Windows для UNIX 3.0, службах Windows для UNIX 3.5 и подсистеме для UNIX-приложений существует уязвимость, которая позволяет злоумышленнику несанкционированно получить права при запуске определенных двоичных setuid-файлов. Воспользовавшийся ею злоумышленник может несанкционированно получить права в системе пользователя. Злоумышленник сможет устанавливать программы, а также просматривать, изменять или удалять данные.
  
**В чем причина уязвимости? **  
Службы Windows для UNIX 3.0, службы Windows для UNIX 3.5 и подсистема для UNIX-приложений неправильно обрабатывают двоичные setuid-файлы.
  
**Что может сделать злоумышленник, воспользовавшись этой уязвимостью? **  
Он может несанкционированно получить права в уязвимой системе. Пользователи, учетные записи которых обладают меньшими правами в операционной системе, запущенной на виртуальной машине, подвергаются **такому же** риску, как пользователи, работающие в ней с правами администратора.
  
**Каким образом злоумышленник может воспользоваться уязвимостью? **  
Злоумышленнику потребуется выполнить локальный вход в уязвимую систему и запустить определенные двоичные setuid-файлы. Также он может попытаться убедить пользователя запустить эти файлы.
  
**Что такое setuid? **  
Пользователи клиентских компьютеров могут устанавливать для файла бит setuid (установка идентификатора пользователя при выполнении). Исполняемый файл с установленным битом setuid выполняется под идентификатором пользователя владельца файла, а не под идентификатором пользователя, выполняющего файл.
  
**Какие системы в первую очередь подвергаются риску? **  
Windows 2000 с пакетом обновления 4 (SP4), Windows XP с пакетом обновления 2 (SP2), Windows Server 2003 с пакетом обновления 1 (SP1) и Windows Server 2003 с пакетом обновления 2 (SP2), в которых установлены службы Windows для UNIX 3.0 или Windows для UNIX 3.5. Windows Server 2003 R2 как расширение Windows Server 2003. Windows Vista, в которой включена подсистема для UNIX-приложений.
  
**Как действует обновление? **  
Обновление устраняет данную уязвимость, исправляя обработку учетных данных подключения двоичных setuid-файлов.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности? **  
Да. О данной уязвимости сообщалось в ограниченном количестве открытых источников.

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
    **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
    **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
    **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
    **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
