Security Bulletin

Бюллетень по безопасности (Майкрософт) MS08-001 - Критическое
=============================================================

Уязвимости протокола TCP/IP в системе Windows делают возможным удаленное выполнение кода (941644)
-------------------------------------------------------------------------------------------------

Дата публикации: 8 января 2008 г. | Дата обновления: 25 января 2008 г.

**Версия:** 3,0

**Версия:** 3,0

### Общие сведения

#### Аннотация

Это критическое обновление для системы безопасности устраняет две уязвимости в протоколе TCP/IP, о которых сообщалось в частном порядке. Воспользовавшись этой уязвимостью, злоумышленник может захватить полный контроль над системой, Таким образом, злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.

Это обновление является критическим для всех поддерживаемых выпусков Windows XP и Windows Vista. Для поддерживаемых выпусков Windows Server 2003 оно имеет существенный уровень опасности, а для поддерживаемых выпусков Microsoft Windows 2000 — средний. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** этого раздела.

Обновление устраняет уязвимость, изменяя способ обработки ядром Windows структур TCP/IP, содержащих многоадресные и ICMP-запросы. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.**  Корпорация Майкрософт рекомендует незамедлительно установить данное обновление.

**Известные проблемы.** Отсутствует

Вопросы и ответы о данном обновлении безопасности
-------------------------------------------------

<span></span>
**Почему этот бюллетень был исправлен 25 января 2008 г.?**
Бюллетень обновлен для внесения ясности в вопрос о воздействии уязвимости при обработке ядром Windows структур TCP/IP, содержащих IGMPv3- и MLDv2-запросы (CVE-2007-0069), в поддерживаемых выпусках Windows Small Business Server 2003 и Windows Home Server. Необходимое обновление для систем Windows Small Business Server 2003 и Windows Home Server можно загрузить с веб-узлов Центра обновления Майкрософт, Центра обновления Windows, анализатора безопасности Microsoft Baseline Security Analyzer и сервера Microsoft Systems Management Server. Чтобы обеспечить безопасность компьютеров с поддерживаемыми версиями указанных систем, необходимо установить данное обновление.

**Почему это обновление устраняет несколько уязвимостей системы безопасности, о которых уже сообщалось?**  
Это обновление разработано для нескольких уязвимостей, поскольку для устранения их причин требуется выполнить изменения в связанных файлах. Вместо установки нескольких почти одинаковых обновлений пользователям необходимо установить только данное обновление.

**На моем компьютере установлена более ранняя версия программного обеспечения, описанного в этом бюллетене по безопасности. Что мне следует сделать?**  
Продукты, перечисленные в этом бюллетене, проверены на наличие уязвимости в тех или иных версиях. Жизненные циклы поддержки прочих версий программного обеспечения истекли. Уязвимость обработки ядром Windows структур TCP/IP, содержащих IGMPv3- и MLDv2-запросы, — CVE-2007-0069  
-------------------------------------------------------------------------------------------------------
  
<span></span>
В способе обработки ядром Windows структур TCP/IP, хранящих сведения о состоянии IGMPv3- и MLDv2-запросов, существует уязвимость, которая делает возможным удаленное выполнение кода. Поддерживаемые выпуски Microsoft Windows XP, Windows Server 2003 и Windows Vista поддерживают обмен IGMPv3-запросами. Помимо этого система Windows Vista поддерживает MDLv2-запросы, которые позволяют выполнять многоадресную рассылку в IPv6-сетях. Анонимный злоумышленник может воспользоваться ею по сети, отправив на компьютер специально созданные IGMPv3- и MLDv2-пакеты. Воспользовавшись этой уязвимостью, злоумышленник может захватить полный контроль над системой, Таким образом, злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.
  
В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2007-0069](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-0069).
  
#### Факторы, снижающие опасность уязвимости обработки ядром Windows структур TCP/IP, содержащих IGMPv3- и MLDv2-запросы, — CVE-2007-0069
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Стандартные параметры конфигурации брандмауэра позволяют защитить сеть от атак из-за пределов среды организации. На непосредственно подключенных к Интернету системах рекомендуется держать открытыми минимально необходимое количество портов. Брандмауэры периметра сети, блокирующие многоадресный трафик (особенно IGMPv3- и MLDv2-запросы), позволяют защитить активы сети от атак из-за пределов среды организации.  
-   Система Microsoft Windows 2000 не подвержена этой уязвимости.  
-   В системе Windows Server 2003 адреса многоадресной рассылки неактивны по умолчанию, поэтому она не подвержена этой уязвимости. Однако установка приложений, выполняющих многоадресную рассылку, делает эту систему уязвимой.  
-   Попытки воспользоваться данной уязвимостью, скорее всего, приведут к отказу в обслуживании. Тем не менее, удаленное выполнение кода будет возможно.
  
#### Методы обхода уязвимости обработки ядром Windows структур TCP/IP, содержащих IGMPv3- и MLDv2-запросы, — CVE-2007-0069
  
К методам обхода уязвимости относятся параметр или изменение конфигурации, которые не приводят к устранению уязвимости, но могут блокировать известные направления атак, прежде чем обновление будет установлено. Корпорация Майкрософт проверила представленные ниже методы обхода и сообщила, снижают ли они функциональные возможности.
  
-   **Отключите обработку IGMP- и MLD-запросов**
  
    Чтобы предотвратить попытки использования уязвимости IGMP-запросов, измените раздел IGMPLevel в реестре.
  
    **Внимание!** Неправильное использование редактора реестра может привести к серьезным проблемам, которые могут потребовать переустановки операционной системы. Корпорация Майкрософт не может гарантировать, что вы сумеете устранить проблемы, возникшие вследствие неправильного использования редактора реестра. Ответственность за работу с редактором реестра несет пользователь. Сведения о порядке редактирования реестра см. в разделе справки «Изменение параметров и их значений» редактора реестра (Regedit.exe) или в разделах справки «Добавление и удаление данных в реестре» и «Редактирование данных реестра» программы Regedt32.exe.
  
    **Примечание**. Корпорация Майкрософт рекомендует перед редактированием реестра сделать его резервную копию.
  
    Чтобы отключить обработку IGMP- и MLD-запросов, выполните указанные ниже действия.
  
    1.  В меню **Пуск** выберите пункт **Выполнить**, введите команду **regedit** и нажмите кнопку **ОК**.  
    2.  Разверните раздел **HKEY\_LOCAL\_MACHINE**.  
    3.  Последовательно разверните разделы **SYSTEM**, **CurrentControlSet** и **Services**.  
    4.  После этого разверните разделы **TCPIP**, **Parameters** и **IGMPLevel**.

-   **Заблокируйте входящий IGMP- и MLD-трафик на брандмауэре Windows Vista**

#### Вопросы и ответы об уязвимости обработки ядром Windows структур TCP/IP, содержащих IGMPv3- и MLDv2-запросы, — CVE-2007-0069
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость связана с запуском программного кода в удаленном режиме. Воспользовавшийся ею злоумышленник может войти в уязвимую систему с более высоким уровнем привилегий и запустить произвольный код. Это позволит ему устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.
  
**В чем причина уязвимости?** В недостаточной проверке ядром Windows сохраняемых сведений о состоянии IGMP-запросов, обрабатываемых протоколом TCP/IP.
  
**Что такое TCP/IP?**  
TCP/IP — набор протоколов связи для обмена данными по сети.
  
**Почему уровень опасности для систем Windows Small Business Server 2003 и Windows Home Server отличается от уровня опасности для системы Windows Server 2003? **  
В системе Windows Server 2003 протокол IGMP по умолчанию отключен, поэтому она подвержена уязвимости в меньшей степени. Поскольку этот протокол в системах Windows Small Business Server 2003 и Windows Home Server включен по умолчанию, данная уязвимость представляет для них больший риск. В связи с повышенным риском уровень опасности для систем Windows Small Business Server 2003 и Windows Home Server оценивается как "критический". Пользователям этих систем рекомендуется установить данное обновление как можно скорее.
  
**Что такое IGMP?**  
Протокол IGMP позволяет выполнять многоадресную рассылку в IPv4-сетях. Многоадресная рассылка — обмен данными по сети между отправителем и несколькими получателями. Протокол IGMP используется для обмена данными о состоянии членства между маршрутизаторами IPv4, поддерживающими многоадресную рассылку и членов групп многоадресной рассылки. Дополнительные сведения об IGMP см. на веб-узле MSDN.
  
**Что такое MLD?**  
MLD (Multicast Listener Discovery) позволяет в подсети IPv6 управлять членством в группах многоадресной рассылки. MLD представляет собой серию из трех протоколов ICMP для передачи сообщений в сети IPv6 (ICMPv6), которая заменяет протокол IGMP, использовавшийся в сетях IPv4. Дополнительные сведения об [MLD](http://msdn2.microsoft.com/en-us/library/ms882958.aspx) см. на веб-узле MSDN.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Воспользовавшийся ею злоумышленник сможет запустить произвольный код. Таким образом, злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Он может создать специальные сетевые пакеты и отправить их в уязвимую систему.
  
**Какие системы в первую очередь подвергаются риску?**    
Клиентские системы. Также риску могут подвергаться любые системы, выполняющие многоадресную рассылку.
  
**Как действует обновление?**    
Оно устраняет уязвимость, изменяя способ обработки ядром Windows структур TCP/IP, содержащих IGMP-запросы.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня безопасности было открыто объявлено о данной уязвимости. Данный бюллетень описывает уязвимость, о которой сообщалось в частном порядке, а также дополнительные проблемы, обнаруженные в результате внутреннего исследования.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.
  
Уязвимость обработки ядром Windows структур TCP/IP, содержащих ICMP-запросы, — CVE-2007-0066  
--------------------------------------------------------------------------------------------
  
<span></span>
В протоколе TCP/IP существует уязвимость отказа в обслуживании, вызванная способом обработки ядром Windows фрагментированных ICMP-уведомлений маршрутизатора. Протокол IRDP по умолчанию отключен. Воспользоваться этой уязвимостью можно только после его включения. В системах Windows 2003 Server и Windows XP протокол RDP может быть включен в настройках DHCP или через реестр. В системе Windows 2000 он может быть включен через реестр. Анонимный злоумышленник может воспользоваться ею по сети, отправив на компьютер специально созданные ICMP-пакеты. Воспользовавшись ею, злоумышленник может сделать так, что система перестанет отвечать на запросы и будет автоматически перезапускаться.
  
В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2007-0066](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-0066).
  
#### Факторы, снижающие опасность уязвимости обработки ядром Windows структур TCP/IP, содержащих ICMP-запросы, — CVE-2007-0066
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   RDP-протокол по умолчанию отключен. Чтобы можно было воспользоваться этой уязвимостью, его необходимо включить. Дополнительные сведения см. [в статье 269734 базы знаний Майкрософт](http://technet2.microsoft.com/windowsserver/en/library/db56b4d4-a351-40d5-b6b1-998e9f6f41c91033.mspx?mfr=true), "RDP-протокол отключен по умолчанию".  
-   Система Windows Vista не подвержена данной уязвимости.

    На уязвимых платформах протокол RDP отключен по умолчанию. Если он включен, используйте указанный ниже метод обхода для его отключения. Чтобы предотвратить попытки использования уязвимости ICMP-запросов, настройте соответствующим образом раздел реестра PerformRouterDiscovery.
  
    **Внимание!** Неправильное использование редактора реестра может привести к серьезным проблемам, которые могут потребовать переустановки операционной системы. Корпорация Майкрософт не может гарантировать, что вы сумеете устранить проблемы, возникшие вследствие неправильного использования редактора реестра. Сведения о порядке редактирования реестра см. в разделе справки «Изменение параметров и их значений» редактора реестра (Regedit.exe) или в разделах справки «Добавление и удаление данных в реестре» и «Редактирование данных реестра» программы Regedt32.exe.

#### Вопросы и ответы об уязвимости обработки ядром Windows структур TCP/IP, содержащих ICMP-запросы, — CVE-2007-0066

Эта уязвимость позволяет провести атаку типа «отказ в обслуживании». Воспользовавшись этой уязвимостью, злоумышленник может заставить систему не отвечать на запросы и перезапускаться. **Что такое TCP/IP?**  
TCP/IP — набор протоколов связи для обмена данными по сети.
  
**Что такое ICMP?**  
Обслуживающий ICMP-протокол позволяет управлять подключениями и маршрутизацией. Дополнительные сведения о возможностях и реализации ICMP-протокола см. в спецификации [RFC 792: Internet Control Message Protocol DARPA Internet Program Protocol Specification](http://www.ietf.org/rfc/rfc0792.txt?number=792).

Он может отправить специально созданный ICMP-пакет в уязвимую систему. Такое сообщение может заставить уязвимую систему перестать отвечать на запросы и автоматически перезагружаться.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Он может создать специальный сетевой пакет и отправить его в уязвимую систему.
  
**Какие системы в первую очередь подвергаются риску?**    
Все системы, в которых включен протокол IRDP.
  
**Как действует обновление?**    
Оно устраняет уязвимость, изменяя способ обработки ядром Windows структур TCP/IP, содержащих ICMP-уведомления маршрутизатора.

Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня безопасности было открыто объявлено о данной уязвимости. **Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленной версии системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели Результаты поиска в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание.** В зависимости от версии операционной системы или установленных программ некоторые из файлов, перечисленных в таблице сведений о файлах, могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, устанавливаемого на компьютере, сравнив ее с версией, приведенной в соответствующей таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

| **/norestart**    | При вводе с параметром /quitet система не перезагрузится после установки, даже если для ее завершения требуется перезагрузка. |

    Эти действия зависят от установленной версии системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.
