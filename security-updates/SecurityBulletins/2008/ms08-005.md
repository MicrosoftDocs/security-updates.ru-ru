Security Bulletin

Бюллетень по безопасности (Майкрософт) MS08-005 - Важное
========================================================

Уязвимость в службах IIS может привести к несанкционированному получению прав (942831)
--------------------------------------------------------------------------------------

Дата публикации: 12 февраля 2008 г. | Дата обновления: 13 февраля 2008 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это существенное обновление устраняет уязвимость в службах IIS, о которой сообщалось в частном порядке. Злоумышленник, который воспользуется ею на локальном компьютере, может получить контроль над системой, что позволит ему устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

Это обновление для системы безопасности является существенным для служб Microsoft IIS 5.0 в системе Microsoft Windows 2000, служб Microsoft IIS 5.1 в системе Windows XP, служб Microsoft IIS 6.0 в системе Windows Server 2003 и служб Microsoft IIS 7.0 в системе Windows Vista. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** данного раздела.

Дополнительные сведения об уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.** Корпорация Майкрософт рекомендует установить это обновление при первой возможности.

**Известные проблемы.** Отсутствует

Уязвимость обработки уведомлений об изменении файлов — CVE-2008-0074  
--------------------------------------------------------------------
  
<span></span>
В способе обработки службами IIS уведомлений об изменении файлов в папках FTPRoot, NNTPFile\\Root и WWWRoot существует уязвимость, приводящая к несанкционированному получению локальных прав. Злоумышленник, который воспользуется ею, сможет выполнять произвольный код в контексте локальной системы. устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.
  
В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2008-0074](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0074).
  
#### Факторы, снижающие опасность уязвимости обработки уведомлений об изменении файлов, — CVE-2008-0074
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   В системах Windows XP Professional с пакетом обновления 2 (SP2) и Windows Server 2003 учетная запись IUSR\_MACHINENAME для анонимного доступа по умолчанию не имеет доступа на запись к папкам WWWRoot.  
-   Специальные файл или папка, которые могут потребоваться для использования этой уязвимости, должны быть созданы локально. Их нельзя создать на FTP-клиенте, подключенном к службе FTP-сервера.  
-   В системах Windows XP, Windows Server 2003 и Windows Vista службы IIS не установлены и не включены по умолчанию.
  
#### Методы обхода уязвимости обработки уведомлений об изменении файлов, — CVE-2008-0074
  
К методам обхода уязвимости относятся параметр или изменение конфигурации, которые не приводят к устранению уязвимости, но могут блокировать известные направления атак, прежде чем обновление будет установлено. Корпорация Майкрософт проверила представленные ниже методы обхода проблемы и сообщила, снижают ли они функциональные возможности.
  
-   **Остановите службы FTP и NNTP в системе Windows Server 2003**
  
    Чтобы воспользоваться уязвимостью, злоумышленник должен иметь доступ на запись к корневым папкам службы FTP или NNTP. По умолчанию учетная запись IIS для анонимного доступа (IUSR\_MACHINENAME) имеет доступ на запись к корневым папкам служб FTP и NNTP. Остановка служб FTP и NNTP предотвратит их реагирование на уведомления об изменении файлов в этих каталогах. Чтобы остановить службы FTP и NNTP выполните указанные ниже действия.
  
    Запустите из командной строки следующие команды:
  
    `net stop msftpsvcnet stop nntpsvc`
  
    **Побочные эффекты** Службы FTP и NNTP перестанут работать.
  
    **Отмена изменений.** Чтобы отменить изменения, выполните указанные ниже действия.
  
    Запустите из командной строки следующие команды:
  
    `net start msftpsvcnet start nntpsvc`
  
-   **Запретите доступ на запись к папкам NNTPRoot, FTPRoot и WWWRoot для любых учетных записей, используемых для запуска ASP-страниц пользователей**.
  
    Убедитесь в том, что к папке WWWRoot запрещен доступ на запись для любых учетных записей, используемых для запуска ASP-страниц пользователей. Чтобы запретить доступ на запись к папкам NNTPRoot, FTPRoot и WWWRoot, выполните указанные ниже действия.
  
    **Примечание**. Убедитесь в том, что в списке управления доступом к корневой папке WWWRoot запрещен доступ на запись для учетных записей для анонимного доступа.
  
    **Примечание**. Приведенные ниже команды запретят доступ на запись к стандартным каталогам FTPRoot и NNTPRoot на компьютере WS2003ENTSP1 для учетной записи IIS для анонимного доступа.
  
    Запустите из командной строки следующие команды:
  
    cacls c:\\inetpub\\ftproot /E /P IUSR\_WS2003ENTSP1:R  
    cacls c:\\inetpub\\ftproot /E /P USERS:R  
    cacls c:\\inetpub\\nntpfile\\root /E /P "ANONYMOUS LOGON":R  
    cacls c:\\inetpub\\nntpfile\\root /E /P EVERYONE:R
  
    **Побочные эффекты** Доступ на запись к папкам NNTPRoot, FTPRoot WWWRoot будет запрещен для любых учетных записей, используемых для запуска пользовательских ASP-страниц.
  
    **Отмена изменений.** Восстановите ранее установленные значения списков ACL для этих папок.
  
#### Вопросы и ответы об уязвимости обработки уведомлений об изменении файлов, — CVE-2008-0074
  
**Какова область воздействия данной уязвимости?**    
Это уязвимость несанкционированного получения локальных прав. Злоумышленник, который воспользуется ею, сможет выполнять произвольный код в контексте локальной системы.
  
**В чем причина уязвимости?** Уязвимость существует в способе обработки службами IIS уведомлений об изменении файлов в папках FTPRoot, NNTPFile\\Root и WWWRoot.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник, который воспользуется этой уязвимостью на локальном компьютере, может получить полный контроль над системой. устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник должен создать или изменить файл только в каталоге FTPRoot, NNTPFile\\Root или WWWRoot на локальном компьютере. Виртуальные каталоги и подкаталоги не подвержены этой уязвимости. Злоумышленнику, действующему удаленно, потребуется отправить сценарий на уязвимый IIS-сервер и запустить его. Отправленный сценарий должен иметь доступ на запись к папке FTPRoot, NNTPFile\\Root или WWWRoot.
  
**Какие системы в первую очередь подвергаются риску?**    
В первую очередь риску использования этой уязвимости подвержены поставщики услуг размещения веб-узлов, разрешающие пользователям размещать на этих узлах ASP-страницы. Кроме того, риску подвержена система Microsoft Windows 2000 с пакетом обновления 4 (SP4) со службами IIS 5.0, поскольку службы IIS в ней включены по умолчанию. Поддерживаемые выпуски Windows XP, Windows Server 2003 и Windows Vista также подвергаются риску, если эти службы включены.
  
**Как действует обновление? **  
Обновление устраняет уязвимость, изменяя способ, которым службы IIS обрабатывают уведомления об изменении файлов в папках FTPRoot, NNTPFile\\Root и WWWRoot.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности? **  
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня безопасности было открыто объявлено о данной уязвимости.

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

-   **Проверка версий файлов**
  
    Эти действия зависят от установленной версии системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.
  
    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.  
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.  
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.  
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
