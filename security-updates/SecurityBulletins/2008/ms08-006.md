Security Bulletin

Бюллетень по безопасности (Майкрософт) MS08-006 - Важное
========================================================

Уязвимость в службах IIS делает возможным удаленное выполнение кода (942830)
----------------------------------------------------------------------------

Дата публикации: 12 февраля 2008 г. | Дата обновления: 20 февраля 2008 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это существенное обновление устраняет уязвимость в службах IIS, о которой сообщалось в частном порядке. В способе обработки службами IIS входных данных на веб-страницах ASP существует уязвимость удаленного выполнения кода. Воспользовавшийся ею злоумышленник сможет выполнять любые действия на сервере IIS с теми же правами, что и рабочий процесс WPI, по умолчанию настроенный с привилегиями учетной записи сетевой службы. Серверы IIS с ASP-страницами, пулы приложений которых настроены с помощью процесса WPI, использующего учетную запись с правами администратора, подвержены более серьезному риску, чем те серверы IIS, для пула приложений которых установлены настройки WPI по умолчанию.

Это обновление для системы безопасности является существенным для служб Microsoft IIS во всех поддерживаемых выпусках Windows XP и Windows Server 2003. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** этого раздела.

Дополнительные сведения о данных уязвимостях см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.** Корпорация Майкрософт рекомендует установить это обновление при первой возможности.

Уязвимость ASP-страниц — CVE-2008-0075  
--------------------------------------
  
<span></span>
В способе обработки службами IIS входных данных веб-страниц ASP существует уязвимость удаленного выполнения кода. Злоумышленник может воспользоваться ею, передав вредоносные входные данные на ASP-страницу веб-узла. Воспользовавшийся ею злоумышленник сможет выполнять любые действия на сервере IIS с теми же правами, что и процесс WPI, по умолчанию настроенный с привилегиями учетной записи сетевой службы.
  
В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2008-0075](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0075).
  
#### Факторы, снижающие опасность уязвимости ASP-страниц, — CVE-2008-0075
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Службы IIS 5.1 не установлены в системе Windows XP Professional с пакетом обновления 2 (SP2) по умолчанию.  
-   В поддерживаемых выпусках Windows Server 2003 службы IIS не установлены или не включены по умолчанию.  
-   В поддерживаемых выпусках Windows Server 2003, в которых включены службы IIS, компонент Classic ASP по умолчанию не установлен и не включен.  
-   Злоумышленник, воспользовавшийся этой уязвимостью в поддерживаемых выпусках Windows Server 2003, в которых включены службы IIS и используется стандартный компонент ASP, по умолчанию может получить только привилегии учетной записи сетевой службы. По умолчанию права учетной записи сетевой службы эквивалентны правам пользователя, прошедшего проверку подлинности.  
-   Среда ASP.NET не подвержена этой уязвимости. Системы, в которых развернута только среда ASP.NET, а компонент ASP не установлен, не подвержены этой уязвимости.
  
#### Методы обхода уязвимости ASP-страниц — CVE-2008-0075
  
К методам обхода уязвимости относятся параметр или изменение конфигурации, которые не приводят к устранению уязвимости, но могут блокировать известные направления атак, прежде чем обновление будет установлено. Корпорация Майкрософт проверила представленные ниже методы обхода проблемы и сообщила, снижают ли они функциональные возможности.
  
-   **Отключите классические сценарии ASP в системе Windows Server 2003**
  
    Чтобы отключить классические сценарии ASP, выполните указанные ниже действия.
  
    1. 1. Откройте меню **Пуск**, щелкните значок **Администрирование**, а затем — **Диспетчер служб IIS**.
  
    2. 2. Щелкните **знак плюса** (+) рядом с именем сервера и выберите папку **Расширения веб-службы**.
  
    3. 3. На правой панели выберите **Active Server Pages** и нажмите кнопку **Запретить.**
  
    **Побочные эффекты**. Работать с классическими ASP-страницами будет невозможно.
  
    **Отмена изменений**.Чтобы отменить изменения, выполните описанные выше действия 1—3, и, выбрав пункт **Active Server Pages**, нажмите кнопку **Разрешить**.
  
#### Вопросы и ответы об уязвимости ASP-страниц — CVE-2008-0075
  
**Какова область воздействия данной уязвимости?**  
Эта уязвимость связана с запуском программного кода в удаленном режиме. Воспользовавшийся ею злоумышленник сможет выполнять действия на сервере IIS с теми же правами, что и рабочий процесс WPI, по умолчанию имеющий права учетной записи сетевой службы.
  
**В чем причина уязвимости?**  
Службы IIS неправильно обрабатывают входные данные ASP-страниц.
  
**Что может сделать злоумышленник, воспользовавшись этой уязвимостью?**  
Он сможет выполнять любые действия на сервере IIS с теми же правами, что и процесс WPI, по умолчанию настроенный с привилегиями учетной записи сетевой службы. Службы, настроенные с привилегиями учетной записи сетевой службы, получают доступ уровня прошедшего проверку пользователя, а не уровня администратора. Дополнительные сведения см. в статье об [учетной записи сетевой службы](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx). Серверы IIS, пул приложений которых настроен с помощью процесса WPI, использующего учетную запись с правами администратора, подвержены большему риску, чем те серверы IIS, для пула приложений которых установлены настройки WPI по умолчанию.
  
Учетная запись сетевой службы использует учетные данные компьютера при удаленной проверке подлинности, но уровень ее привилегий на сервере значительно снижен, поэтому она не имеет прав администратора на локальном компьютере. Дополнительные сведения см. в статьях на английском языке [Учетная запись сетевой службы](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx), [IIS и встроенные учетные записи (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/3648346f-e4f5-474b-86c7-5a86e85fa1ff.mspx), [Настройка рабочих процессов (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/76be66e2-1db1-432e-95be-14ff59c2ec75.mspx) и [Настройка пулов приложений в IIS 6.0 (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/2a231dcb-d786-4b6d-b3ba-bda04061a5e7.mspx).
  
**Каким образом злоумышленник может воспользоваться уязвимостью?**  
Для этого ему необходимо передать специально созданные входные данные на ASP-страницу. ASP-страница может существовать в управляемой среде или может быть отправлена пользователем в эту среду.
  
**Что такое рабочие процессы в IIS?**  
Рабочие процессы — это процессы, запускающие серверные задачи обработки для веб-сервера**.** Сведения о рабочих процессах и их реализации в IIS 6.0 см. в статье [Рабочие процессы (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/29f53968-0115-451f-b26d-5ad58d87b5d1.mspx) (на английском языке). Дополнительные сведения о рабочих процессах и запуске с их помощью веб-приложений в изолированной среде см. в статье [Режим изоляции рабочих процессов (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/333f3410-b566-49bc-8399-57bcc404db8c.mspx) (на английском языке).
  
**Что такое ASP?**  
ASP-страницы — это технология серверных сценариев, используемая для создания динамических и интерактивных веб-приложений. ASP-страница — это страница HTML, содержащая сценарии, выполняющиеся на стороне сервера, которые обрабатываются веб-сервером перед отправкой в обозреватель пользователя.
  
**Какие системы в первую очередь подвергаются риску?**  
Это веб-серверы Windows Server 2003 и Windows XP Professional со включенными службами IIS и сценариями ASP. Кроме того, повышенному риску подвержены поставщики услуг хостинга веб-узлов, разрешающие пользователям размещать на них ASP-страницы.
  
**Как действует обновление?**  
Оно устраняет уязвимость, изменяя способ обработки службами IIS входных данных ASP-страниц.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**  
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня безопасности было открыто объявлено о данной уязвимости.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**  
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленной версии системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание.** В зависимости от версии операционной системы или установленных программ некоторые из файлов, перечисленных в таблице сведений о файлах, могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, устанавливаемого на компьютере, сравнив ее с версией, приведенной в соответствующей таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
