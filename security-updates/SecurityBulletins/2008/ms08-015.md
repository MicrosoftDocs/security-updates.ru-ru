Security Bulletin

Бюллетень по безопасности (Майкрософт) MS08-015 - Критическое
=============================================================

Уязвимость в Microsoft Outlook делает возможным удаленное выполнение кода (949031)
----------------------------------------------------------------------------------

Дата публикации: 11 марта 2008 г. | Дата обновления: 4 июня 2008 г.

**Версия:** 1.5

### Общие сведения

#### Аннотация

Это критическое обновление для системы безопасности устраняет уязвимость приложения Microsoft Office Outlook, о которой сообщалось в частном порядке. Она делает возможным удаленное выполнение кода, если в Outlook передается специально созданный URI mailto. что позволит ему устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора. Этой уязвимостью нельзя воспользоваться при просмотре сообщения электронной почты из области просмотра Outlook.

Данное обновление для системы безопасности является критическим для поддерживаемых выпусков Microsoft Office Outlook 2000 с пакетом обновления 3 (SP3), Outlook 2002 с пакетом обновления 3 (SP3), Outlook 2003 с пакетами обновления 3 (SP3) и 2 (SP2), а также Outlook 2007. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** этого раздела.

Данное обновление для системы безопасности устраняет эти уязвимости, изменяя способ обработки URI mailto приложением Microsoft Outlook. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.** Майкрософт рекомендует пользователям установить обновление немедленно.

**Известные проблемы.** [В статье 949031 базы знаний Майкрософт](http://support.microsoft.com/kb/949031) содержится список известных проблем, которые могут возникнуть при установке этого обновления для системы безопасности, и рекомендуемые способы их решения. Для случаев, когда известные проблемы и методы их решения относятся к определенным выпускам программного обеспечения, предоставляются ссылки на соответствующие статьи.

Описываемая в данном обновлении для системы безопасности уязвимость существует в пакете Microsoft Office, однако ею можно воспользоваться только с помощью приложений, перечисленных в таблице уязвимого программного обеспечения. Уязвимость URI в Outlook — CVE-2008-0110  
----------------------------------------
  
<span></span>
В Outlook существует уязвимость, которая делает возможным удаленное выполнение кода, Она делает возможным удаленное выполнение кода, если в Outlook передается специально созданный URI mailto. что позволит ему устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.
  
В перечне CVE этой уязвимости присвоен номер [CVE-2008-0110](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0110).
  
#### Факторы, снижающие опасность уязвимости URI в Outlook, — CVE-2008-0110
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   В случае атаки через Интернет злоумышленник может создать вредоносный веб-узел и убедить пользователя щелкнуть специально созданный URI mailto. В любом случае злоумышленник не может заставить пользователей посетить эти веб-узлы. Он старается склонить их к этому, убеждая, как правило, щелкнуть ссылку, ведущую на веб-узел, в сообщении электронной почты или программы обмена мгновенными сообщениями.  
-   Уязвимость не может быть использована автоматически посредством электронной почты. Чтобы атака достигла цели, пользователь должен открыть сообщение электронной почты, содержащее специально созданный URI mailto, и щелкнуть ссылку на электронную почту.  
-   Этой уязвимостью нельзя воспользоваться при просмотре сообщения электронной почты из области просмотра Outlook.
  
#### Методы обхода уязвимости URI в Outlook — CVE-2008-0110
  
-   **Отключите обработчик mailto в Outlook**
  
    Отключение обработчика mailto в разделе системного реестра защищает уязвимые системы от попыток использовать эту уязвимость.
  
    **Примечание.** Неправильное использование редактора реестра может привести к серьезным проблемам, для устранения которых может потребоваться переустановка операционной системы. Корпорация Microsoft не гарантирует решение проблем, возникших в результате неправильной работы с редактором реестра. Ответственность за использование редактора реестра несет пользователь. Сведения о порядке редактирования реестра см. в разделе справки «Изменение параметров и их значений» редактора реестра (Regedit.exe) или в разделах справки «Добавление и удаление данных в реестре» и «Редактирование данных реестра» программы Regedt32.exe.
  
    Выполните указанные ниже действия.
  
    1.  В меню **Пуск** выберите пункт **Выполнить**, введите команду **regedit** и нажмите кнопку **ОК**.  
    2.  Разверните раздел **HKEY\_CLASSES\_ROOT**.  
    3.  Разверните подраздел **mailto**.  
    4.  Чтобы создать резервную копию реестра перед его изменением, в меню **Файл** выберите пункт **Экспорт...** В диалоговом окне **Экспорт файла реестра** введите **mailto\_backup.reg** и нажмите кнопку **Сохранить**.  
    5.  Щелкните правой кнопкой мыши mailto и выберите пункт **Удалить**.
  
    **Побочные эффекты:** В системах, в которых применен этот метод обхода, не будут обрабатываться mailto-запросы, обычно запускающие mailto-действия в приложениях.
  
    **Отмена изменений**
  
    1.  В меню **Пуск** выберите пункт **Выполнить**, введите команду **regedit** и нажмите кнопку **ОК**.  
    2.  Разверните раздел **HKEY\_CLASSES\_ROOT**.  
    3.  Разверните подраздел **mailto**.  
    4.  Чтобы восстановить реестр, щелкните меню **Файл** выберите команду **Импорт...** В диалоговом окне **Импорт файла реестра** выберите **mailto\_backup.reg** и нажмите кнопку **Открыть**.
  
#### Вопросы и ответы об уязвимости URI в Outlook — CVE-2008-0110
  
**Какова область воздействия данной уязвимости? **  
Эта уязвимость делает возможным удаленное выполнение кода в приложении Outlook. Воспользовавшись этой уязвимостью, злоумышленник может захватить полный контроль над системой,
  
**В чем причина уязвимости? **  
Приложение Microsoft Outlook выполняет недостаточную проверку URI mailto, переданных клиенту Outlook.
  
**Что может сделать злоумышленник, воспользовавшись этой уязвимостью? **  
Воспользовавшись ею, злоумышленник может выполнить произвольный код в контексте вошедшего в систему пользователя. Если пользователь вошел в систему с правами администратора, злоумышленник может получить полный контроль над ней. что позволит ему устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.
  
**Каким образом злоумышленник может воспользоваться уязвимостью? **  
Атака возможна только в том случае, если в уязвимое приложение Microsoft Outlook будет передан искаженный URI mailto.
  
В случае атаки через Интернет злоумышленник может создать вредоносный веб-узел и убедить пользователя щелкнуть специально созданный URI mailto. Злоумышленник не может заставить пользователей посетить специальный веб-узел. Вместо этого он старается склонить его посетить веб-узел, убеждая, как правило, щелкнуть ссылку.
  
**Какие системы в первую очередь подвергаются риску? **  
Клиенты для настольных компьютеров и серверы терминалов, в которых используется приложение Microsoft Outlook.
  
**Как действует обновление? **  
Оно устраняет уязвимость, изменяя способ, которым клиент Microsoft Outlook проверяет переданные ему URI mailto.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности? **  
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня безопасности было открыто объявлено о данной уязвимости. Данный бюллетень описывает уязвимость, о которой сообщалось в частном порядке, а также дополнительные проблемы, обнаруженные в результате внутреннего исследования.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками? **  
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.
