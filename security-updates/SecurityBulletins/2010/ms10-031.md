Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-031 - Критическое
=============================================================

Уязвимость в среде Microsoft Visual Basic для приложений делает возможным удаленное выполнение кода (978213)
------------------------------------------------------------------------------------------------------------

Дата публикации: 11 мая 2010 г. | Дата обновления: 19 мая 2010 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в среде Microsoft Visual Basic для приложений. Эта уязвимость делает возможным удаленное выполнение кода в том случае, если ведущее приложение откроет особым образом созданный файл и передаст его в среду выполнения Visual Basic для приложений. Если пользователь вошел в систему с правами администратора, то злоумышленник, которому удалось воспользоваться уязвимостью, может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

Это обновление для системы безопасности имеет уровень серьезности "критический" для всех поддерживаемых версий Microsoft Visual Basic для приложений с пакетом SDK и сторонних приложений, использующих Microsoft Visual Basic для приложений. Это обновление для системы безопасности также имеет уровень серьезности "существенный" для всех поддерживаемых выпусков Microsoft Office XP, Microsoft Office 2003 и выпуска 2007 системы Microsoft Office. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Это обновление устраняет уязвимость путем изменения способа поиска внедренных в документы элементов управления ActiveX, который используется в Visual Basic для приложений. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация.** Майкрософт рекомендует пользователям установить обновление немедленно.

**Известные проблемы.** В [статье 978213 базы знаний Майкрософт](http://support.microsoft.com/kb/978213) описаны известные на данный момент проблемы, с которыми пользователи могут столкнуться при установке этого обновления для системы безопасности. В этой статье также приводятся решения, рекомендованные для устранения этих проблем. Для случаев, когда известные проблемы и методы их решения относятся к определенным выпускам программного обеспечения, предоставляются ссылки на соответствующие статьи.

Вопросы и ответы о данном обновлении безопасности  
-------------------------------------------------
  
<span></span>
**Почему выпущена новая редакция этого бюллетеня от 19 мая 2010 г.?**    
Корпорация Майкрософт выпустила новую редакцию этого бюллетеня, чтобы яснее подчеркнуть, что обновления для Microsoft Office применяются ко всем поддерживаемым пакетам Microsoft Office и другим программам Microsoft Office, содержащим уязвимую версию VBE6.dll. В число таких программ входят, в частности, поддерживаемые версии Microsoft Office Visio и Microsoft Office Project. В это число входят также программы Office, которые содержат библиотеку VBE6.dll, но не используют уязвимый код, — например, поддерживаемые версии Microsoft Office Outlook и Microsoft Office OneNote.
  
Это изменение касается только текста бюллетеня. Алгоритм обнаружения и файлы обновления не изменялись. Пользователям, которые уже успешно обновили свои компьютеры, не требуется выполнять никаких действий (это относится также и к пользователям, у которых на компьютере включена функция автоматического обновления). Пользователи, которые ранее не установили это обновление, могут изменить решение о необходимости установки этого обновления на основании измененной таблицы "Подвержены уязвимости" и раздела часто задаваемых вопросов для этого обновления.
  
**Обновление затрагивает общий компонент, используемый в Microsoft Office. Будет ли это обновление предлагаться для программы Microsoft Office, включающей в себя этот общий компонент, если эта программа не использует уязвимый код?**    
Да, обновление будет предлагаться для систем, в которых обнаружен уязвимый общий компонент Office. Это относится даже к случаям, когда программа Office содержит библиотеку VBE6.dll, но не использует уязвимый код. Примеры программ Office, к которым применимо это обновление, приведены в таблице ниже.
  
| Microsoft Office                                                                            | Входит ли библиотека VBE6.dll в состав этой программы? | Использует ли программа уязвимый код?               |  
|---------------------------------------------------------------------------------------------|--------------------------------------------------------|-----------------------------------------------------|  
| Microsoft Office XP, Microsoft Office 2003, выпуск 2007 системы Microsoft Office            | Да                                                     | Да (такие компоненты, как Word, Excel и PowerPoint) |  
| Microsoft Office OneNote 2003, Microsoft Office OneNote 2007                                | Да                                                     | Нет                                                 |  
| Microsoft Office Outlook 2003, Microsoft Office Outlook 2007                                | Да                                                     | Нет                                                 |  
| Microsoft Office Project 2002, Microsoft Office Project 2003, Microsoft Office Project 2007 | Да                                                     | Да                                                  |  
| Microsoft Office Visio 2002, Microsoft Office Visio 2003, Microsoft Office Visio 2007       | Да                                                     | Да                                                  |
  
**Примечание** Используйте эту таблицу только в качестве общей рекомендации. В нее включены в качестве примеров программы, вопросы о которых задают чаще всего.
  
В тех случаях, когда обновление предлагается, Майкрософт рекомендует пользователям установить это обновление независимо от того, использует ли данная программа уязвимый код. Установка обновления позволяет гарантировать, что у пользователей установлена самая последняя версия библиотеки VBE6.dll, используемой программой Microsoft Office.
  
**Является ли предложение обновления версии ПО, не считающейся уязвимой, ошибкой работы механизма обновления Майкрософт?**    
Нет. Это не является ошибкой. Установить обновление предлагается после обнаружения в системе файлов более ранних версий, чем те, которые содержатся в пакете обновления.
  
**Где находятся дополнительные сведения о файлах?**    
Местонахождение дополнительных сведений о файлах см. в справочных таблицах раздела **Развертывание обновления для системы безопасности**.
  
**Я применил необходимые обновления безопасности Майкрософт, но на моем компьютере по-прежнему установлена уязвимая версия среды выполнения Visual Basic для приложений (VBE6.dll). Как обновить эту DLL-библиотеку?**    
Есть случаи, когда на компьютере будет оставаться уязвимая версия VBE6.dll даже после установки необходимых обновлений безопасности для Microsoft Office и обновления для Microsoft Visual Basic для приложений, описанного в данном бюллетене.
  
Если библиотека VBE6.dll установлена поддерживаемой версией Microsoft Office, то при применении обновления безопасности к уязвимой версии Microsoft Office библиотека VBE6.dll будет заменена на обновленную версию, которая устраняет уязвимость, описанную в данном бюллетене. Но если библиотека VBE6.dll установлена на компьютер сторонним приложением, может потребоваться установка обновления для этой программы.
  
Обновить библиотеку VBE6.dll для сторонних приложений можно двумя различными способами — в зависимости от того, каким образом реализована среда VBA в стороннем приложении. Если вы уверены, что стороннее приложение соответствует рекомендованным лучшим решениям по использованию общего компонента в качестве параллельной сборки, то в случае применения обновления для Microsoft Visual Basic для приложений (KB974945) библиотека VBE6.dll в общей папке будет заменена на обновленную версию, которая устраняет уязвимость, описанную в этом бюллетене.
  
С другой стороны, если стороннее приложение не помещает библиотеку VBE6.dll в общую папку, как это рекомендуется в лучших решениях, обратитесь к разработчику стороннего приложения и попросите предоставить обновленную версию приложения, в которой содержится новая версия библиотеки VBE6.dll, которая устраняет уязвимость, описанную в данном бюллетене. Обратите внимание, что не каждое стороннее приложение, которое устанавливает библиотеку VBE6.dll, подвержено уязвимости, описанной в данном бюллетене, так как приложение может не использовать библиотеку VBE6.dll для поиска элементов управления ActiveX тем способом, который представляет опасность. Однако определить это может только разработчик программы.
  
**Данное обновление для системы безопасности применимо только к программному обеспечению Майкрософт. Как определить, что сторонние приложения установили на компьютер уязвимую версию среды выполнения Visual Basic для приложений (VBE6.dll)?**    
Сторонние приложения, поддерживающие VBA, могут установить библиотеку VBE6.dll в папку, которая не обновляется данным обновлением безопасности. Если у вас установлено стороннее приложение, которое поставляется с отдельным экземпляром VBE6.dll, то для обеспечения полной защиты компьютера от уязвимости, описанной в данном бюллетене, необходимо обратиться напрямую к разработчику или поставщику, отвечающему за поддержку данного стороннего приложения.
  
Инструкции по обнаружению на компьютере экземпляров библиотеки VBE6.dll см. в [статье 978213 базы знаний Майкрософт](http://support.microsoft.com/kb/978213). Обнаружение всех экземпляров VBE6.dll на компьютере может помочь выявить потенциально уязвимые экземпляры, которые могли быть установлены сторонними приложениями.
  
**Я занимаюсь разработкой приложений и использую в своем приложении среду выполнения Microsoft Visual Basic для приложений. Уязвимо ли оно и как его обновить?**    
Разработчики, которые распространяют библиотеку VBE6.dll среды выполнения Microsoft Visual Basic для приложений, должны обязательно обновить версию Microsoft Visual Basic для приложений, установленную в своем приложении, загрузив обновленную версию Microsoft Visual Basic для приложений с пакетом SDK с веб-сайта компании [Summit Software](http://www.summsoft.com/). Дополнительные сведения о рекомендациях по использованию распространяемого компонента см. в [статье 835322 базы знаний Майкрософт](http://support.microsoft.com/kb/835322) и в статье MSDN, посвященной [изолированным приложениям и параллельным сборкам](http://msdn.microsoft.com/en-us/library/aa375193.aspx).
  
**Я независимый поставщик ПО. Где можно взять обновление для Microsoft Visual Basic для приложений с пакетом SDK?**    
Обновление можно загрузить с веб-сайта компании [Summit Software](http://www.summsoft.com/). Компания Summit Software — это глобальный поставщик программных продуктов для доработки приложений и услуг по поддержки интеграции, которые предназначены для независимых поставщиков ПО (ISV) и корпоративных разработчиков. В июне 1996 г. компания Summit Software заключила соглашение с Microsoft относительно продажи среды Microsoft Visual Basic для приложений и сопутствующих технологий и услуг. Компания Summit по-прежнему занимается продажами и поддержкой Microsoft VBA.
  
**Почему это обновление имеет уровень серьезности "критический" для Microsoft Visual Basic для приложений, но всего лишь уровень "существенный" для Microsoft Office?**    
В приложении Microsoft Office XP и более поздних версиях имеется функция, которая перед открытием документа выводит запрос с вариантами выбора "Открыть", "Сохранить" и "Отмена". Этот смягчающий фактор снижает уровень серьезности этой уязвимости с "критического" до "существенного", поскольку для проведения атаки с ее использованием необходимо выполнение пользователем более одного действия.
  
**На компьютере установлен выпуск Microsoft Office 2007 с пакетом обновления 1 (SP1). Включены ли в это обновление какие-либо дополнительные функции безопасности?**    
Да, согласно порядку предоставления обновлений для Microsoft Office 2007 при установке обновления для Microsoft Office 2007 с пакетом обновления 1 (SP1) система приобретает функции безопасности, которые изначально входили в состав пакета обновления 2 (SP2) для Microsoft Office 2007. Все обновления для Microsoft Office 2007, выпущенные после 24 апреля 2009 г., содержат функции безопасности, представленные в пакете обновления 2 (SP2) для Microsoft Office 2007. Уязвимость стека VBE6.dll, приводящая к повреждению памяти (CVE-2010-0815)
--------------------------------------------------------------------------

<span></span>
В способе поиска элементов управления ActiveX, используемом средой Microsoft Visual Basic для приложений, имеется уязвимость, которая делает возможным удаленное выполнение кода. Эта уязвимость делает возможным удаленное выполнение кода в том случае, если ведущее приложение откроет особым образом созданный файл и передаст его в среду выполнения Visual Basic для приложений. Если пользователь вошел в систему с правами администратора, то злоумышленник, которому удалось воспользоваться уязвимостью, может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2010-0815](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0815).

#### Смягчающие факторы для уязвимости стека VBE6.dll, приводящей к повреждению памяти (CVE-2010-0815)

К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.

-   По умолчанию выпуск 2007 системы Microsoft Office отображает предупреждение безопасности перед тем, как активировать элемент управления ActiveX, внедренный в документ Office. Компьютеры пользователей, которые отказались от включения элемента управления, защищены благодаря этому параметру по умолчанию. Дополнительные сведения см. в статье на веб-сайте Microsoft Office Online [Включение и выключение элементов управления ActiveX в документах Office](http://office.microsoft.com/en-us/help/ha100310671033.aspx).
-   Злоумышленник, успешно воспользовавшийся данной уязвимостью, может получить те же самые права, которые имеет локальный пользователь. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.
-   В случае атаки через Интернет злоумышленник может разместить в сети веб-сайт, содержащий веб-страницу, предназначенную для использования данной уязвимости. Кроме того, веб-сайты, подвергшиеся атаке злоумышленников, равно как и веб-сайты, принимающие или размещающие пользовательские материалы, также могут иметь специальное содержимое, рассчитанное на использование данной уязвимости. Однако в любом случае злоумышленник не может заставить пользователей посетить эти веб-сайты. Вместо этого он пытается склонить пользователей к посещению этого веб-сайта, убеждая, как правило, щелкнуть соответствующую ссылку в сообщении электронной почты или запросе программы обмена мгновенными сообщениями, а затем открыть специально созданный файл Office.
-   Уязвимость не может быть использована автоматически посредством электронной почты. Чтобы атака злоумышленника посредством электронной почты прошла успешно, пользователь должен открыть вложение, отправленное в сообщении электронной почты.

#### Временные решения для уязвимости стека VBE6.dll, приводящей к повреждению памяти (CVE-2010-0815)

К методам обхода уязвимости относятся параметр или изменение конфигурации, которые не приводят к устранению уязвимости, но могут блокировать известные направления атак, прежде чем обновление будет установлено. Корпорация Майкрософт проверила представленные ниже методы обхода проблемы и сообщила, снижают ли они функциональные возможности.

-   **Отключение элементов управления ActiveX в выпуске 2007 системы Microsoft Office**

    Можно отключить элементы управления ActiveX в выпуске 2007 системы Microsoft Office с помощью **TrustCenter**. Инструкции по отключению всех элементов управления без уведомления см. в статье на веб-сайте Microsoft Office Online [Включение и выключение элементов управления ActiveX в документах Office](http://office.microsoft.com/en-us/help/ha100310671033.aspx).

    **Побочные эффекты использования временного решения.** Внедренные элементы управления ActiveX (например, макросы) не будут выполняться в документах Office.

-   **Ограничение доступа к файлу VBE6.dll**

    Для Microsoft Windows 2000, Windows XP, и Windows Server 2003: введите в командной строке следующую команду:

    `cacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /E /P everyone:N`

    Для Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2: введите в командной строке с повышенными правами следующую команду:

    `takeown /f "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll"icacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /save %TEMP%\VBE6_ACL.TXTicacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /deny everyone:(F)`

    **Побочные эффекты использования временного решения.** Внедренные элементы управления ActiveX (например, макросы) не будут выполняться в документах Office. Например, пользователи не смогут вставлять объекты в документы Office.

    **Отмена изменений.**

    Для Microsoft Windows 2000, Windows XP, и Windows Server 2003: введите в командной строке следующую команду:

    `cacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /E /R everyone`

    Для Windows Vista, Windows Server 2008, Windows 7, and Windows Server 2008 R2: введите в командной строке следующую команду:

    `icacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\" /restore %TEMP%\VBE6_ACL.TXT`

-   **Открывайте файлы из неизвестного или ненадежного источника в среде MOICE**

    Среда MOICE позволяет защитить приложение Office 2003 путем усиления защиты при открытии файлов двоичных форматов Word, Excel и PowerPoint.

    Для установки среды MOICE необходимо наличие на компьютере пакета Office 2003 или выпуска 2007 системы Microsoft Office.

#### Часто задаваемые вопросы об уязвимости стека VBE6.dll, приводящей к повреждению памяти (CVE-2010-0815)

**Какова область воздействия данной уязвимости?**  
Это уязвимость Microsoft Visual Basic для приложений (VBA), которая делает возможным удаленное выполнение кода. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

**В чем причина уязвимости?** Эта уязвимость вызвана используемым в Visual Basic для приложений способом поиска элементов управления ActiveX в документе, поддерживающем VBA. В результате, ведущее приложение (например, Microsoft Office или стороннее приложение, разработанное для программирования с Visual Basic) может передать особым образом созданный документ с внедренными элементами управления ActiveX в среду выполнения VBA, что создаст возможность выполнения произвольного кода.

**Что такое Visual Basic для приложений (VBA)?**  
Среда Microsoft VBA — это технология разработки пакетных приложений для клиентских настольных компьютеров и их интеграции в существующие данные и системы. Среда Microsoft VBA основана на системе разработки Microsoft Visual Basic. Среда VBA входит в состав приложений пакета Microsoft Office и используется в них для выполнения определенных функций. Ее также можно использовать для разработки индивидуальных приложений на основе существующего ведущего приложения.

**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**  
Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

Чтобы атака была успешной, пользователь должен открыть особым образом созданный документ, который поддерживает VBA. Документ может представлять собой любой документ, в котором поддерживается VBA (например документ Word, электронная таблица Excel или презентация PowerPoint).

Для этого при атаке через электронную почту он может отправить такой файл Office и убедить пользователя открыть его.

В случае атаки через Интернет на веб-сайте злоумышленника должен находиться документ, поддерживающий VBA (например, файл Office), предназначенный для использования этой уязвимости. Кроме того, веб-сайты, подвергшиеся атаке злоумышленников, равно как и веб-сайты, принимающие или размещающие пользовательские материалы, также могут иметь специальное содержимое, рассчитанное на использование данной уязвимости. Злоумышленник не может заставить пользователей посетить специальный веб-сайт. Вместо этого он пытается склонить пользователей к посещению этого веб-сайта, убеждая, как правило, щелкнуть соответствующую ссылку в сообщении электронной почты или запросе программы обмена мгновенными сообщениями, а затем открыть специально созданный файл Office.

**Какие системы в первую очередь подвергаются риску?**  
В первую очередь риску подвергаются рабочие станции и серверы терминалов, на которых используется Microsoft Visual Basic для приложений. Серверы могут быть подвержены большему риску, если администраторы позволяют пользователям входить в систему на сервере и запускать программы. Однако общепринятой практикой настоятельно рекомендуется не предоставлять пользователям таких полномочий.

**Как действует обновление?**  
Это обновление устраняет уязвимость путем изменения способа поиска внедренных в документы элементов управления ActiveX, который используется в Microsoft Visual Basic для приложений.

**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**  
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.

**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**  
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.
