Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-025 - Критическое
=============================================================

Уязвимость служб Microsoft Windows Media делает возможным удаленное выполнение кода (980858)
--------------------------------------------------------------------------------------------

Дата публикации: 13 апреля 2010 г. | Дата обновления: 27 апреля 2010 г.

**Версия:** 3.0

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в службах Windows Media, работающих в ОС Microsoft Windows 2000 Server. Эта уязвимость позволяет злоумышленнику удаленно выполнять код, если он отправит особым образом созданный пакет с транспортной информацией на компьютер под управлением Microsoft Windows 2000 Server, на котором запущены службы Windows Media. Стандартные параметры конфигурации брандмауэра позволяют защитить сеть от атак из-за пределов среды организации. На непосредственно подключенных к Интернету системах рекомендуется держать открытыми минимально необходимое количество портов. В Microsoft Windows 2000 Server службы Windows Media являются дополнительным компонентом и по умолчанию не устанавливаются.

Это обновление для системы безопасности имеет уровень серьезности "критический" для всех поддерживаемых выпусков Microsoft Windows 2000 Server, в которых запущены службы Windows Media. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Обновление для системы безопасности повторно выпущено 27 апреля 2010 г. Повторно выпущенное обновление устраняет уязвимость, изменяя способ, используемый службой Windows Media Unicast (nsum.exe) для обработки пакетов с информацией о транспорте. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которые хотят применить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив его наличие в [Центре обновления Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

Вопросы и ответы о данном обновлении безопасности
-------------------------------------------------

<span></span>
**Почему выпущена новая редакция этого бюллетеня от 27 апреля 2010 года?**  
В новой редакции бюллетеня предлагается новая редакция обновления для системы безопасности для служб Windows Media, запущенных в Microsoft Windows 2000 Server с пакетом обновления 4 (SP4). Исходное обновление для системы безопасности не обеспечивало защиту систем от уязвимости, описанной в этом бюллетене. Майкрософт рекомендует пользователям, у которых установлено уязвимое программное обеспечение, немедленно применить повторно выпущенное обновление для системы безопасности.

**Надо ли мне удалять исходное обновление безопасности перед установкой новой редакции обновления?**  
Нет, удалять исходное обновление для системы безопасности перед установкой новой редакции обновления не требуется.

**Почему выпущена новая редакция этого бюллетеня от 21 апреля 2010 года?**  
Корпорация Майкрософт выпустила новую редакцию этого бюллетеня, чтобы объявить, что исходное обновление безопасности для Microsoft Windows 2000 Server (KB980858) более недоступно из-за проблем с качеством. Исходное обновление для системы безопасности не обеспечивало защиту систем от уязвимости, описанной в этом бюллетене. Майкрософт рекомендует пользователям применить одно из временных решений, описанных в этом бюллетене, для снижения степени воздействия на уязвимые системы до тех пор, пока не будет доступно исправленное обновление безопасности.

**Где находятся дополнительные сведения о файлах?**  
Местонахождение дополнительных сведений о файлах см. в справочных таблицах раздела **Развертывание обновления для системы безопасности**.

**На моем компьютере не запущены службы Windows Media. Будет ли мне предложено это обновление?**  
Нет, для компьютеров под управлением Microsoft Windows 2000 Server, на которых не запущены дополнительные службы Windows Media, это обновление для системы безопасности не предлагается. Если службы Windows Media запущены, это обновление для системы безопасности будет предложено.

Уязвимость служб Windows Media, приводящая к переполнению стекового буфера (CVE-2010-0478)  
------------------------------------------------------------------------------------------
  
<span></span>
В Microsoft Windows 2000 Server с пакетом обновления 4 (SP4), в котором запущен дополнительный компонент служб Windows Media, существует уязвимость, делающая возможным удаленное выполнение кода; эта уязвимость вызвана способом, используемым службой Windows Media Unicast для обработки особым образом созданных пакетов с информацией о транспорте. В Microsoft Windows 2000 Server с пакетом обновления 4 (SP4) службы Windows Media являются дополнительным компонентом и по умолчанию не устанавливаются. Этой уязвимости подвержены только компьютеры под управлением Microsoft Windows 2000 Server, в которых включены службы Windows Media.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2010-0478](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0478).
  
#### Смягчающие факторы для уязвимости служб Windows Media, приводящей к переполнению стекового буфера (CVE-2010-0478)
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Поддерживаемые выпуски Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2 не подвержены уязвимости, описанной в этом бюллетене.  
-   В Microsoft Windows 2000 Server службы Windows Media по умолчанию не включены. Чтобы уязвимость представляла опасность для Microsoft Windows 2000 Server, сервер должен быть настроен в качестве сервера потоковой передачи мультимедиа путем добавления компонента служб Windows Media в мастере компонентов Windows.
  
#### Временные решения для уязвимости служб Windows Media, приводящей к переполнению стекового буфера (CVE-2010-0478)
  
К методам обхода уязвимости относятся параметр или изменение конфигурации, которые не приводят к устранению уязвимости, но могут блокировать известные направления атак, прежде чем обновление будет установлено. Корпорация Майкрософт проверила представленные ниже методы обхода проблемы и сообщила, снижают ли они функциональные возможности.
  
-   **Остановка и отключение службы Windows Media Unicast**
  
    Выполнив вход с правами администратора, отключите службу Windows Media Unicast, введя в командной строке следующую программу:  
    `sc stop nsunicast & sc config nsunicast start= disabled`
  
    **Побочные эффекты использования временного решения.** Не будут разрешены подключения к серверу потоковой передачи мультимедиа через службу Windows Media Unicast.
  
    **Отмена изменений.**
  
    Выполнив вход с правами администратора, включите службу Windows Media Unicast, введя в командной строке следующую программу:  
    `sc config nsunicast start= auto & sc start nsunicast`
  
-   **Удаление компонента служб Windows Media с помощью мастера компонентов Windows**
  
    1.  Выполните вход в систему с правами администратора или члена группы "Администраторы".  
    2.  Щелкните **Start** (Пуск) и выберите **Settings** (Настройка), **Control Panel** (Панель управления).  
    3.  На панели управления дважды щелкните **Установка и удаление программ**.  
    4.  Щелкните **Добавление и удаление компонентов Windows**. Запускается мастер компонентов Windows, и отображается экран "Компоненты Windows".  
    5.  Снимите флажок **Службы Windows Media**. Нажмите кнопку **Далее** и выполните инструкции, отображаемые в мастере компонентов Windows.
  
    **Побочные эффекты использования временного решения. **Сервер больше не будет настроен в качестве сервера потоковой передачи мультимедиа с использованием служб Windows Media.
  
    **Отмена изменений. **Установите службы Windows Media с помощью мастера компонентов Windows.
  
#### Часто задаваемые вопросы об уязвимости служб Windows Media, приводящей к переполнению стекового буфера (CVE-2010-0478)
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость связана с запуском программного кода в удаленном режиме. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами.
  
**В чем причина уязвимости?** Эта уязвимость вызвана способом, используемым службами Windows Media, запущенными в Microsoft Windows 2000 Server с пакетом обновления 4 (SP4), для обработки сетевых пакетов с информацией о транспорте.
  
**Что такое службы Windows Media?**    
Службы Microsoft Windows Media — это отраслевая платформа для передачи потокового аудио/видеоконтента в реальном времени или по запросу через Интернет или интрасеть. Службы Windows Media используются для настройки и управления одним или несколькими серверами Windows Media, которые доставляют контент на клиентские устройства.
  
Вашими клиентскими устройствами могут быть:
  
-   компьютеры или устройства, воспроизводящие контент с помощью проигрывателя (например, проигрыватель Windows Media);  
-   компьютеры с работающими службами Windows Media, используемые для в качестве прокси-серверов, для кэширования или распространения контента;  
-   пользовательские приложения, разработанные с использованием пакета SDK Windows Media.
  
**Какой компонент служб Windows Media подвержен уязвимости?**    
Уязвимый код присутствует в службе Windows Media Unicast (nsum.exe).
  
**Что такое служба Windows Media Unicast?**    
Служба Windows Media Unicast является одной из служб Windows Media — семейства служб, с помощью которых поставщики цифрового контента отправляют потоковое мультимедиа своим клиентам. В это семейство входят четыре службы:
  
-   служба Windows Media Unicast, которая позволяет отправлять потоковое мультимедиа конкретному конечному пользователю;  
-   служба Windows Media Station, которая позволяет отправлять один поток мультимедиа нескольким конечным пользователям одновременно;  
-   служба Windows Media Program, которая контролирует, сколько раз воспроизводится группа потоков;  
-   служба Windows Media Monitor, которая позволяет поставщику цифрового контента выполнять мониторинг конечных пользователей, подключенных к пунктам трансляции.
  
Этой уязвимости подвержена только служба Windows Media Unicast. Эта уязвимость не может быть использована против остальных трех служб.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Чтобы воспользоваться этой уязвимостью, злоумышленник может послать особым образом созданный пакет с информацией о транспорте на компьютер под управлением Microsoft Windows 2000 Server с пакетом обновления 4 (SP4), на котором запущены службы Windows Media. Лучшие решения для брандмауэров и стандартные конфигурации брандмауэра по умолчанию позволяют защитить сети от атак извне пределов периметра корпоративной сети. На непосредственно подключенных к Интернету системах рекомендуется держать открытыми минимально необходимое количество портов.
  
В Microsoft Windows 2000 Server с пакетом обновления 4 (SP4) службы Windows Media являются дополнительным компонентом и по умолчанию не устанавливаются.
  
**Какие системы в первую очередь подвергаются риску?**    
Этой уязвимости подвержены компьютеры под управлением Microsoft Windows 2000 Server с пакетом обновления 4 (SP4), на которых работают службы Windows Media.
  
**Как действует обновление?**    
Это обновление устраняет уязвимость путем изменения способа, используемого службой Windows Media Unicast для обработки пакетов с информацией о транспорте.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**    
Нет. Хотя исходное сообщение было получено в результате компетентного обнаружения, позже о данной уязвимости сообщалось в открытых источниках.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

**Установка обновления**

При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка разделов реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
