Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-104 - Важное
========================================================

Уязвимость в Microsoft SharePoint делает возможным удаленное выполнение кода (2455005)
--------------------------------------------------------------------------------------

Дата публикации: 14 декабря 2010 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в Microsoft SharePoint. Уязвимость делает возможным удаленное выполнение кода в контексте безопасности гостевого пользователя, если злоумышленник отправил особым образом созданный запрос SOAP в Службу запуска для преобразования документов в среде сервера SharePoint, которая использует Службу балансировки нагрузки для преобразования документов. По умолчанию Служба балансировки нагрузки для преобразования документов и Служба запуска для преобразования документов не включены в Microsoft Office SharePoint Server 2007.

Это обновление для системы безопасности имеет уровень серьезности "существенный" для всех поддерживаемых выпусков Microsoft Office SharePoint Server 2007. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** этого раздела.

Данное обновление устраняет эту уязвимость, изменяя способ, которым Служба запуска для преобразования документов проверяет особым образом созданные запросы SOAP. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация**. Корпорация Майкрософт рекомендует установить это обновление при первой возможности.

**Известные проблемы.** Нет.

Уязвимость, связанная с обработкой неверно сформированных запросов и делающая возможным выполнение кода - CVE-2010-3964  
-----------------------------------------------------------------------------------------------------------------------
  
<span></span>
В способе, которым Служба запуска для преобразования документов проверяет запросы SOAP до обработки севером SharePoint, существует уязвимость, делающая возможным удаленное выполнение кода. Злоумышленник, успешно воспользовавшийся этой уязвимостью, может запустить произвольный код на подверженном этой уязвимости сервере SharePoint в контексте безопасности учетной записи гостя.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2010-3964](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3964).
  
#### Смягчающие факторы для уязвимости, связанной с обработкой неверно сформированных запросов и делающей возможным выполнение кода - CVE-2010-3964
  
К смягчающим факторам относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие смягчающие факторы могут снизить опасность использования уязвимости.
  
-   По умолчанию Служба балансировки нагрузки для преобразования документов и Служба запуска для преобразования документов не включены в Microsoft Office SharePoint Server 2007.  
-   Злоумышленник, успешно воспользовавшийся этой уязвимостью, может выполнить код в виде гостевой учетной записи, которая имеет права только на определенный временный каталог. Содержимое этого временного каталога регулярно очищается.
  
#### Смягчающие факторы для уязвимости, связанной с обработкой неверно сформированных запросов и делающей возможным выполнение кода - CVE-2010-3964
  
К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.
  
-   **Остановка и отключение Службы запуска для преобразования документов**
  
    В каждой системе, в которой работает Служба запуска для преобразования документов, войдите в систему с правами администратора и в командной строке введите:  
    `sc stop dclauncher`  
    `sc config dclauncher start= disabled`
  
    **Побочные эффекты.** Преобразование документов Office отключено.
  
    **Отмена временного решения.**
  
    Войдите в систему с правами администратора и в командной строке введите:  
    `sc config dclauncher start= auto`  
    `sc start dclauncher`
  
-   **Блокирование портов, используемых Службой запуска для преобразования документов, на брандмауэре**
  
    Заблокируйте следующий порт на брандмауэре:  
    Порт TCP 8082
  
    Этот порт используется для установки соединения с уязвимым компонентом. Их блокировка на брандмауэре организации позволит защитить системы за брандмауэром от возможных попыток воспользоваться уязвимостью. Для предотвращения возможных атак через другие порты или протоколы корпорация Майкрософт рекомендует блокировать весь непредусмотренный входящий трафик из Интернета. Дополнительные сведения о портах см. в статье TechNet [Назначения портов TCP и UDP](http://go.microsoft.com/fwlink/?linkid=21312).
  
    **Побочные эффекты.** Приложения и службы могут использовать уязвимый порт. Блокирование доступа к портам может привести к тому, что различные службы и приложения перестанут работать.
  
    **Отмена временного решения.**
  
    Разблокируйте порт TCP 8082 на брандмауэре. Дополнительные сведения о портах см. в статье TechNet [Назначения портов TCP и UDP](http://go.microsoft.com/fwlink/?linkid=21312).
  
#### Часто задаваемые вопросы об уязвимости, связанной с обработкой неверно сформированных запросов и делающей возможным выполнение кода - CVE-2010-3964
  
**Какова область воздействия этой уязвимости?**    
Эта уязвимость связана с запуском программного кода в удаленном режиме. Злоумышленник, успешно воспользовавшийся этой уязвимостью, может выполнить произвольный код в контексте учетной записи гостя уязвимого сервера SharePoint.
  
**В чем причина уязвимости?**    
Эта уязвимость вызвана тем, что Служба запуска для преобразования документов неверно проверяет особым образом созданные запросы Simple Object Access Protocol (SOAP).
  
**Что такое Служба запуска для преобразования документов?**    
Служба запуска для преобразования документов планирует и запускает преобразование документов на сервере. Когда Microsoft Office SharePoint Server 2007 передает этой службе запрос на преобразование документа, Служба запуска для преобразования документов вызывает соответствующий конвертер файлов. В большинстве конфигураций ферм серверов Служба запуска для преобразования документов и программы-конвертеры, которые она запускает, должны быть установлены на сервере приложений. На каждом сервере приложений может быть включена только одна Служба запуска для преобразования документов.
  
Служба запуска для преобразования документов работает под учетной записью HVU\_*ComputerName*. Учетная запись HVU\_*ComputerName* является локальной учетной записью гостя, созданной на сервере приложений при включении Службы запуска для преобразования документов.
  
**Что такое Служба балансировки нагрузки для преобразования документов?**    
Служба балансировки нагрузки для преобразования документов балансирует запросы на преобразование документов. Службу балансировки нагрузки для преобразования документов можно настроить на изолированном сервере или на одном сервере в ферме. При получении запроса на преобразование от Microsoft Office SharePoint Server 2007 Служба балансировки нагрузки для преобразования документов возвращает Uniform Resource Identifier (URI), указывающий на соответствующую Службу запуска для преобразования документов. SharePoint Server 2007 связывается с указанной службой запуска при помощи запросов SOAP и делает запрос на преобразование определенного документа.
  
Обратите внимание на то, что отключение Службы балансировки нагрузки для преобразования документов останавливает процесс преобразования документов, а также отключает Службу запуска для преобразования документов.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник, успешно воспользовавшийся этой уязвимостью, может выполнить произвольный код в контексте безопасности учетной записи гостя целевого сервера.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может послать особым образом созданный запрос SOAP в Службу запуска для преобразования документов в среде сервера SharePoint, который использует Службу балансировки нагрузки для преобразования документов. Когда Служба запуска для преобразования документов обрабатывает особым образом созданный запрос SOAP, уязвимость может сделать возможным удаленное выполнение кода в контексте безопасности учетной записи гостя в уязвимой системе.
  
**Какие системы в первую очередь подвержены риску?**    
Среды сервера SharePoint, которые используют Службу балансировки нагрузки для преобразования документов, в первую очередь подвержены риску из-за этой уязвимости. Данная уязвимость делает возможным удаленное выполнение кода на серверах, на которых запущена Служба запуска для преобразования документов в среде сервера SharePoint.
  
**Как действует обновление?**    
Данное обновление устраняет эту уязвимость, изменяя способ проверки особым образом созданных запросов SOAP Службой запуска для преобразования документов.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

Если приложение установлено с сервера, администратор должен установить на сервере административное обновление и перенести его на компьютер пользователя. Данное обновление для системы безопасности требует наличия на компьютере установщика Windows 3.1 или более поздней версии.
  
Чтобы установить установщик Windows 3.1 или более поздней версии, посетите один из перечисленных веб-сайтов Майкрософт.
  
-   [Распространяемый пакет установщика Windows 4.5 для Windows Server 2008, Windows Vista, Windows Server 2003 и Windows XP](http://www.microsoft.com/downloads/details.aspx?familyid=5a58b56f-60b6-4412-95b9-54d056d6f9f4&displaylang=en)  
-   [Распространяемый пакет установщика Windows 3.1 для Windows Server 2003, Windows XP и Windows 2000](http://www.microsoft.com/downloads/details.aspx?familyid=889482fc-5f56-4a38-b838-de776fd4138c&displaylang=en)
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).
  
Обновление для системы безопасности поддерживает следующие параметры установки.
  
| Параметр                  | Описание                                                                                                                                    |  
|---------------------------|---------------------------------------------------------------------------------------------------------------------------------------------|  
| **/? **или **/help**      | Выводит диалоговое окно использования.                                                                                                      |  
| **/passive**              | Задает пассивный режим. Вмешательство пользователя не требуется. Он видит основные диалоговые окна выполнения, но ничего не может отменить. |  
| **/quiet**                | Задает скрытый режим или подавляет сообщения об извлечении файлов.                                                                          |  
| **/norestart**            | Подавляет перезагрузку системы, если ее требует обновление.                                                                                 |  
| **/forcerestart**         | Автоматически перезагружает систему после применения обновления, даже если оно этого не требует.                                            |  
| **/extract**              | Извлечение файлов без выполнения установки. Будет предложено указать целевую папку.                                                         |  
| **/extract:&lt;path&gt;** | Заменяет команду установки, определенную автором. Задает имя и путь к установочному файлу Setup.inf или .exe.                               |  
| **/lang:&lt;LCID&gt;**    | Предлагает использовать определенный язык, поддерживаемый пакетом обновления.                                                               |  
| **/log:&lt;log file&gt;** | Включает ведение журнала (Vnox и установщиком) во время установки обновления.                                                               |
  
**Примечание.** В одной командной строке можно использовать сразу несколько параметров. Для обеспечения обратной совместимости обновлениями безопасности поддерживаются параметры, которые использовались в программе установки более ранней версии. Дополнительные сведения о поддерживаемых параметрах установки см. в [статье 262841 базы знаний Майкрософт.](http://support.microsoft.com/kb/262841)
  
**Удаление обновления**
  
Установленное обновление для системы безопасности нельзя удалить.
