Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-043 - Критическое
=============================================================

Уязвимость в драйвере дисплея Canonical Display Driver делает возможным удаленное выполнение кода (2032276)
-----------------------------------------------------------------------------------------------------------

Дата публикации: 13 июля 2010 г. | Дата обновления: 14 июля 2010 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет опубликованную уязвимость в драйвере дисплея Canonical Display Driver (cdd.dll). Хотя возможно, что эта уязвимость делает возможным выполнение кода, успешное выполнение кода маловероятно из-за рандомизации памяти. В большинстве случаев намного более вероятно, что злоумышленник, успешно воспользовавшийся этой уязвимостью, может вызвать прекращение ответов и автоматическую перезагрузку на уязвимой системе.

Это обновление для системы безопасности имеет уровень серьезности "критический" для 64-разрядных выпусков Windows 7 и уровень серьезности "существенный" для Windows Server 2008 R2. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** в этом разделе.

Данное обновление безопасности устраняет эту уязвимость путем исправления способа, используемого драйвером Canonical Display Driver для анализа обработки информации, скопированной из пользовательского режима в режим ядра. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

Кроме того, это обновление устраняет уязвимость, впервые описанную в [выпуске 2028859 советов Майкрософт по безопасности](http://technet.microsoft.com/security/advisory/2028859).

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которые хотят применить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив его наличие в [Центре обновления Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

**Известные проблемы.** Нет

Уязвимость драйвера Canonical Display Driver, связанная с переполнением целочисленного значения (CVE-2009-3678)  
---------------------------------------------------------------------------------------------------------------
  
<span></span>
Уязвимость, делающая возможным удаленное выполнение кода без проверки подлинности, связана со способом, используемым драйвером Microsoft Canonical Display Driver (cdd.dll) для анализа информации, скопированной из пользовательского режима в режим ядра. Хотя возможно, что эта уязвимость делает возможным выполнение кода, успешное выполнение кода маловероятно из-за рандомизации памяти. В большинстве случаев намного более вероятно, что злоумышленник, успешно воспользовавшийся этой уязвимостью, может вызвать прекращение ответов и автоматическую перезагрузку на уязвимой системе. Злоумышленник, успешно воспользовавшийся этой уязвимостью, может выполнить произвольный код и получить полный контроль над уязвимой системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2009-3678](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-3678).
  
#### Смягчающие факторы для уязвимости драйвера Canonical Display Driver, связанной с переполнением целочисленного значения (CVE-2009-3678)
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Эта уязвимость затрагивает только системы Windows, в которых установлена тема Windows Aero. По умолчанию интерфейс Windows Aero не включен в Windows Server 2008 R2. Кроме того, в Windows Server 2008 R2 нет графических драйверов с поддержкой Windows Aero. Чтобы включить Windows Aero, необходимо получить графические драйверы у стороннего поставщика или от производителя видеоадаптера.  
-   В случае атаки через Интернет злоумышленник может разместить в сети веб-сайт, содержащий веб-страницу, предназначенную для использования данной уязвимости. Кроме того, веб-сайты, подвергшиеся атаке злоумышленников, равно как и веб-сайты, принимающие или размещающие пользовательские материалы, также могут иметь специальное содержимое, рассчитанное на использование данной уязвимости. Однако в любом случае злоумышленник не может заставить пользователей посетить эти веб-сайты. Поэтому злоумышленнику придется убеждать пользователей посетить такой веб-сайт, обычно предлагая щелкнуть ссылку, ведущую на веб-сайт, в сообщении электронной почты или программе обмена мгновенными сообщениями.
  
#### Временные решения для уязвимости драйвера Canonical Display Driver, связанной с переполнением целочисленного значения (CVE-2009-3678)
  
К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.
  
-   **Отключение темы Windows Aero**
  
Отключение темы Windows Aero предотвращает возможность использования этой уязвимости в уязвимых системах.
  
**Использование интерактивного метода**
  
Чтобы отключить Windows Aero путем изменения темы, выполните следующие действия для каждого пользователя в системе:
  
1.  Нажмите кнопку **Пуск**, выберите пункт "Панель управления" и щелкните пункт **Оформление и персонализация**.  
2.  В категории **Персонализация** щелкните пункт **Изменение темы**.  
3.  Прокрутите список тем до конца и выберите одну из доступных тем в разделе **Базовые (упрощенные) темы и темы с высокой контрастностью**.
  
**Использование групповой политики**
  
**Примечание.** Оснастка консоли MMC групповых политик можно использовать для задания политики для компьютера, для подразделения или для всего домена. Дополнительные сведения о групповой политике можно найти на следующих веб-сайтах Майкрософт.
  
[Набор групповых политик](http://technet2.microsoft.com/windowsserver/en/library/6d7cb788-b31d-4d17-9f1e-b5ddaa6deecd1033.mspx?mfr=true)
  
[Что такое редактор объектов групповой политики?](http://technet2.microsoft.com/windowsserver/en/library/47ba1311-6cca-414f-98c9-2d7f99fca8a31033.mspx?mfr=true)
  
[Основные средства и параметры групповой политики](http://technet2.microsoft.com/windowsserver/en/library/e926577a-5619-4912-b5d9-e73d4bdc94911033.mspx?mfr=true)
  
Чтобы отключить интерфейс Windows Aero путем включения настройки по умолчанию через групповую политику, выполните следующие действия:

#### Часто задаваемые вопросы об уязвимости драйвера Canonical Display Driver, связанной с переполнением целочисленного значения (CVE-2009-3678)
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость связана с запуском программного кода в удаленном режиме. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами.
  
**В чем причина уязвимости?** Драйвер Windows Canonical Display Driver некорректно анализирует информацию, скопированную из пользовательского режима в режим ядра.
  
**Что такое Canonical Display Driver (cdd.dll)?**    
Драйвер дисплея Canonical Display Driver (cdd.dll) используется композицией рабочего стола для совместного отображения графических объектов GDI и DirectX. Драйвер CDD эмулирует интерфейс драйвера дисплея Windows XP для взаимодействия с графическим процессором Win32k GDI.
  
**Что такое GDI?**    
Интерфейс графических устройств Microsoft Windows (GDI) позволяет приложениям использовать графику и форматированный текст на мониторе и при печати. Приложения Windows не обращаются к графическим устройствам напрямую. Вместо них с драйверами устройств взаимодействует интерфейс GDI. Дополнительные сведения о GDI см. в статье MSDN [Windows GDI](http://msdn.microsoft.com/en-us/library/dd145203.aspx).
  
**Что такое Windows Aero?**    
Windows Aero это интерфейс рабочего стола, используемый на компьютерах, на которых установлен совместимый видеоадаптер и выпуск Home Premium, Professional, Ultimate или Enterprise операционной системы Windows 7. Интерфейс Windows Aero — это устанавливаемый компонент в Windows Server 2008 R2.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
В большинстве случаев злоумышленник, успешно воспользовавшийся этой уязвимостью, может заставить систему не отвечать на запросы и автоматически перезагружаться.
  
Также теоретически возможно, но маловероятно из-за рандомизации памяти, что злоумышленник, успешно воспользовавшийся этой уязвимостью, сможет выполнить произвольный код. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами.
  
**Почему для этой проблемы маловероятно выполнение кода?**    
Злоумышленник, который захочет воспользоваться этой уязвимостью для выполнения кода, должен будет записать исполняемое содержимое в определенный участок памяти ядра. Но поскольку начальный адрес будет случайным, окончательное назначение указателя будет трудно предугадать. Предугадать это становится еще сложнее вследствие реализации технологии ASLR (Address Space Layout Randomization) по умолчанию. В большинстве случаев код эксплойта скорее приведет к отказу в обслуживании, а не к выполнению кода.
  
**Что такое технология Address Space Layout Randomization (ASLR)?**    
В системах, в которых используется технология ASLR (Address Space Layout Randomization), точки входа функций, местоположение которых обычно предсказуемо, перемещаются в памяти псевдослучайным образом. Дополнительные сведения о технологии ASLR см. в журнальной статье на веб-сайте TechNet [Внутри ядра Windows Vista: часть 3](http://technet.microsoft.com/en-us/magazine/2007.04.vistakernel.aspx?pr=blog).
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Для успешного использования этой уязвимости необходимо, чтобы пользователь просмотрел особым образом созданный файл изображения в уязвимом приложении. Эта проблема затрагивает только приложения, использующие для рендеринга изображения API-интерфейсы для GDI.
  
В случае атаки через Интернет на веб-сайте злоумышленника должен находиться специально созданный файл изображения, предназначенный для использования этой уязвимости. Кроме того, веб-сайты, подвергшиеся атаке злоумышленников, равно как и веб-сайты, принимающие или размещающие пользовательские материалы, также могут иметь специальное содержимое, рассчитанное на использование данной уязвимости. Злоумышленник не может заставить пользователей посетить специальный веб-сайт. Однако он может склонить их к этому, убедив щелкнуть ссылку в сообщении электронной почты или в запросе программы обмена мгновенными сообщениями.
  
**Какие системы в первую очередь подвергаются риску?**    
Риску использования данной уязвимости в первую очередь подвержены рабочие станции. Серверы также подвержены риску в том случае, если включены функции рабочего стола Aero.
  
**Подвержены ли этой проблеме приложения сторонних поставщиков?**    
Некоторые сторонние приложения для просмотра изображений могут быть подвержены этой проблеме, если они используют API-интерфейсы для GDI для рендеринга изображений. Для обеспечения полной защиты компьютера от уязвимости, описанной в данном бюллетене, вам следует обратиться напрямую к разработчику или поставщику, отвечающему за поддержку данного стороннего приложения.
  
**Как действует обновление?**    
Данное обновление устраняет эту уязвимость путем исправления способа, используемого драйвером Canonical Display Driver для анализа обработки информации, скопированной из пользовательского режима в режим ядра.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**    
Да. О данной уязвимости сообщалось в открытых источниках. В списке "Common Vulnerabilities and Exposures" данной уязвимости присвоен номер [CVE-2009-3678](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-3678).
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.
