Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-074 - Среднее
=========================================================

Уязвимость в классах Microsoft Foundation делают возможным удаленное выполнение кода (2387149)
----------------------------------------------------------------------------------------------

Дата публикации: 12 октября 2010 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Данное обновление для системы безопасности устраняет опубликованную уязвимость библиотеки Microsoft Foundation Class (MFC). Эта уязвимость делает возможным удаленное выполнение кода, если пользователь, вошедший в систему, откроет приложение, созданное с помощью библиотеки MFC. Злоумышленник может получить те же права, что и вошедший в систему пользователь. Если пользователь вошел в систему с правами администратора, злоумышленник может получить полный контроль над ней. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

Это обновление безопасности имеет уровень серьезности "средний" для всех поддерживаемых выпусков Microsoft Windows. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Данное обновление для системы безопасности устраняет эту уязвимость, добавляя дополнительные проверки запросов на изменение названий окон. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация**. У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которые хотят применить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это, используя программное обеспечение для управления обновлениями или проверив его наличие в [Центре обновления Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

**Известные проблемы**. Нет

Уязвимость, связанная с обновлением названий документов Windows MFC, которая приводит к переполнению буфера (CVE-2010-3227)  
---------------------------------------------------------------------------------------------------------------------------
  
<span></span>
В способе, который приложения, созданные с помощью библиотеки MFC, используют для управления названиями окон, существует уязвимость, делающая возможным удаленное выполнение кода. Несмотря на то что уязвимость MFC существует в уязвимых операционных системах, злоумышленник может воспользоваться ею только тогда, когда он может изменить название любого окна в приложении MFC. Злоумышленник, успешно воспользовавшийся этой уязвимостью, может выполнить произвольный код в контексте безопасности текущего пользователя.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227).
  
#### Смягчающие факторы для уязвимости, связанной с обновлением названий документов Windows MFC, которая приводит к переполнению буфера (CVE-2010-3227)
  
Корпорация Майкрософт не обнаружила смягчающих факторов для данной уязвимости.
  
#### Временные решения для уязвимости, связанной с обновлением названий документов Windows MFC, которая приводит к переполнению буфера (CVE-2010-3227)
  
Корпорацией Майкрософт не найдено временных решений для устранения этой уязвимости.
  
#### Часто задаваемые вопросы об уязвимости, связанной с обновлением названий документов Windows MFC, которая приводит к переполнению буфера (CVE-2010-3227)
  
**Какова область воздействия этой уязвимости?**    
Эта уязвимость связана с удаленным выполнением кода в контексте безопасности текущего пользователя.
  
**В чем причина уязвимости?**    
Эта уязвимость вызвана способом обработки библиотекой MFC запросов приложения на переименование названий окон.
  
**Что такое библиотека Microsoft Foundation Class (MFC)?**    
Библиотека Microsoft Foundation Class (MFC) — это среда выполнения для программирования в Microsoft Windows. В библиотеке MFC, написанной на языке программирования C++, содержится большая часть кода, необходимого для выполнения таких стандартных операций, как управление окнами, меню и диалоговыми окнами, а также выполнения основных операций ввода/вывода или хранения наборов объектов данных.
  
**Будет ли предложено установить это обновление, если в системе отсутствуют факторы, которые делают возможными подобные атаки?**    
Да, это обновление предлагается установить на все компьютеры, на которых установлены программы, указанные в разделе "Подвержены уязвимости". Если на компьютере не установлены приложения, использующие библиотеку MFC, рекомендуется установить это обновление в качестве дополнительной меры защиты на случай, если в будущем в системе будет обнаружен фактор, делающий возможным такую атаку.
  
**Существуют ли приложения Майкрософт, делающие возможным подобную атаку?**    
Мы тщательно проверили все приложения Майкрософт, но не нашли ни одного приложения, которое может быть использовано для проведения подобной атаки.
  
**Известно ли Майкрософт о любых приложениях сторонних разработчиков, использующих библиотеку MFC, которые могут быть использованы для проведения подобных атак?**    
Да. Нам известно об одном приложении стороннего разработчика, которое может быть использовано для проведения подобной атаки. Злоумышленник может воспользоваться данной уязвимостью с помощью этого приложения стороннего разработчика только в том случае, если пользователь откроет особым образом созданное приложение MFC и дважды щелкнет в нем папку.
  
**Может ли собственное приложение пользователя, созданное с помощью библиотеки MFC, быть использовано для проведения подобных атак?**    
Собственное приложение пользователя может быть использовано для проведения атак тогда, когда соблюдены все причисленные ниже условия.
  
-   Оно использует библиотеку MFC.  
-   Собственное приложение назначает названия окон, используя данные, управляемые удаленным пользователем.
  
**Примечание.** Рекомендуется, чтобы приложения MFC динамически связывались с версиями библиотеки MFC, которые поставляются с операционной системой Windows. Если собственное приложение пользователя предоставляет уязвимую версию библиотеки MFC и устанавливает с ней статическую связь, рекомендуется обновить приложение, чтобы оно динамически связывалось с версией библиотеки MFC, поставляемой с операционной системой Windows.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник может получить те же права, что и вошедший в систему пользователь. Если пользователь вошел в систему с правами администратора, злоумышленник может получить полный контроль над ней. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Чтобы воспользоваться этой уязвимостью, на целевой системе должно быть установлено приложение MFC, которое может быть использовано для проведения атак. В уязвимом приложении MFC должен существовать способ, который удаленный пользователь может использовать для изменения названия окон в приложении MFC. Примером может служить изменение названия окна на основе содержания файла, открытого в приложении MFC, или на основе ввода данных для отправки по сети (приложение для разговора или веб-браузер). В зависимости от направления атаки злоумышленнику может потребоваться убедить пользователя открыть особым образом созданный файл с помощью определенного приложения MFC, которое установлено на компьютере пользователя, а затем выполнит ряд определенных действий. В случае с другим направлением атаки, например при использовании приложения для разговора, чтобы воспользоваться уязвимостью, злоумышленнику достаточно изменить название комнаты разговоров или использовать особым образом созданный дескриптор во время разговора с пользователем.
  
**Какие системы в первую очередь подвержены риску?**    
В первую очередь риску подвержены рабочие станции и серверы терминалов, на которых установлены приложения MFC, позволяющие удаленному пользователю изменять названия окон в приложении MFC. Серверы могут быть подвержены большему риску, если администраторы позволяют пользователям входить на серверы и запускать такие программы MFC. Однако общепринятой практикой настоятельно рекомендуется не предоставлять пользователям таких полномочий.
  
**Как действует обновление?**    
Данное обновление для системы безопасности устраняет эту уязвимость, добавляя дополнительные проверки запросов на изменение названий окон.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Да. О данной уязвимости сообщалось в открытых источниках. В списке "Common Vulnerabilities and Exposures" данной уязвимости присвоен номер [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227).
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

**Установка обновления**

При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

Обновления для системы безопасности могут не содержать всех этих вариантов. **Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.
  
Если один из файлов был обновлен ранее другим исправлением, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае будут скопированы файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).
  
Дополнительные сведения об установщике см. на [веб-сайте Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer (MBSA)**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
