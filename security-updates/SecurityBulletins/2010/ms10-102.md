Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-102 - Важное
========================================================

Уязвимость в Hyper-V делает возможной атаку типа "отказ в обслуживании" (2345316)
---------------------------------------------------------------------------------

Дата публикации: 14 декабря 2010 г. | Дата обновления: 12 января 2011 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Данное обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в Windows Server 2008 Hyper-V и Windows Server 2008 R2 Hyper-V. Эта уязвимость делает возможным отказ в обслуживании, если прошедший проверку пользователь отправит особым образом созданный пакет в VMBus в одной из гостевых виртуальных машин, размещенных на сервере Hyper-V. Чтобы воспользоваться уязвимостью, злоумышленник должен обладать действительными учетными данными и иметь возможность отправить особым образом созданное содержимое с гостевой виртуальной машины. Данной уязвимостью не могут воспользоваться удаленные или анонимные пользователи.

Уровень важности этого обновления для системы безопасности определен как существенный для всех поддерживаемых 64-разрядных выпусков Microsoft Windows Server 2008 и Windows Server 2008 R2. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** данного раздела.

Это обновление для системы безопасности устраняет уязвимость, исправляя способ проверки сервером Hyper-V неверно сформированных пакетов, посланных на VMBus внутри гостевых виртуальных машин. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

Часто задаваемые вопросы о данном обновлении безопасности
---------------------------------------------------------

<span></span>
**У меня не включена роль Hyper-V. Могу ли я установить это обновление?** 
Нет. Обновление можно установить только на подверженные уязвимости операционные системы, в которых включена роль Hyper-V. Системы, в которых роль Hyper-V не включена, не подвержены этой уязвимости.

**Будет ли данное обновление предлагаться, если Hyper-V не включен?**  
Это обновление предлагается через функцию автоматического обновления, только если роль Hyper-V включена. Это относится ко всем системам, при развертывании которых выбиралась установка ядра сервера.

**Требуется ли применять данное обновление к гостевым виртуальным машинам Hyper-V?**  
Данное обновление не должно применяться к гостевым виртуальным машинам Hyper-V, а только к серверам с включенной ролью Hyper-V. Через функцию автоматического обновления оно будет предлагаться только в системе сервера Hyper-V, а не внутри виртуальных гостевых машин.

**Что такое Hyper-V Server и подвержен ли он данной уязвимости?**  
Microsoft Hyper-V Server — автономный продукт, предоставляющий надежное оптимизированное решение виртуализации. Он позволяет организациям улучшать использование серверов и снижать расходы. Так как Hyper-V Server — специализированный автономный продукт, содержащий только Windows Hypervisor, модель драйверов Windows Server и компоненты виртуализации, он требует мало памяти и времени процессора.

Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 Service Pack 2 и Microsoft Hyper-V Server 2008 R2 подвержены данной уязвимости. Для снижения опасности данной уязвимости необходимо установить соответствующие обновления для Windows Server 2008 и Windows Server 2008 R2. Если функция автоматического обновления включена, установка этих обновлений предлагается автоматически.

**С какими известными проблемами могут столкнуться пользователи при удалении этого обновления для системы безопасности?**  
При удалении этого обновления необходимо дважды перезагрузить систему из-за наличия в нем некоторых двоичных файлов. Системы, не прошедшие две перезагрузки после удаления данного обновления, могут потерять связь с сетью. Дополнительные сведения об этой проблеме см. в [статье 2345316 базы знаний Майкрософт.](http://support.microsoft.com/kb/2345316)

Уязвимость Hyper-V VMBus - CVE-2010-3960  
----------------------------------------
  
<span></span>
Существует уязвимость в Windows Server 2008 Hyper-V и Windows Server 2008 R2 Hyper-V, которая делает возможным отказ в обслуживании, если прошедший проверку пользователь отправит особым образом созданный пакет на VMBus в одной из гостевых виртуальных машин, размещенных на сервере Hyper-V. Чтобы воспользоваться данной уязвимостью, злоумышленник должен обладать действительными учетными данными для входа в гостевую виртуальную машину и иметь возможность локального входа. Данной уязвимостью не могут воспользоваться удаленные или анонимные пользователи.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2010-3960](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3960).

#### Часто задаваемые вопросы об уязвимости Hyper-V VMBus - CVE-2010-3960
  
**Какова область воздействия этой уязвимости?**    
Эта уязвимость позволяет провести атаку типа «отказ в обслуживании». Воспользовавшись этой уязвимостью, злоумышленник может заставить сервер Hyper-V не отвечать на запросы, что потребует его перезапуска. Обратите внимание, что уязвимость типа "отказ в обслуживании" не позволяет злоумышленнику выполнить код или повысить уровень своих прав, но может привести к тому, что система перестанет принимать запросы.
  
**В чем причина уязвимости?**    
Эта уязвимость вызвана тем, что сервер Hyper-V некорректно проверяет особым образом созданные пакеты, посланные на Hyper-V VMBus. Из-за недостаточной проверки при обработке машинных команд приложение Hyper-V Server может перестать отвечать на запросы.
  
**Что такое Hyper-V?**    
Hyper-V — технология на основе гипервизора. Это ключевой компонент Windows Server 2008 и Windows Server 2008 R2. Она обеспечивает масштабируемую, надежную, высоко доступную платформу виртуализации. Польза среды виртуализации заключается в том, что пользователи могут создавать и контролировать на одном физическом компьютере виртуальные машины с разными изолированными операционными системами.
  
**Что такое VMBus?**    
VMBus — это логический канал связи между разделами. Дополнительные сведения об архитектуре VMBus и Hyper-V см. в статье MSDN [Архитектура Hyper-V](http://msdn.microsoft.com/en-us/library/cc768520(bts.10).aspx).
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Действия злоумышленника, воспользовавшегося этой уязвимостью, могут привести к тому, что система пользователя перестанет отвечать на запросы до ее перезапуска. Обратите внимание, что из-за данной уязвимости может перестать отвечать на запросы сервер Hyper-V, включая все размещенные на нем виртуальные машины.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник должен быть прошедшим проверку пользователем в одной из размещенных на сервере Hyper-V гостевых виртуальных машин и иметь возможность выполнения произвольного кода в данной системе. Затем злоумышленнику необходимо создать специальный драйвер, который будет генерировать особым образом созданные пакеты и посылать их на VMBus, таким образом вызывая отказ обслуживания на сервере Hyper-V.
  
**Какие системы в первую очередь подвержены риску?**    
Наибольшую угрозу эта уязвимость представляет для серверов Hyper-V с размещенными гостевыми виртуальными машинами, разрешающими доступ ненадежных пользователей.
  
**Как действует обновление?**    
Данное обновление устраняет эту уязвимость, исправляя способ проверки и обработки сервером Hyper-V особым образом созданных пакетов, приходящих с виртуальных гостевых машин на Hyper-V VMBus.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

#### Сведения о развертывании
  
**Установка обновления**
  
При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).
  
Обновление для системы безопасности поддерживает следующие параметры установки.
  
| Параметр                                       | Описание                                                                                                                         |  
|------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help**                              | Вывод справки о поддерживаемых параметрах.                                                                                       |  
| **/quiet**                                     | Подавление отображения сообщений о состоянии или ошибке.                                                                         |  
| **/norestart**                                 | При вводе с параметром **/quiet** система не перезагрузится после установки, даже если для ее завершения требуется перезагрузка. |  
| **/warnrestart:&lt;секунды&gt;**               | При использовании вместе с параметром **/quiet** установщик предупреждает пользователя перед запуском перезагрузки.              |  
| **/promptrestart**                             | При использовании вместе с параметром **/quiet** установщик выдает запрос перед запуском перезагрузки.                           |  
| **/forcerestart**                              | При использовании вместе с параметром **/quiet** установщик принудительно закрывает приложения и запускает перезагрузку.         |  
| **/log:&lt;имя файла&gt;**                     | Включается ведение журнала в указанном файле.                                                                                    |  
| **/extract:&lt;назначение&gt;**                | Содержимое пакетов извлекается в указанную папку назначения.                                                                     |  
| **/uninstall /kb:&lt;номер в базе знаний&gt;** | Удаление обновления для системы безопасности.                                                                                    |
  
**Примечание.** Дополнительные сведения об установщике wusa.exe см. в разделе "Автономный установщик обновлений Windows" статьи TechNet [Другие изменения в Windows 7](http://technet.microsoft.com/en-us/library/dd871148(ws.10).aspx).
  
**Проверка успешного применения обновления**
  
-   **Анализатор безопасности Microsoft Baseline Security Analyzer**
  
    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.
  
-   **Проверка версий файлов**
  
    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.
  
    1.  Нажмите кнопку **Пуск** и затем введите имя файла обновления в поле **Начать поиск**.  
    2.  Когда файл появится в меню **Программы**, щелкните имя файла правой кнопкой мыши и выберите пункт **Свойства**.  
    3.  Сравните размер файла на вкладке **Общие** с данными таблиц сведений о файлах, приведенных в разделе статьи базы знаний этого бюллетеня.  
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.  
    4.  Также можно открыть вкладку **Дополнительно** и сравнить версии файлов и дату их изменения с данными, приведенными в таблицах сведений о файлах раздела статьи базы знаний этого бюллетеня.  
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.  
    5.  Наконец, также можно открыть вкладку **Предыдущие версии** и сравнить предыдущие версии файлов с данными о новых или обновленных версиях.
