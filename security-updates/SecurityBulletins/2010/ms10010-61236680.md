Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-010 - Важное
========================================================

Уязвимость в Windows Server 2008 Hyper-V делает возможной атаку типа "отказ в обслуживании" (977894)
----------------------------------------------------------------------------------------------------

Дата публикации: 9 февраля 2010 г. | Дата обновления: 9 февраля 2010 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Данное обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в Windows Server 2008 Hyper-V и Windows Server 2008 R2 Hyper-V. Эта уязвимость делает возможным отказ в обслуживании, если прошедший проверку пользователь выполнит неверно сформированную последовательность машинных команд в одной из гостевых виртуальных машин, размещенных на сервере Hyper-V. Чтобы воспользоваться данной уязвимостью, злоумышленник должен обладать действительными учетными данными для входа в гостевую виртуальную машину и иметь возможность локального входа. Данной уязвимостью не могут воспользоваться удаленные или анонимные пользователи.

Уровень важности этого обновления для системы безопасности определен как существенный для всех поддерживаемых 64-разрядных выпусков Microsoft Windows Server 2008 и Windows Server 2008 R2. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** данного раздела.

Это обновление для системы безопасности устраняет уязвимость, изменяя способ проверки сервером Hyper-V кодирования машинных команд, выполняемых внутри своих гостевых виртуальных машин. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

Вопросы и ответы о данном обновлении безопасности
-------------------------------------------------

<span></span>
**Где находятся дополнительные сведения о файлах?**  
Местонахождение дополнительных сведений о файлах см. в справочных таблицах раздела **Развертывание обновления для системы безопасности**.

**Будет ли данное обновление предлагаться, если роль Hyper-V не включена?**
**На уязвимых платформах данное обновление можно установить вручную,** даже если роль Hyper-V не включена. Однако оно предлагается через функцию автоматического обновления, только если роль Hyper-V включена.

**Требуется ли применять данное обновление к гостевым виртуальным машинам Hyper-V?**
Данное обновление не должно применяться к гостевым виртуальным машинам Hyper-V, только к компьютеру с включенной ролью Hyper-V. Через функцию автоматического обновления оно будет предлагаться только в системе сервера Hyper-V, а не внутри виртуализованных гостевых машин.

**Что такое Hyper-V Server и подвержен ли он данной уязвимости?**
Microsoft Hyper-V Server — автономный продукт, предоставляющий надежное оптимизированное решение виртуализации. Он позволяет организациям улучшать использование серверов и снижать расходы. Так как Hyper-V Server — специализированный автономный продукт, содержащий только Windows Hypervisor, модель драйверов Windows Server и компоненты виртуализации, он требует мало памяти и времени процессора.
Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 Service Pack 2 и Microsoft Hyper-V Server 2008 R2 подвержены данной уязвимости. Для ее устранения необходимо применить обновления для Windows Server 2008 и Windows Server 2008 R2 соответственно. Если функция автоматического обновления включена, установка этих обновлений предлагается автоматически.

Уязвимость, связанная с выборочным подтверждением приема Hyper-V, — CVE-2010-0026  
---------------------------------------------------------------------------------
  
<span></span>
В сервере Hyper-V для Windows Server 2008 и Windows Server 2008 R2 существует уязвимость, связанная с отказом в обслуживании. Она вызвана недостаточной проверкой сервером Hyper-V специальных последовательностей машинных команд. Воспользовавшись этой уязвимостью, злоумышленник может заставить сервер Hyper-V не отвечать на запросы. Это может затронуть все виртуальные машины, размещенные в такой системе.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2010-0026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0026).
  
#### Факторы, снижающие опасность уязвимости, связанной с выборочным подтверждением приема Hyper-V, — CVE-2010-0026
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Чтобы воспользоваться данной уязвимостью, злоумышленник должен обладать действительными учетными данными для входа в виртуальную машину Hyper-V и иметь возможность локального входа. Данной уязвимостью не могут воспользоваться удаленные или анонимные пользователи.
  
#### Методы обхода уязвимости, связанной с выборочным подтверждением приема Hyper-V, — CVE-2010-0026
  
Корпорацией Майкрософт не найдены методы обхода этой уязвимости.
  
#### Вопросы и ответы об уязвимости, связанной с выборочным подтверждением приема Hyper-V, — CVE-2010-0026
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость позволяет провести атаку типа «отказ в обслуживании». Воспользовавшись этой уязвимостью, злоумышленник может заставить сервер Hyper-V не отвечать на запросы, что потребует его перезапуска. Обратите внимание, что уязвимость типа "отказ в обслуживании" не позволяет злоумышленнику выполнить код или повысить уровень своих прав, но может привести к тому, что система перестанет принимать запросы.
  
**В чем причина уязвимости?** Данная уязвимость вызвана неправильной проверкой сервером Hyper-V кодирования специальных машинных команд, выполняемых внутри гостевых виртуальных машин. Из-за недостаточной проверки при обработке машинных команд приложение Hyper-V Server может перестать отвечать на запросы.
  
**Что такое Hyper-V?**    
Hyper-V — технология на основе гипервизора. Это ключевой компонент Windows Server 2008 и Windows Server 2008 R2. Она обеспечивает масштабируемую, надежную, высоко доступную платформу виртуализации. Польза среды виртуализации заключается в том, что пользователи могут создавать и контролировать на одном физическом компьютере виртуальные машины с разными изолированными операционными системами.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Действия злоумышленника, воспользовавшегося этой уязвимостью, могут привести к тому, что система пользователя перестанет отвечать на запросы до ее перезапуска. Обратите внимание, что из-за данной уязвимости может перестать отвечать на запросы сервер Hyper-V, включая все размещенные на нем виртуальные машины.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник должен быть прошедшим проверку пользователем в одной из размещенных на сервере Hyper-V гостевых виртуальных машин и иметь возможность выполнения произвольного кода в данной системе. В этом случае злоумышленник может выполнить в системе ненадежный исполняемый файл, вызывающий неверно сформированную последовательность машинных команд, из-за которой сервер Hyper-V перестает отвечать на запросы.
  
**Какие системы в первую очередь подвергаются риску?**    
Наибольшую угрозу эта уязвимость представляет для серверов Hyper-V с размещенными гостевыми виртуальными машинами, разрешающими доступ ненадежных пользователей.
  
**Как действует обновление?**    
Это обновление устраняет уязвимость, изменяя способ проверки сервером Hyper-V кодирования машинных команд, выполняемых внутри своих гостевых виртуальных машин.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.
