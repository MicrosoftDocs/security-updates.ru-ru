Security Bulletin

Бюллетень по безопасности (Майкрософт) MS13-092 - Важное
========================================================

Уязвимость в Microsoft Agent делает возможным несанкционированное получение прав (2893986)
------------------------------------------------------------------------------------------

Дата публикации: 12 ноября 2013 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в Microsoft Windows. Данная уязвимость делает возможным несанкционированное получение прав, если злоумышленник передаст в гипервызове гипервизору специально созданный параметр функции из запущенной виртуальной машины. Эта уязвимость также делает возможным отказ в обслуживании хоста Hyper-V, если злоумышленник передаст в гипервызове гипервизору специально созданный параметр функции из запущенной виртуальной машины.

Это обновление безопасности имеет уровень серьезности "существенный" для Windows 8 для 64-разрядных систем и Windows Server 2012. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Данное обновление для системы безопасности устраняет эту уязвимость, обеспечивая правильную очистку пользовательского ввода в Hyper-V. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

Администраторам корпоративных установок и конечным пользователям, которые планируют установить это обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это при первой возможности, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

Обновленные часто задаваемые вопросы
------------------------------------

<span></span>
**У меня не включена система Hyper-V. Могу ли я установить это обновление?** **
**Нет. Обновление можно установить только на подверженные уязвимости операционные системы, в которых включена система Hyper-V. Системы, в которых компонент Hyper-V не включен, не подвержены этой уязвимости.

**Будет ли данное обновление предлагаться, если система Hyper-V не включена?** **
**Это обновление предлагается через функцию автоматического обновления, только если система Hyper-V включена. Это относится ко всем системам, при развертывании которых выбиралась установка ядра сервера.

**Требуется ли применять данное обновление к гостевым виртуальным машинам Hyper-V?** **
**Нет. Данное обновление не должно применяться к гостевым виртуальным машинам Hyper-V, а только к серверам с включенной системой Hyper-V. Обновление будет предлагаться посредством функции автоматического обновления только в системах сервера Hyper-V, а не внутри виртуальных гостевых машин.

**Что такое Hyper-V?**
Hyper-V — технология на основе гипервизора. Это ключевой компонент Windows Server 2008, Windows Server 2008 R2, Windows 8, Windows Server 2012, Windows 8.1 и Windows Server 2012 R2. Она предоставляет масштабируемую, надежную, высоко доступную платформу виртуализации. Среда такого типа полезна, поскольку в ней можно создавать виртуальные машины и управлять ими, что позволяет запускать несколько операционных систем на одном физическом компьютере и изолировать операционные системы друг от друга.

**Что такое Hyper-V Server и подвержен ли он также данной уязвимости?** **
**Microsoft Hyper-V Server — автономный продукт, предоставляющий надежное оптимизированное решение виртуализации. Он позволяет организациям улучшать использование серверов и снижать расходы. Так как Hyper-V Server — специализированный автономный продукт, содержащий только Windows Hypervisor, модель драйверов Windows Server и компоненты виртуализации, он требует мало памяти и времени процессора.

Сервер Microsoft Hyper-V Server 2012 подвержен данной уязвимости. Данное обновление необходимо применить к Hyper-V Server, чтобы защититься от уязвимости, описанной в этом бюллетене. Если функция автоматического обновления включена, установка этого обновления предлагается автоматически.

Уязвимость, связанная с повреждением адресов (CVE-2013-3898)
------------------------------------------------------------

<span></span>
В Hyper-V для Windows 8 и Windows Server 2012 существует уязвимость, приводящая к несанкционированному получению прав. Воспользовавшийся ею злоумышленник может выполнить произвольный код в контексте учетной записи System в другой виртуальной машине общего хоста Hyper-V. Злоумышленник не сможет выполнить код на хосте Hyper-V, только в гостевых виртуальных машинах на одном хосте. Эта уязвимость также делает возможным отказ в обслуживании хоста Hyper-V на одинаковых платформах, позволяя злоумышленникам добиться того, что хост Hyper-V перестанет отвечать на запросы или перезапустится.

В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2013-3898](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2013-3898).

#### Смягчающие факторы

Корпорация Майкрософт не обнаружила смягчающих факторов для данной уязвимости.

#### Временные решения

Корпорацией Майкрософт не найдено временных решений для устранения этой уязвимости.

#### Часто задаваемые вопросы

**Какова область воздействия этой уязвимости?**
Это уязвимость несанкционированного получения прав, которая также делает возможной атаку типа "отказ в обслуживании".

**В чем причина уязвимости?**
Данная уязвимость возникает при неправильной проверке значения структуры данных, позволяя использовать недопустимый адрес памяти.

**Может ли** **злоумышленник получить контроль над хостом Hyper-V?**
Нет. Хост Hyper-V не подвержен уязвимости несанкционированного повышения привилегий.

**Уязвимы только гостевые виртуальные машины?**
Нет. Хотя только гостевые виртуальные машины уязвимы к несанкционированному повышению привилегий, хост Hyper-V уязвим для атак типа "отказ в обслуживании" из любой запущенной гостевой виртуальной машины. Такой отказ в обслуживании также может повлиять на все остальные виртуальные машины данного хоста, делая их недоступными, пока сервер не будет перезапущен.

**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**
Злоумышленник, успешно воспользовавшийся данной уязвимостью, может вызвать сбой хоста Hyper-V, а затем и сбой гостевых виртуальных машин. Потенциально злоумышленник также может выполнить код в другой гостевой виртуальной машине. Чтобы это было возможно, целевая виртуальная машина должна размещаться на том же хосте, что и виртуальная машина из которой действует злоумышленник.

**Каким образом злоумышленник может воспользоваться этой** **уязвимостью?**
Прошедший проверку подлинности злоумышленник с правами администратора в исходной виртуальной машине может воспользоваться этой уязвимостью, передав в гипервызове гипервизору хоста специально созданный параметр функции.

**Какие системы в первую очередь подвержены риску?**
Риску использования данной уязвимости в первую очередь подвергаются уязвимые системы с включенным компонентом Hyper-V.

**Как действует обновление?**
Данное обновление устраняет эту уязвимость, обеспечивая правильную очистку пользовательского ввода в Hyper-V.

**Была ли эта уязвимость опубликована до выпуска настоящего бюллетеня по безопасности?**
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.

**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что эта уязвимость была использована злоумышленниками?**
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

Доступно несколько ресурсов, чтобы помочь администраторам развернуть обновления для системы безопасности.

-   Анализатор безопасности Microsoft Baseline Security Analyzer (MBSA) позволяет администраторам проверять локальные и удаленные системы с целью выявления неустановленных обновлений для системы безопасности и типичных ошибок в конфигурации системы безопасности.
-   Службы Windows Server Update Services (WSUS), Systems Management Server (SMS) и System Center Configuration Manager помогают администраторам распространять обновления для системы безопасности.
-   Компоненты средства оценки совместимости с обновлениями, включенные в набор средств для обеспечения совместимости приложений, облегчают тестирование и проверку совместимости обновлений Windows с установленными приложениями.

Информацию об этих и других доступных инструментах см. в статье [Инструменты обеспечения безопасности](http://technet.microsoft.com/security/cc297183).

Чтобы повысить уровень защиты пользователей, корпорация Майкрософт предоставляет сведения об уязвимостях крупным поставщикам программного обеспечения безопасности перед ежемесячным выпуском обновлений. Эта информация необходима им для усовершенствования программного обеспечения и оборудования для защиты пользователей (антивирусных программ, сетевых систем обнаружения вторжений, а также индивидуальных систем предотвращения вторжений). Сведения о средствах защиты, предоставляемых поставщиками программного обеспечения безопасности, доступны на соответствующих веб-сайтах партнеров, перечисленных в списке партнеров [MAPP](http://go.microsoft.com/fwlink/?linkid=215201).
