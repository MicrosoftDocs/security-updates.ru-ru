Security Bulletin

Бюллетень по безопасности (Майкрософт) MS11-064 - Важное
========================================================

Уязвимости в стеке TCP/IP могут привести к отказу в обслуживании (2563894)
--------------------------------------------------------------------------

Дата публикации: 9 августа 2011 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет две обнаруженных пользователями уязвимости в Microsoft Windows. Эти уязвимости могут привести к отказу в обслуживании, если злоумышленник посылает целевой системе последовательность специально созданных сообщений ICMP (Internet Control Message Protocol) или отправляет специально созданный запрос URL на сервер, который обслуживает веб-контент и у которого включена функция качества обслуживания (QoS) на основе URL-адресов.

Это обновление для системы безопасности является существенным для всех поддерживаемых выпусков Windows Server 2008 и Windows Server 2008 R2. Уровень его важности для всех поддерживаемых выпусков Windows Vista и Windows 7 определен как средний. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** данного раздела.

Это обновление устраняет данные уязвимости путем изменения способа, используемого стеком Windows TCP/IP для обработки сообщений ICMP и работы с URL в памяти при включенной функции Качество обслуживания (QoS) на основе URL-адресов. Дополнительные сведения об этих уязвимостях см. в подразделе ответов на часто задаваемые вопросы для каждой уязвимости, упомянутой в разделе **Сведения об уязвимостях**.

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которым необходимо установить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

**Известные проблемы.** Нет

Уязвимость в протоколе ICMP, приводящая к отказу от обслуживания - CVE-2011-1871  
--------------------------------------------------------------------------------
  
<span></span>
В стеке Windows TCP/IP существует уязвимость, приводящая к отказу от обслуживания; эта уязвимость вызвана неправильной обработкой стеком TCP/IP последовательности специально созданных сообщений ICMP. Воспользовавшись ею, злоумышленник может заставить целевую систему не отвечать на запросы и автоматически перезагружаться.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2011-1871](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1871).
  
#### Смягчающие факторы для уязвимости в протоколе ICMP, приводящей к отказу в обслуживании - CVE-2011-1871
  
Корпорация Майкрософт не обнаружила смягчающих факторов для данной уязвимости.
  
#### Временные решения для уязвимости в протоколе ICMP, приводящей к отказу в обслуживании - CVE-2011-1871
  
К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.
  
-   **Блокировка ICMP на брандмауэре**
  
    Блокировка всех сообщений ICMP на пограничном брандмауэре сети или на роутере позволит защитить системы за брандмауэром или роутером от попыток использования этой уязвимости. Мы рекомендуем блокировать весь непредусмотренный входящий трафик из Интернета.
  
    **Побочные эффекты использования временного решения.** Это временное решение может негативно сказаться на производительности, так как не позволит TCP оптимизировать сетевое взаимодействие. С помощью сетевых пакетов ICMP можно устранить фрагментацию на роутерах, соединяющих сети с разными MTU. Фрагментация снижает пропускную способность TCP и увеличивает перегрузку сети.
  
#### Часто задаваемые вопросы об уязвимости в протоколе ICMP, приводящей к отказу в обслуживании - CVE-2011-1871

Уязвимость в протоколе TCP/IP QOS, приводящая к отказу от обслуживания - CVE-2011-1965  
--------------------------------------------------------------------------------------
  
<span></span>
В стеке Windows TCP/IP существует уязвимость, приводящая к отказу от обслуживания; эта уязвимость вызвана неправильной обработкой стеком TCP/IP находящихся в памяти URL при включенной функции Качество обслуживания (QoS) на основе URL-адресов. Воспользовавшись ею, злоумышленник может заставить целевую систему не отвечать на запросы и автоматически перезагружаться.
  
В перечне "Common Vulnerabilities and Exposures" этой уязвимости присвоен номер [CVE-2010-1965](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1965).
  
#### Смягчающие факторы для уязвимости в протоколе TCP/IP QOS, приводящей к отказу от обслуживания - CVE-2011-1965
  
К смягчающим факторам относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие смягчающие факторы могут снизить опасность использования уязвимости.
  
-   По умолчанию функция Качество обслуживания (QoS) на основе URL-адресов выключена во всех операционных системах Windows. Чтобы система стала подвержена данной уязвимости, пользователи должны сами установить данную функцию.
  
#### Временные решения для уязвимости в протоколе TCP/IP QOS, приводящей к отказу от обслуживания - CVE-2011-1965
  
К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.
  
-   Отключение функции Qos на основе политик. Дополнительные сведения см. в статье Microsoft TechNet [Качество обслуживания (QoS) на основе политик](http://technet.microsoft.com/en-us/library/dd919203(ws.10).aspx).
  
#### Часто задаваемые вопросы об уязвимости в протоколе TCP/IP QOS, приводящей к отказу от обслуживания - CVE-2011-1965
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость делает возможной атаку типа "отказ в обслуживании".
  
**В чем причина уязвимости?** Уязвимость вызвана тем, что стек Windows TCP/IP неправильно обрабатывает последовательность особым образом созданных URL, находящихся в памяти. Данной уязвимости подвержены системы, на которых размещен веб-контент на основе сервисов типа IIS, и где включена функция QoS на основе URL-адресов.
  
**Какой компонент подвержен этой уязвимости?**    
Этой уязвимости подвержен компонент Tcpip.sys.
  
**Что такое Качество обслуживания (QoS) на основе URL-адресов?**    
Качество обслуживания (QoS) для сетей - это широко распространенный набор стандартов и механизмов обеспечения высокой производительности важных приложений. Администраторы сетей могут использовать механизмы QoS для настройки приоритетов и управления скоростью отправки исходящего сетевого трафика. Использование этих механизмов позволяет обеспечить эффективное использование ресурсов и требуемый уровень обслуживания.
  
В Windows Server 2008 R2 ИТ-специалисты могут приоритизировать веб-трафик на основе URL-адресов. С помощью QoS на основе URL-адресов ИТ-специалисты могут гарантировать, что обработка важного веб-трафика будет производиться до обработки менее важного трафика, повышая таким образом производительность в загруженных сетях. Например, ИТ-специалист может назначить веб-трафику важного внутреннего веб-сайта более высокий приоритет, чем трафику внешних сайтов, максимизируя производительность в моменты высокой загруженности сети. Дополнительные сведения см. в статье Microsoft TechNet [Качество обслуживания (QoS) на основе URL-адресов](http://technet.microsoft.com/en-us/library/dd637810(ws.10).aspx).
  
**Что такое URI?**    
Универсальный код ресурса (URI) — это строка символов, используемая для распознавания ресурсов в Интернете или в сети и управления ими. URL-адрес является типичным примером URI, ссылающимся на такой ресурс, как веб-узел. Дополнительные сведения об URI см. в документе [RFC 2396](http://www.ietf.org/rfc/rfc2396.txt) (на английском языке).
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Воспользовавшись ею, злоумышленник может заставить целевую систему не отвечать на запросы и автоматически перезагружаться.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
В случае удаленной атаки злоумышленник, не прошедший проверку подлинности, может использовать данную уязвимость, отправляя особым образом созданный URL-запрос на сервер, где размещен веб-контент и включена функция QoS на основе URL-адресов.
  
**Какие системы в первую очередь подвергаются риску?**    
В первую очередь риску подвержены серверы, на которых размещен веб-контент и включена функция QoS на основе URL-адресов. Серверы, на которых не включена функция QoS на основе URL-адресов или не размещен веб-контент, не подвержены данной уязвимости. Рабочие станции с установленной операционной системой Windows 7, на которых можно запускать службу Internet Information Services (IIS), подвержены повышенному риску, если на них включена и служба IIS, и функция QoS на основе URL-адресов.
  
**Подвержены ли повышенному риску системы под управлением Windows 7, на которых настроен веб-сервер с IIS?**    
Да. На станциях с ОС Windows 7 можно устанавливать и настраивать серверное ПО, например, IIS. Хотя предполагается, что системы с ОС Windows 7 - это рабочие станции, в таких случаях они выступают в качестве серверов и, следовательно, в большей степени подвержены данной уязвимости.
  
**Как действует обновление?**    
Это обновление устраняет данную уязвимость путем изменения способа, используемого стеком Windows TCP/IP для обработки URL, находящихся в памяти, при включенной функции Качество обслуживания (QoS) на основе URL-адресов.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

Облегчить тестирование и проверку обновлений Windows для установленных приложений можно с помощью [компонентов оценки совместимости обновлений](http://technet2.microsoft.com/windowsvista/en/library/4279e239-37a4-44aa-aec5-4e70fe39f9de1033.mspx?mfr=true), входящих в состав [набора средств для обеспечения совместимости приложений](http://www.microsoft.com/downloads/details.aspx?familyid=24da89e9-b581-47b0-b45e-492dd6da2971&displaylang=en).
  
Набор средств для обеспечения совместимости приложений содержит необходимые средства и документацию для выявления проблем совместимости приложений и уменьшения степени воздействия этих проблем перед развертыванием системы Microsoft Windows Vista, обновлений из Центра обновления Windows, обновлений с веб-сайта безопасности Майкрософт или новой версии проводника.

**Установка обновления**
  
При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).
  
Обновление для системы безопасности поддерживает следующие параметры установки.
  
| Параметр          | Описание                                                                                                                         |  
|-------------------|----------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Вывод справки о поддерживаемых параметрах.                                                                                       |  
| **/quiet**        | Подавление отображения сообщений о состоянии или ошибке.                                                                         |  
| **/norestart**    | При вводе с параметром **/quiet** система не перезагрузится после установки, даже если для ее завершения требуется перезагрузка. |
  
**Примечание**. Дополнительные сведения об установщике wusa.exe см. в [статье 934307 базы знаний Майкрософт](http://support.microsoft.com/kb/934307).
  
**Проверка успешного применения обновления**
  
-   **Анализатор безопасности Microsoft Baseline Security Analyzer**
  
    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.
  
-   **Проверка версий файлов**
  
    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.
  
    1.  Нажмите кнопку **Пуск** и затем введите имя файла обновления в поле **Начать поиск**.  
    2.  Когда файл появится в меню **Программы**, щелкните имя файла правой кнопкой мыши и выберите пункт **Свойства**.  
    3.  Сравните размер файла на вкладке **Общие** с данными таблиц сведений о файлах, приведенных в разделе статьи базы знаний этого бюллетеня.  
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.  
    4.  Также можно открыть вкладку **Дополнительно** и сравнить версии файлов и дату их изменения с данными, приведенными в таблицах сведений о файлах раздела статьи базы знаний этого бюллетеня.  
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, в отдельных случаях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.  
    5.  Наконец, также можно открыть вкладку **Предыдущие версии** и сравнить предыдущие версии файлов с данными о новых или обновленных версиях.
