Security Bulletin

Бюллетень по безопасности (Майкрософт) MS11-086 - Важное
========================================================

Уязвимость в Active Directory делает возможным несанкционированное получение прав (2630837)
-------------------------------------------------------------------------------------------

Дата публикации: 8 ноября 2011 г.

**Версия:** 1.0

### Общие сведения

#### **Аннотация**

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в Active Directory, службе ADAM и службе Active Directory облегченного доступа к каталогам (AD LDS). Эта уязвимость делает возможным несанкционированное получение прав, если служба Active Directory настроена на использование LDAP через SSL (LDAPS), злоумышленник получает отозванный сертификат, связанный с действительной учетной записью домена, а затем используется его для прохождения проверки подлинности в домене Active Directory. По умолчанию служба Active Directory не настроена на использование LDAP через SSL.

Уровень важности этого обновления определен как "существенный" для служб Active Directory, ADAM и AD LDS, установленных в поддерживаемых выпусках Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008 (кроме Itanium), Windows 7 и Windows Server 2008 R2 (кроме Itanium). Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Данное обновление для системы безопасности устраняет эту уязвимость, изменяя способ, используемый службой Active Directory для проверки сертификатов по списку отзыва сертификатов. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

Администраторам корпоративных установок и конечным пользователям, которые планируют установить это обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это при первой возможности, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

**Известные проблемы.** Нет

Уязвимость, связанная с обходом проверки подлинности LDAPS (CVE-2011-2014)
--------------------------------------------------------------------------

<span></span>
В службе Active Directory существует уязвимость, делающая возможным несанкционированное получение прав, если данная служба настроена на использование LDAP через SSL (LDAPS). Злоумышленник может воспользоваться этой уязвимостью, используя ранее отозванный сертификат для прохождения проверки подлинности в домене Active Directory и получения доступа к сетевым ресурсам или выполнения кода с правами конкретного авторизованного пользователя, с которым связан данный сертификат.

Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2011-2014](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-2014).

#### Смягчающие факторы для уязвимости, связанной с обходом проверки подлинности LDAPS (CVE-2011-2014)

К смягчающим факторам относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие смягчающие факторы могут снизить опасность использования уязвимости.

-   Стандартные параметры конфигурации брандмауэра позволяют защитить сеть от атак из-за пределов корпоративной среды. Согласно лучшим методикам, на подключенных к Интернету системах рекомендуется держать открытыми лишь минимально необходимое количество портов.

#### Временные решения для уязвимости, связанной с обходом проверки подлинности LDAPS (CVE-2011-2014)

К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.

-   **Удалите** **учетную запись** **пользователя** **из домена, в котором бел отозван** **сертификат.**
-   **В дополнение к LDAPS используйте подключения на базе IPsec в режиме "Всегда проверять список отзыва сертификатов" .**

    Инструкции по включению проверки списка отзыва сертификатов с IPsec см. в статье TechNet [Проверка списка отзыва сертификатов — выбор приложения](http://social.technet.microsoft.com/wiki/contents/articles/964.aspx).

    **Примечания**

    1.  Используйте IPsec для шифрования и защиты подключения к LDAPS с использованием проверки подлинности сертификатов с проверкой списка отзыва сертификатов.
    2.  Используйте IPsec для защиты подключения к LDAPS с использованием проверки подлинности сертификатов с проверкой списка отзыва сертификатов и без шифрования. В этом случае LDAPS выполняет шифрование.

        Во всех уязвимых выпусках Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2 можно использовать AuthIP (расширение IPsec, позволяющее выполнять проверку подлинности пользователей), чтобы злоумышленник не мог выполнить проверку подлинности сертификата IPsec при помощи сертификата компьютера и затем использовать отозванный сертификат пользователя для выполнения проверки подлинности LDAPS.

    3.  Используйте нулевую инкапсуляцию для LDAPS для предотвращения двойного шифрования и помните, что проверка сертификатов на уровне IPsec независима от проверки подлинности, выполняемой в SSL. Это значит, что использование отдельных сертификатов на каждом уровне все же может разрешить доступ к LDAPS через просроченный сертификат.

#### Часто задаваемые вопросы об уязвимости, связанной с обходом проверки подлинности LDAPS (CVE-2011-2014)

**Какова область воздействия этой уязвимости?**
Эта уязвимость может приводить к повышению уровня прав. Воспользовавшийся ею злоумышленник может получить доступ к сетевым ресурсам или выполнить код с правами конкретного авторизованного пользователя.

**В чем причина уязвимости?**
Данная уязвимость возникает, когда службе Active Directory, настроенной на использование LDAPS, не удается проверить по списку отзыва сертификатов состояние отзыва сертификата SSL, связанного с данной учетной записью домена. В результате такой сертификат может быть принят как действительный, даже после его отзыва центром сертификации.

**Что такое центр сертификации (ЦС)?** **
**Центры сертификации — это организации, выпускающие сертификаты. Они устанавливают и проверяют достоверность открытых ключей, принадлежащих пользователям или другим центрам сертификации, а также проверяют удостоверение пользователей или организаций, запросивших сертификат.

**Что** **такое** **службы** **Active** **Directory?**
Службы Active Directory — это расширяемый и масштабируемый набор служб для эффективного управления корпоративными удостоверениями, учетными данными, защитой информации и параметрами системы и приложений. Службы Active Directory включают службы сертификатов Active Directory (AD CS), доменные службы Active Directory (AD DS), Services службы федерации Active Directory (AD FS), службы Active Directory облегченного доступа к каталогам (AD LDS) и службы управления правами Active Directory (AD RMS).

**Что такое служба Active Directory облегченного доступа к каталогам** **(AD LDS)?
**Служба Active Directory облегченного доступа к каталогам (AD LDS) представляет собой независимый режим службы Active Directory, который предоставляет специализированные службы каталогов для приложений. Служба AD LDS доступна операционных систем Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2. Служба AD LDS заменяет режим Active Directory Application Mode (ADAM), доступный для Windows XP и Windows Server 2003. Дополнительные сведения о службе AD LDS см. в статье TechNet [Служба Active Directory облегченного доступа к каталогам](http://technet.microsoft.com/en-us/library/cc731868(ws.10).aspx) (на английском языке).

**Что такое режим Active Directory Application Mode?** **
**Режим Active Directory Application Mode (ADAM) — служба каталогов LDAP, которая запускается в системах Windows XP и Windows Server 2003 в качестве пользовательской, а не системной службы. Дополнительные сведения о режиме ADAM см. в статье TechNet [Основные сведения об ADAM](http://technet.microsoft.com/ru-ru/library/cc780837(ws.10).aspx).

**Что такое протокол LDAP?
**Протокол LDAP — это протокол службы каталогов, который работает поверх стека протоколов TCP/IP. По своей модели информации (как данных, так и пространства имен) протокол LDAP аналогичен службе каталогов X.500 OSI, однако отличается от нее меньшим количеством функций и более низкими требованиями к ресурсам. В отличие от большинства других интернет-протоколов, в LDAP имеется дополнительный API-интерфейс, который упрощает написание интернет-приложений для служб каталогов. API-интерфейс протокола LDAP применяется в управлении каталогами, а также в браузерных приложениях, которые по основной своей функциональности не поддерживают службы каталогов. Протокол LDAP не позволяет создавать каталоги или указывать способы работы службы каталогов.

**Что такое список отзыва сертификатов?** **
**Список отзыва сертификатов — это список с цифровой подписью, выпущенный центром сертификации, в котором перечислены сертификаты, выпущенные и позднее отозванные данным ЦС. Для каждого отозванного сертификата в списке указывается его серийный номер, дата отзыва сертификата и причина отзыва. Приложения могут проверять список отзыва сертификатов, чтобы определить текущее состояние отзыва сертификата. Дополнительные сведения о проверке отзыва сертификатов см. в статье TechNet, посвященной [проверке отзыва и состояния сертификатов](http://technet.microsoft.com/en-us/library/ee619730(ws.10).aspx).

**Что такое LDAP через SSL** **(LDAPS)?
**LDAP через SSL (LDAPS) шифрует трафик между клиентскими компьютерами и серверами с помощью технологии Secure Sockets Layer (SSL)/Transport Layer Security (TLS).

**Что такое доменные службы Active Directory?
**Доменные службы Active Directory (AD DS) хранят данные каталогов и управляют взаимодействием между пользователями и доменами, включая процесс входа пользователей, проверку подлинности и поиск в каталогах. Контроллер домена Active Directory — это сервер, на котором выполняются службы AD DS. Дополнительные сведения о службах AD DS см. в статье TechNet, посвященной [доменным службам Active Directory](http://technet.microsoft.com/en-us/library/cc770946(ws.10).aspx).

**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**
Воспользовавшийся ею злоумышленник может получить доступ к сетевым ресурсам или выполнить код с правами конкретного авторизованного пользователя. Если у пользователя есть права администратора, то злоумышленник сможет устанавливать программы, просматривать, изменять, удалять данные или создавать новые учетные записи с полными правами пользователя.

**Каким образом злоумышленник может воспользоваться этой** **уязвимостью?**
Для использования этой уязвимости злоумышленник должен сначала получить отозванный сертификат, связанный с действительной учетной записью домена. Злоумышленник может воспользоваться этой уязвимостью, используя ранее отозванный сертификат для прохождения проверки подлинности в домене Active Directory и получения доступа к сетевым ресурсам или выполнения кода с правами конкретного авторизованного пользователя, с которым связан данный сертификат.

**Какие системы в первую очередь подвержены риску?**
Этой уязвимости подвержены все системы, на которых запущена операционная система с установленными службами Active Directory.

**Как действует обновление?**
Данное обновление устраняет уязвимость путем изменения способа, используемого службами Active Directory для проверки сертификатов по списку отзыва сертификатов.

**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.

**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

#### Сведения о развертывании

**Установка обновления**

При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

Если один из файлов был обновлен ранее другим исправлением, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае будут скопированы файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

Дополнительные сведения об установщике см. в [статье 832475 базы знаний Майкрософт](http://support.microsoft.com/kb/832475).

Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **Руководство и средства по диагностике и развертыванию** данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, в отдельных случаях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка разделов реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

#### Сведения о развертывании

**Установка обновления**

При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

Если один из файлов был обновлен ранее другим исправлением, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае будут скопированы файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

Дополнительные сведения об установщике см. в [статье 832475 базы знаний Майкрософт](http://support.microsoft.com/kb/832475).

Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).
