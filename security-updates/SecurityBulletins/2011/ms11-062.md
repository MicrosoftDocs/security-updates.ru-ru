Security Bulletin

Бюллетень по безопасности (Майкрософт) MS11-062 - Важное
========================================================

Уязвимость в драйвере службы удаленного доступа NDISTAPI делает возможным несанкционированное получение прав (2566454)
----------------------------------------------------------------------------------------------------------------------

Дата публикации: 9 августа 2011 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Настоящее обновление для системы безопасности устраняет обнаруженную пользователями уязвимость во всех поддерживаемых выпусках Windows XP и Windows Server 2003. Уровень важности этого обновления определен как "существенный" для всех выпусков Windows XP и Windows Server 2003. Операционные системы Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2 не подвержены данной уязвимости. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Уязвимость делает возможным несанкционированное получение прав, если злоумышленник вошел в уязвимую систему и запустил специально созданное приложение, разработанное для использования данной уязвимости и получения полного контроля над уязвимой системой. Чтобы воспользоваться этой уязвимостью, злоумышленник должен обладать действительными учетными данными и возможностью локального входа в систему.

Данное обновление безопасности устраняет эту уязвимость путем исправления способа, используемого в драйвере NDISTAPI для проверки введенных данных перед передачей введенных данных из пользовательского режима в ядро Windows. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация**. У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которым необходимо установить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

Уязвимость подсистемы NDISTAPI, приводящая к несанкционированному получению прав (CVE-2011-1974)  
------------------------------------------------------------------------------------------------
  
<span></span>
Уязвимость в драйвере службы удаленного доступа NDISTAPI делает возможным несанкционированное получение прав. Эта уязвимость вызвана тем, что драйвер NDISTAPI неправильно проверяет введенные пользователем данные при передаче данных из пользовательского режима в ядро. Злоумышленник, воспользовавшийся ею на локальном компьютере, может выполнить произвольный код и получить полный контроль над системой, что позволит ему устанавливать программы, просматривать, изменять и удалять данные или создавать новые учетные записи с неограниченными полномочиями.
  
В перечне "Common Vulnerabilities and Exposures" этой уязвимости присвоен номер [CVE-2011-1974](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1974).
  
#### Смягчающие факторы для уязвимости подсистемы NDISTAPI, приводящей к несанкционированному получению прав (CVE-2011-1974)
  
К смягчающим факторам относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие смягчающие факторы могут снизить опасность использования уязвимости.
  
-   Чтобы воспользоваться этой уязвимостью, злоумышленник должен обладать действительными учетными данными и возможностью локального входа в систему.
  
#### Временные решения для уязвимости подсистемы NDISTAPI, приводящей к несанкционированному получению прав (CVE-2011-1974)
  
Корпорацией Майкрософт не найдено временных решений для устранения этой уязвимости.
  
#### Часто задаваемые вопросы об уязвимости подсистемы NDISTAPI, приводящей к несанкционированному получению прав (CVE-2011-1974)
  
**Какова область воздействия этой уязвимости?**    
Эта уязвимость может приводить к повышению уровня прав.
  
**В чем причина уязвимости?**    
Эта уязвимость вызвана тем, что драйвер NDISTAPI неправильно проверяет введенные пользователем данные при передаче данных из пользовательского режима в ядро.
  
**Какие компоненты подвержены этой уязвимости?**    
Этой уязвимости подвержен компонент NDISTAPI.sys.
  
**Что такое удаленный вызов процедур (RAS)?**    
Удаленный вызов процедур (RAS) позволяет пользователям подключаться к удаленному компьютеру по телефонной линии, через Интернет или с помощью другого сетевого соединения и работать таким образом, как если бы их система была физически подключена к удаленной сети. RAS позволяет пользователям удаленно выполнять такие действия, как отправка и получение электронной почты, факсов, файлов и печать документов на офисном принтере. Дополнительные сведения см. в статье MSDN [Обзор архитектуры RAS (NDIS 5.1)](http://msdn.microsoft.com/en-us/library/ff563285(v=vs.85).aspx).
  
RAS является собственной службой в Windows XP и Windows Server 2003. В более поздних операционных системах RAS была заменена на службу маршрутизации и удаленного доступа (RRAS). Служба RRAS позволяет компьютеру работать в качестве маршрутизатора в сети. RRAS также обеспечивает серверную функциональность следующего поколения для RAS для Windows. Серверная функциональность RRAS основана на службе удаленного доступа (RAS), входившей в состав предыдущих версий операционной системы
  
**Что такое драйвер NDISTAPI?**    
Драйвер NDISTAPI является частью архитектуры RAS и обеспечивает взаимодействие служб NDISWAN и TAPI. Приложения телефонии выполняют задачи телефонии и могут посылать запросы TAPI, чтобы совершать звонки, отслеживать их состояние, разрывать соединения и звонки. Драйвер NDISTAPI передает запросы TAPI драйверу NDISWAN. Дополнительные сведения см. в статье MSDN [Обзор NDISTAPI (NDIS 5.1)](http://msdn.microsoft.com/en-us/library/ff554813(v=vs.85).aspx).
  
**Что такое NDISWAN?**    
Стандарт интерфейса сетевых драйверов в глобальных сетях (Network Driver Interface Standard Wide Area Network) (NDISWAN) - это системный промежуточный драйвер NDIS, который предоставляет драйверам минипортов WAN такую функциональность как сжатие данных, шифрование, петлевое и простое PPP-фреймирование. Драйвера минипортов WAN требуются для реализации функций, специфичных для метода передачи данных (например, для ISDN необходим сигнальный протокол Q931). Дополнительные сведения см. в статье MSDN [Обзор NDISWAN](http://msdn.microsoft.com/en-us/library/ff564653(v=vs.85).aspx).
  
**Что такое служба TAPI?**    
Интерфейс прикладного программирования для телефонии (TAPI) - это интерфейс прикладного программирования, который предоставляет услуги телефонии приложениям, способным пользоваться телефонной связью.
  
**Что такое NDIS?**    
Стандарт интерфейса сетевых драйверов (Network Driver Interface Standard (NDIS)) является частью набора драйверов Windows Driver Kit, который представляет собой полностью встроенную систему разработки драйверов, содержащую Набор драйверов устройств Windows Driver Device Kit, а также тесты стабильности и надежности драйверов Windows. Библиотека NDIS отделяет сетевое аппаратное обеспечение от сетевых драйверов. NDIS определяет стандартный интерфейс между многоуровневыми сетевыми драйверами, отделяя драйвера нижнего уровня, которые управляют работой аппаратного обеспечения от верхнеуровневых драйверов, таких как драйвера сетевого транспорта. NDIS также хранит информацию о состоянии и параметрах сетевых драйверов, включая указатели функций, обработчики, блоки параметров связи и другие системные значения.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник, успешно воспользовавшийся этой уязвимостью, может выполнить произвольный код в контексте безопасности учетной записи Local System. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Для использования этой уязвимости злоумышленник должен войти в систему. Затем он может запустить специальное приложение, способное воспользоваться уязвимостью, и таким образом установить контроль над уязвимой системой.
  
**Какие системы в первую очередь подвержены риску?**    
В первую очередь риску подвергаются рабочие станции и серверы терминалов. Серверы могут быть подвержены большему риску, если администраторы позволяют пользователям входить в систему на сервере и запускать программы. Однако общепринятой практикой настоятельно рекомендуется не предоставлять пользователям таких полномочий.
  
**Как действует обновление?**    
Данное обновление устраняет эту уязвимость путем исправления способа, используемого в драйвере NDISTAPI для проверки введенных данных перед передачей введенных данных из пользовательского режима в ядро Windows.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

**Установка обновления**

При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, в отдельных случаях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка разделов реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

**Установка обновления**
  
При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, в отдельных случаях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка разделов реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
