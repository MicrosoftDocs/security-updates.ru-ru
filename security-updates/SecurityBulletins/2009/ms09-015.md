Security Bulletin

Бюллетень по безопасности (Майкрософт) MS09-015 - Среднее
=========================================================

Уязвимость для смешанных угроз в функции SearchPath делает возможным несанкционированное получение прав (959426)
----------------------------------------------------------------------------------------------------------------

Дата публикации: 14 апреля 2009 г. | Дата обновления: 15 апреля 2009 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Данное обновление для системы безопасности устраняет уязвимость в функции SearchPath системы Windows, о которой сообщалось в открытых источниках. Она может привести к несанкционированному получению прав при загрузке пользователем специально созданного файла в определенное расположение и последующем открытии приложения, которое при определенных обстоятельствах может загрузить файл.

Это обновление для системы безопасности имеет средний уровень важности для всех поддерживаемых выпусков Windows XP, Windows Server 2003, Windows Vista и Windows Server 2008. Для Microsoft Windows 2000 оно только классифицировано как улучшение для системы безопасности. Дополнительные сведения см. в разделе **Вопросы и ответы о данном обновлении для системы безопасности**. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** данного раздела.

Обновление устраняет уязвимость, изменяя способ загрузки системой Windows файлов с рабочего стола. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

Кроме того, данное обновление для системы безопасности устраняет уязвимость, впервые описанную в [советах корпорации Майкрософт по безопасности 953818](http://technet.microsoft.com/security/advisory/953818).

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения о специфических параметрах конфигурации автоматического обновления см. в [статье 294781 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которые хотят применить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это, используя программное обеспечение для управления обновлениями или проверив его наличие в [Центре обновления Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

Вопросы и ответы о данном обновлении безопасности
-------------------------------------------------

<span></span>
**Где находятся дополнительные сведения о файлах?**  
Дополнительные сведения о файлах см. в [статье 959426 базы знаний Майкрософт](http://support.microsoft.com/kb/959426) (на английском языке).

**Какое отношение имеет данное обновление к обновлению MS09-014?**  
Уязвимость для смешанных угроз, приводящая к несанкционированному получению прав ([CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540)), описанная в этом бюллетене по безопасности, также затрагивает браузер Internet Explorer. Корпорация Майкрософт рекомендует установить обновление, содержащееся в бюллетене [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), "Накопительное обновление для системы безопасности браузера Internet Explorer (963027)".

**Производит ли данное обновление какие-либо изменения в функциях системы, связанные с безопасностью?**  
Да. Кроме изменений, перечисленных в разделе **сведений об уязвимости** данного бюллетеня, это обновление включает значительное улучшение системы безопасности — новый интерфейс API под названием SetSearchPathMode.

**Какие возможности предоставляет улучшение для системы безопасности SetSearchPathMode?**  
Этот новый интерфейс API позволяет выполнять поиск файлов с использованием функции SearchPath в режиме отдельных процессов. Благодаря этому приложения могут принудительно выполнять поиск в текущем каталоге после поиска в каталоге приложения и системных папках.

**Позволит ли установка только этого обновления для системы безопасности полностью устранить уязвимость для смешанных угроз CVE-2008-2540?**  
Нет. Необходимо также отдельно установить значительное улучшение системы безопасности, которое содержится в накопительном обновлении для системы безопасности браузера Internet Explorer [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). Однако необходимо убедиться в том, что использование такой меры подходит для используемой среды.

Чтобы применить значительное улучшение системы безопасности браузера Internet Explorer, включите контрольный ключ функции SearchPath, описанный в разделе "Вопросы и ответы о данном обновлении для системы безопасности" бюллетеня [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). Дополнительные сведения см. в перечне Common Vulnerability and Exposures, где этой уязвимости для смешанных угроз присвоен номер [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).

**Почему в таблице подверженного уязвимости программного обеспечения для системы Microsoft Windows 2000 не указаны уровень опасности и воздействие уязвимости?**
Операционная система Microsoft Windows 2000 внесена в список без указания уровня опасности и воздействия уязвимости, потому что в данном обновлении для нее отсутствует исправление, которое предназначено для других поддерживаемых выпусков операционных систем Windows. Вместо него в обновлении содержится только улучшение системы безопасности, специально предназначенное для этой операционной системы.

**Почему исправление, устраняющее уязвимость для смешанных угроз и предназначенное для системы Microsoft Windows 2000, отличается от исправлений для других платформ? **
В операционной системе Microsoft Windows 2000 отсутствует архитектура для реализации функции SetSearchPathMode, что делает устранение уязвимости в системе невозможным. Для этого необходимо изменение архитектуры значительной части операционной системы Microsoft Windows 2000, а не только уязвимого компонента. Конечный результат таких действий может оказаться в значительной степени несовместимым с операционной системой Microsoft Windows 2000, что может помешать дальнейшей работе созданных для нее приложений. Однако корпорации Майкрософт неизвестно о каких-либо действительных направлениях атак с использованием данной уязвимости в операционной системе Microsoft Windows 2000.

В связи с этими изменениями в архитектуре операционной системы Microsoft Windows 2000 необходимо применить дополнительную меру защиты, специально предназначенную для Microsoft Windows 2000, используя следующий раздел реестра: HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode. Инструкции содержатся в [статье 959426 базы знаний Майкрософт](http://support.microsoft.com/kb/959426). Дополнительные сведения см. на сайте MSDN в статье [Порядок поиска библиотек динамической компоновки (DLL)](http://msdn.microsoft.com/en-us/library/ms682586.aspx) (на английском языке).

**Что такое функция SearchPath?**  
Функция SearchPath используется для поиска конкретного файла, расположенного по определенному пути. Дополнительные сведения см. в статье MSDN [Функция SearchPath](http://msdn.microsoft.com/en-us/library/aa365527.aspx) (на английском языке).

**Что такое функция LoadLibrary?**  
Функция LoadLibrary служит для загрузки указанного модуля в адресное пространство вызывающего процесса. Этот модуль может вызвать загрузку других модулей. Дополнительные сведения см. в статье MSDN [Функция LoadLibrary](http://msdn.microsoft.com/en-us/library/ms684175(vs.85).aspx) (на английском языке).

**На моем компьютере установлена более ранняя версия программного обеспечения, описанного в этом бюллетене по безопасности. Что мне следует сделать?**  
Продукты, перечисленные в этом бюллетене, проверены на наличие уязвимости в тех или иных версиях. Жизненные циклы поддержки прочих версий программного обеспечения истекли. Сведения о жизненных циклах поддержки версий используемых программных продуктов можно найти на веб-узле [Microsoft Support Lifecycle](http://go.microsoft.com/fwlink/?linkid=21742).

Пользователям более ранних версий программного обеспечения следует срочно перейти на поддерживаемые версии, чтобы снизить вероятность наличия в системе уязвимостей. Дополнительные сведения о жизненном цикле продуктов Windows см. на веб-узле [Правила по срокам поддержки продуктов Microsoft](http://go.microsoft.com/fwlink/?linkid=21742). Дополнительные сведения о продленном периоде поддержки обновлений для системы безопасности данных версий или выпусков продуктов можно найти на веб-узле [службы технической поддержки Microsoft](http://go.microsoft.com/fwlink/?linkid=33328) (на английском языке).

Уязвимость для смешанных угроз, приводящая к несанкционированному получению прав, — CVE-2008-2540  
-------------------------------------------------------------------------------------------------
  
<span></span>
В способе обнаружения и открытия функцией SearchPath файлов в системе Windows существует уязвимость для смешанных угроз, которая может привести к несанкционированному получению прав. Злоумышленник может воспользоваться ей, убедив пользователя загрузить специально созданный файл в определенное расположение, а затем запустить приложение, которое при определенных обстоятельствах может загрузить файл.
  
В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).
  
#### Факторы, снижающие опасность уязвимости для смешанных угроз, приводящей к несанкционированному получению прав, — CVE-2008-2540
  
Корпорация Майкрософт не обнаружила факторов, снижающих опасность данной уязвимости.
  
#### Методы обхода уязвимости для смешанных угроз, приводящей к несанкционированному получению прав, — CVE-2008-2540
  
Корпорацией Майкрософт не найдены методы обхода этой уязвимости.
  
#### Вопросы и ответы об уязвимости для смешанных угроз, приводящей к несанкционированному получению прав, — CVE-2008-2540
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость для смешанных угроз приводит к несанкционированному получению прав. Воспользовавшись этой уязвимостью, злоумышленник может удаленно установить полный контроль над системой. что позволит ему устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.
  
**В чем причина уязвимости?** Из-за особого расположения библиотек приложений в системе Windows браузер Internet Explorer может открыть специально созданный файл, находящийся на рабочем столе. Смешанная угроза, позволяющая загружать файлы на компьютере пользователя без предварительного вывода запроса, может заставить браузер Internet Explorer непреднамеренно запустить загруженный файл с рабочего стола, а не из системы Windows.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Воспользовавшийся ею злоумышленник сможет запустить произвольный код. что позволит ему устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными полномочиями.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может создать специальный файл и убедить пользователя загрузить его на рабочий стол. Затем пользователю будет необходимо запустить приложение, которое откроет этот специально созданный файл. Однако злоумышленник не может заставить пользователей загружать такие файлы и сохранить их на рабочем столе.
  
**Какие системы в первую очередь подвергаются риску?**    
Риску использования данной уязвимости подвержены все операционные системы, в которых пользователи могут загружать файлы и запускать приложения, например рабочие станции и серверы терминалов.
  
**Как действует обновление?**    
Обновление изменяет порядок поиска библиотек приложений (например, перед поиском в текущих рабочих каталогах сначала выполняется поиск в системных каталогах). Кроме того, оно содержит улучшение системы безопасности — новый API под названием SetSearchPathMode, который помогает опытным пользователям установить необходимый порядок поиска для компонента SearchPath. Дополнительные сведения об улучшении системы безопасности см. в разделе **Вопросы и ответы о данном обновлении для системы безопасности**.
  
**Почему в бюллетене MS09-014, "Накопительное обновление для системы безопасности браузера Internet Explorer (963027)" этой уязвимости и уязвимости для смешанных угроз, делающей возможным удаленное выполнение кода, присвоен одинаковый номер CVE (CVE-2008-2540)?**  
С одной стороны уязвимость [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) затрагивает браузер Internet Explorer 7 для Windows XP с пакетами обновления 2 (SP2) и 3 (SP3), а также браузер Internet Explorer 7 для Windows XP Professional x64 Edition и Windows XP Professional x64 Edition с пакетом обновления 2 (SP2), а с другой стороны — функцию SearchPath в системе Windows. Корпорация Майкрософт рекомендует установить обновление, содержащееся в бюллетене [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), "Накопительное обновление для системы безопасности браузера Internet Explorer (963027)". Улучшение системы безопасности для защиты от этой уязвимости, которое включено в данное обновление (API под названием SetSearchPathMode), также соотносится с исправлением, описанным в бюллетене [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). См. пункт "Производит ли данное обновление какие-либо изменения, связанные с безопасностью, в функциях системы?" в разделе **Вопросы и ответы о данном обновлении для системы безопасности.**
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**    
Да. О данной уязвимости сообщалось в открытых источниках. В перечне Common Vulnerability and Exposures этой уязвимости присвоен номер [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540). Кроме того, данное обновление для системы безопасности устраняет уязвимость, впервые описанную в [советах корпорации Майкрософт по безопасности 953818](http://technet.microsoft.com/security/advisory/953818).
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

#### Сведения о развертывании

**Установка обновления**

При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

Если один из файлов был обновлен ранее другим исправлением, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае будут скопированы файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

Дополнительные сведения об установщике см. на [веб-узле Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.
