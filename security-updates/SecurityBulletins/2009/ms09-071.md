Security Bulletin

Бюллетень по безопасности (Майкрософт) MS09-071 - Критическое
=============================================================

Уязвимость в службе проверки подлинности в Интернете делает возможным удаленное выполнение кода (974318)
--------------------------------------------------------------------------------------------------------

Дата публикации: 14 декабря 2009 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет две обнаруженных пользователями уязвимости в Microsoft Windows. Наиболее серьезная из этих уязвимостей делает возможным удаленное выполнение кода, если сообщения, полученные сервером службы проверки подлинности в Интернете, неправильно копируются в память при обработке попыток проверки подлинности PEAP. В Windows Server 2008 служба проверки подлинности в Интернете заменена сервером политики сети. Злоумышленник, успешно воспользовавшийся любой из этих уязвимостей, может получить полный контроль над уязвимой системой. Серверы, использующие службу проверки подлинности в Интернете или сервер политики сети, подвержены уязвимости только в случае использования проверки подлинности PEAP с MS-CHAP v2.

Это обновление для системы безопасности имеет уровень "критический" для Windows Server 2008 для 32-разрядных систем с пакетом обновления 2 (SP2) и Windows Server 2008 для 64-разрядных систем с пакетом обновления 2 (SP2). Для всех остальных подверженных уязвимости и поддерживаемых выпусков Windows см. подраздел **Подвержены и не подвержены уязвимости** в этом разделе.

Это обновление для системы безопасности устраняет уязвимость, исправляя способ проверки запросов клиентов PEAP службой проверки подлинности в Интернете. Дополнительные сведения об этих уязвимостях см. в подразделе ответов на часто задаваемые вопросы для каждой уязвимости, упомянутой в разделе **Сведения об уязвимостях**.

**Рекомендация.**  У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которые хотят применить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив его наличие в [Центре обновления Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

**Известные проблемы.** Нет

Часто задаваемые вопросы о данном обновлении для системы безопасности
---------------------------------------------------------------------

<span></span>
**Почему 9 декабря 2009 г. была опубликована новая редакция этого бюллетеня?**  
Новая редакция этого бюллетеня была сделана для внесения в него следующих дополнений и исправлений:

-   Добавлена информация о сервере политики сети в Windows Server 2008 в разделы **Аннотация**, **Часто задаваемые вопросы об уязвимости в службе проверки подлинности в Интернете, связанной с повреждением памяти (CVE-2009-2505)** и в раздел **Часто задаваемые вопросы об уязвимости, связанной с обходом проверки подлинности MS-CHAP (CVE-2009-3677)**.
-   Добавлена информация о сервере проверки подлинности в Интернете в Microsoft Windows 2000 Server в раздел **Часто задаваемые вопросы об уязвимости в службе проверки подлинности в Интернете, связанной с повреждением памяти (CVE-2009-2505)** и в раздел **Часто задаваемые вопросы об уязвимости, связанной с обходом проверки подлинности MS-CHAP (CVE-2009-3677)**.
-   Исправлена информация о публикации в разделе **Часто задаваемые вопросы об уязвимости в службе проверки подлинности в Интернете, связанной с повреждением памяти (CVE-2009-2505)**.
-   Исправлена информация о проверке разделов реестра для Windows XP Professional x64 Edition с пакетом обновления 2 (SP2) в разделе **Развертывание обновления для системы безопасности**.

**Где находятся подробные сведения о файлах?**  
Местонахождение дополнительных сведений о файлах см. в справочных таблицах раздела **Развертывание обновления для системы безопасности**.

**Почему это обновление устраняет несколько уязвимостей системы безопасности, о которых уже сообщалось?**  
Это обновление разработано для нескольких уязвимостей, поскольку для устранения их причин требуется выполнить изменения в связанных файлах. Вместо установки нескольких почти одинаковых обновлений пользователям необходимо установить только данное обновление.

**Почему в функции автоматического обновления и на веб-сайте Центра загрузки Майкрософт указан уровень серьезности, не соответствующий уровню, указанному в таблице "Подвержены уязвимости"?**  
Некоторые из подверженных уязвимости выпусков Windows имеют различные уровни серьезности для различных версий операционной системы. В каждом конкретном случае для загружаемых пакетов обновлений для системы безопасности, доступных для автоматической загрузки обновлений или для загрузки с веб-сайта Центра загрузки Майкрософт, будет указываться общий уровень серьезности для операционной системы.

**На моем компьютере установлена более ранняя версия программного обеспечения, описанного в этом бюллетене по безопасности. Что мне следует сделать?**  
Продукты, перечисленные в этом бюллетене, проверены на наличие уязвимости в тех или иных версиях. Жизненные циклы поддержки прочих версий программного обеспечения истекли. Дополнительные сведения о жизненном цикле продуктов см. на веб-странице [сроков поддержки продуктов Майкрософт](http://go.microsoft.com/fwlink/?linkid=21742).

Пользователям более ранних версий программного обеспечения следует срочно перейти на поддерживаемые версии, чтобы снизить вероятность наличия в системе уязвимостей. Чтобы определить стадии жизненного цикла поддержки для вашего выпуска программного обеспечения, перейдите на веб-страницу [Выберите продукт для отображения информации о стадиях жизненного цикла](http://go.microsoft.com/fwlink/?linkid=169555). Дополнительные сведения о пакетах обновления для этих выпусков программного обеспечения см. на веб-странице[Пакеты обновления, поддерживаемые на стадиях жизненного цикла](http://go.microsoft.com/fwlink/?linkid=89213).

Для получения сведений о возможных вариантах обслуживания более ранних версий программного обеспечения свяжитесь с представителем группы по работе с заказчиками корпорации Майкрософт, ее техническим менеджером или представителем соответствующей партнерской компании Майкрософт. Пользователи, у которых нет договора типа Alliance, Premier или Authorized, могут обратиться в местное представительство корпорации Майкрософт. Для получения контактной информации посетите веб-сайт [Microsoft Worldwide Information](http://go.microsoft.com/fwlink/?linkid=33329), выберите страну в списке "Контактная информация" и нажмите кнопку **Go** (Перейти), чтобы просмотреть список телефонных номеров. Дозвонившись, попросите связать вас с местным менеджером по продажам службы поддержки Premier. Дополнительные сведения см. в разделе [Часто задаваемые вопросы о политике поддержки продуктов Майкрософт на разных стадиях жизненного цикла](http://go.microsoft.com/fwlink/?linkid=169557).

Уязвимость службы проверки подлинности в Интернете, связанная с повреждением памяти (CVE-2009-2505)  
---------------------------------------------------------------------------------------------------
  
<span></span>
В реализациях протокола PEAP (Protected Extensible Authentication Protocol) в службе проверки подлинности в Интернете существует уязвимость, которая делает возможным удаленное выполнение кода. Эта уязвимость вызвана неправильным копированием в память сообщений, полученных сервером при обработке попыток проверки подлинности PEAP. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2009-2505](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2505).
  
#### Смягчающие факторы, снижающие опасность уязвимости в службе проверки подлинности в Интернете, связанной с повреждением памяти (CVE-2009-2505)
  
К факторам, снижающим степень уязвимости, относятся параметры, стандартные конфигурации и общие рекомендации, реализованные в режиме по умолчанию и позволяющие снизить опасность использования уязвимости злоумышленниками. Следующие смягчающие факторы могут оказаться полезными в данной ситуации:
  
-   Серверы, использующие службу проверки подлинности в Интернете, уязвимы, только если на них настроено использование проверки подлинности PEAP с MS-CHAP v2.
  
#### Временные решения для уязвимости в службе проверки подлинности в Интернете, связанной с повреждением памяти (CVE-2009-2505)
  
К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.
  
-   **Использование на сервере службы проверки подлинности в Интернете протокола проверки подлинности, отличного от PEAP с MS-CHAP v2**
  
    **Побочные эффекты временного решения.** Изменение протоколов проверки подлинности может потребовать повторной настройки клиентов.
  
#### Часто задаваемые вопросы об уязвимости в службе проверки подлинности в Интернете, связанной с повреждением памяти (CVE-2009-2505)
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость делает возможным удаленное выполнение кода. Воспользовавшись этой уязвимостью, злоумышленник может удаленно установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и удалять данные, а также создавать новые учетные записи с неограниченными правами.
  
**В чем причина этой уязвимости?** Эта уязвимость вызвана недостаточной проверкой запросов проверки подлинности PEAP. Это может привести к ошибке при копировании специальных структур памяти, когда IAS-сервер обрабатывает попытки проверки подлинности PEAP.
  
**Что такое протокол PEAP (Protected Extensible Authentication Protocol)?**  
PEAP относится к семейству протоколов EAP (Extensible Authentication Protocol). В PEAP протокол TLS используется для создания шифрованного канала между запрашивающим проверку подлинности PEAP-клиентом (таким как мобильный компьютер в беспроводной сети) и средством проверки подлинности PEAP (таким как служба проверки подлинности в Интернете или RADIUS-сервер). PEAP не определяет метод проверки подлинности, но обеспечивает дополнительную защиту других протоколов проверки подлинности EAP, таких как EAP-MS-CHAP v2, которые могут работать через зашифрованные с помощью TLS каналы, предоставленные протоколом PEAP. PEAP используется как метод проверки подлинности для компьютеров беспроводных клиентов 802.1X, однако он не поддерживается для виртуальных частных сетей (VPN) или других клиентов удаленного доступа.
  
**Что такое протокол MS-CHAP v2 (Microsoft Challenge Handshake Authentication Protocol версии 2)?**  
В MS-CHAP используется схема "запрос-ответ" для проверки подлинности соединений без отправки паролей. MS-CHAP v2 обеспечивает взаимную проверку подлинности, создание более надежных ключей шифрования исходных данных для шифрования Point-to-Point Microsoft (MPPE) и разных ключей шифрования для отправляемых и принимаемых данных. Чтобы минимизировать риск компрометации паролей при их изменении, более старые методы изменения паролей в MS-CHAP не поддерживаются.
  
MS-CHAP v2 — это протокол взаимной проверки подлинности, т.е. и клиент, и сервер должны подтверждать знание пароля пользователя. Сначала сервер удаленного доступа обращается за подтверждением к клиенту удаленного доступа, отправляя ему запрос. Затем клиент удаленного доступа обращается к серверу удаленного доступа за подтверждением, отправляя запрос обратно серверу. Если сервер не может подтвердить, что ему известен пароль пользователя, правильно ответив на запрос клиента, клиент разрывает такое соединение. Без взаимной проверки подлинности клиент удаленного доступа не может установить соединение с несанкционированным сервером удаленного доступа.
  
**Что такое служба проверки подлинности в Интернете и сервер политики сети?**    
В Microsoft Windows 2000 Server служба проверки подлинности в Интернете — это реализация корпорацией Майкрософт RADIUS-сервера. В качестве RADIUS-сервера, эта служба выполняет централизованную проверку подлинности соединений, охватывая большое число типов сетевого доступа, включая беспроводные и VPN-соединения. В Microsoft Windows 2000 Server, служба проверки подлинности в Интернете не поддерживает RADIUS-прокси, так как она основана на версии Option Pack.
  
В Windows Server 2003, служба проверки подлинности в Интернете — это реализация корпорацией Майкрософт RADIUS-сервера и RADIUS-прокси. В качестве RADIUS-сервера, эта служба выполняет централизованную проверку подлинности соединений, охватывая большое число типов сетевого доступа, включая беспроводные и VPN-соединения. Как RADIUS-прокси, служба проверки подлинности в Интернете пересылает сообщения проверки подлинности и учета другим RADIUS-серверам.
  
Сервер политики сети — это реализация корпорацией Майкрософт RADIUS-сервера и RADIUS-прокси в Windows Server 2008. Сервер политики сети заменяет службу проверки подлинности в Интернете в Windows Server 2003.
  
И в службе проверки подлинности в Интернете, и в сервере политики сети доступен код PEAP MS-CHAP v2, подверженный этой уязвимости.
  
**Почему уровень серьезности этой уязвимости для клиентов операционных систем Windows определен как "существенный"?**    
В клиентах операционных систем Windows содержится уязвимый код MS-CHAP v2, однако ни один из компонентов данных операционных систем не использует его таким образом, чтобы быть подверженным этой уязвимости. Теоретически система может стать уязвимой в случае использования приложений сторонних разработчиков, поддерживающих проверку подлинности с помощью MS-CHAP v2. Так как клиенты не предназначены для предоставления таких функциональных возможностей, такой сценарий маловероятен. Тем не менее, корпорация Майкрософт предоставляет в составе данного бюллетеня безопасности обновление, устраняющее основную причину данной уязвимости клиентских платформ.
  
**Почему уровень серьезности этой уязвимости в Windows Server 2008 for Itanium-based Systems определен как "существенный"?**    
В выпуске Windows Server 2008 for Itanium-based Systems не разрешена установка службы проверки подлинности в Интернете. Но подверженные уязвимости двоичные файлы на компьютере имеются. Теоретически система может стать уязвимой в случае использования приложений сторонних разработчиков, поддерживающих проверку подлинности с помощью MS-CHAP v2. Так как эти серверы не предназначены для предоставления таких функциональных возможностей, такой сценарий маловероятен. Тем не менее, Майкрософт предоставляет в составе данного бюллетеня безопасности обновление, устраняющее основную причину данной уязвимости в Windows Server 2008 for Itanium-based Systems.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник, успешно воспользовавшийся данной уязвимостью, может выполнить произвольный код с правами администратора. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и удалять данные, а также создавать новые учетные записи.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может послать на IAS-сервер неправильно сформированный запрос проверки подлинности PEAP.
  
**Какие системы в первую очередь подвергаются риску?**    
Наибольшую угрозу эта уязвимость представляет для серверов со службами проверки подлинности в Интернете, которые позволяют удаленным ненадежным пользователям проходить проверку подлинности.
  
**Как действует обновление?**    
Данное обновление изменяет службу проверки подлинности в Интернете, чтобы обеспечить правильную проверку запросов проверки подлинности MS-CHAP v2.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

В качестве RADIUS-сервера, эта служба выполняет централизованную проверку подлинности соединений, охватывая большое число типов сетевого доступа, включая беспроводные и VPN-соединения. Как RADIUS-прокси, служба проверки подлинности в Интернете пересылает сообщения проверки подлинности и учета другим RADIUS-серверам.
  
Сервер политики сети — это реализация корпорацией Майкрософт RADIUS-сервера и RADIUS-прокси в Windows Server 2008. Сервер политики сети заменяет службу проверки подлинности в Интернете в Windows Server 2003.
  
И в службе проверки подлинности в Интернете, и в сервере политики сети доступен код PEAP MS-CHAP v2, подверженный этой уязвимости.
  
**Почему уровень серьезности этой уязвимости для клиентов операционных систем Windows определен как "средний"?**    
В клиентах операционных систем Windows содержится уязвимый код MS-CHAP v2, однако ни один из компонентов данных операционных систем не использует его таким образом, чтобы быть подверженным этой уязвимости. Теоретически система может стать уязвимой в случае использования приложений сторонних разработчиков, поддерживающих проверку подлинности с помощью MS-CHAP v2. Так как клиенты не предназначены для предоставления таких функциональных возможностей, такой сценарий маловероятен. Однако корпорация Майкрософт предоставляет в составе данного бюллетеня безопасности обновление, устраняющее основную причину данной уязвимости на клиентских операционных системах.
  
**Почему уровень серьезности этой уязвимости в Windows Server 2008 for Itanium-based Systems определен как "средний"?**    
В выпуске Windows Server 2008 for Itanium-based Systems не разрешена установка службы проверки подлинности в Интернете. Но подверженные уязвимости двоичные файлы на компьютере имеются. Теоретически система может стать уязвимой в случае использования приложений сторонних разработчиков, поддерживающих проверку подлинности с помощью MS-CHAP v2. Так как эти серверы не предназначены для предоставления таких функциональных возможностей, такой сценарий маловероятен. Тем не менее, Майкрософт предоставляет в составе данного бюллетеня безопасности обновление, устраняющее основную причину данной уязвимости в Windows Server 2008 for Itanium-based Systems.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник, успешно воспользовавшийся этой уязвимостью, может получить доступ к сетевым ресурсам с правами конкретного авторизованного пользователя. Обратите внимание, что в зависимости от требований проверки подлинности размещенных в сети приложений, у пользователя может не быть прямого доступа к конкретным данным или информации.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может послать службе проверки подлинности в Интернете неправильно сформированный запрос проверки подлинности MS-CHAP v2.
  
**Какие системы в первую очередь подвергаются риску?**    
В первую очередь этой уязвимости подвержены системы, использующие службу проверки подлинности в Интернете для предоставления внешним пользователям доступа к сетевым ресурсам.
  
**Как действует обновление?**    
Обновление устраняет уязвимость, изменяя службу проверки подлинности в Интернете, чтобы обеспечить правильную проверку запросов проверки подлинности MS-CHAP v2.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Последующие действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  В окне **Результаты поиска** в области **Помощник по поиску** выберите **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка разделов реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

#### Сведения о развертывании

**Установка обновления**

При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.

Если один из файлов ранее уже был обновлен с помощью исправления, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае копируются файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

Дополнительные сведения об установщике см. на [веб-сайте Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Последующие действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  В окне **Результаты поиска** в области **Помощник по поиску** выберите **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка разделов реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

#### Сведения о развертывании
  
**Установка обновления**
  
При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.
  
Если один из файлов ранее уже был обновлен с помощью исправления, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае копируются файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).
  
Дополнительные сведения об установщике см. на [веб-сайте Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).
