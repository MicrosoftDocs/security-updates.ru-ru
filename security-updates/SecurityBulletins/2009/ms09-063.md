Security Bulletin

Бюллетень по безопасности (Майкрософт) MS09-063 - Критическое
=============================================================

Уязвимость веб-служб API-интерфейса устройств, делающая возможным удаленное выполнение кода (973565)
----------------------------------------------------------------------------------------------------

Дата публикации: 10 ноября 2009 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость веб-служб в API-интерфейсе устройств (WSDAPI) в операционной системе Windows. Эта уязвимость делает возможным удаленное выполнение кода, если подверженная уязвимости система Windows получит особым образом созданный пакет. Эта уязвимость может использоваться злоумышленниками только в локальной подсети.

Это обновление для системы безопасности имеет уровень серьезности "критический" для всех поддерживаемых выпусков Windows Vista и Windows Server 2008. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** в этом разделе.

Это обновление устраняет уязвимость, исправляя способ обработки заголовков в WSD-сообщениях. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которые хотят применить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив его наличие в [Центре обновления Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

Часто задаваемые вопросы о данном обновлении для системы безопасности
---------------------------------------------------------------------

<span></span>
**Что такое веб-службы Майкрософт в устройствах?**  
Веб-службы в устройствах позволяет клиенту Windows обнаруживать удаленные устройства и получать к ним доступ. К таким устройствам относятся КПК и периферийные устройства компьютера (включая принтеры и камеры), а также бытовая электроника и связанные с ними сетевые службы. Поддерживается обнаружение, описание устройств и управление ими. Разработчики могут создавать клиентские прокси-серверы WSDAPI и соответствующие заглушки для узлов устройств.

Веб-службы в API-интерфейсе устройств (WSDAPI) реализуют профиль устройств для веб-служб (DPWS) в Windows Vista и Windows Server 2008. Профиль DPWS ограничивает спецификации веб-служб таким образом, чтобы облегчить обнаружение устройств клиентами Windows. После обнаружения устройства клиент может получить описание служб, размещенных на данном устройстве, и использовать эти службы.

Уязвимость веб-служб в API устройств, связанная с повреждением памяти (CVE-2009-2512)  
-------------------------------------------------------------------------------------
  
<span></span>
В системах Windows существует уязвимость веб-службы в API-интерфейсе устройств (WSDAPI), делающая возможным удаленное выполнение кода. Эта уязвимость вызвана тем, что эти службы неправильно обрабатывают WSDAPI-сообщение с особым образом созданным заголовком. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2009-2512](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2512).
  
#### Смягчающие факторы для уязвимости веб-служб в API устройств, связанных с повреждением памяти (CVE-2009-2512)
  
К факторам, снижающим степень уязвимости, относятся параметры, стандартные конфигурации и общие рекомендации, реализованные в режиме по умолчанию и позволяющие снизить опасность использования уязвимости злоумышленниками. Следующие смягчающие факторы могут оказаться полезными в данной ситуации:
  
-   Стандартные параметры конфигурации брандмауэра позволяют защитить сеть от атак из-за пределов корпоративной среды. Согласно рекомендованным решениям, на непосредственно подключенных к Интернету системах следует держать открытыми лишь минимально необходимое количество портов.  
-   Только злоумышленники в локальной подсети могут использовать эту уязвимость без вмешательства пользователя. Даже в частных сетях для уязвимой службы опасность представляют только входящие соединения от локальной подсети, защищенной брандмауэром Windows.
  
#### Временные решения для уязвимости веб-служб в API устройств, связанной с повреждением памяти (CVE-2009-2512)
  
К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.
  
-   **Блокировка входящих TCP-портов 5357 и 5358 и исходящего UDP-порта 3702**
  
    В брандмауэре Windows настройте блокировку входящих TCP-портов 5357 и 5358, а также исходящего UDP-порта 3702.
  
    Блокировка этих входящих портов позволяет предотвратить отправку вредоносными удаленными узлами особым образом созданных заголовков уязвимому компоненту. Уязвимый компьютер также может инициировать соединение с удаленными, потенциально вредоносными узлами, когда пользователь просматривает устройства в подсети. В этом случае блокирование исходящего порта позволяет предотвратить отправку специально созданных ответов вредоносными узлами.
  
    Следует отметить, что остается возможность, когда пользователь вручную вводит URL-адрес вредоносного устройства и инициирует соединение, позволяющее вредоносному устройству подвергнуть риску систему пользователя. Такая атака требует существенных усилий с применением социотехники.
  
    **Побочные эффекты временного решения.** Когда это временное решение включено, обнаружение устройств не действует.
  
#### Часто задаваемые вопросы об уязвимости веб-служб в API устройств, связанной с повреждением памяти (CVE-2009-2512)
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость делает возможным удаленное выполнение кода. Воспользовавшись этой уязвимостью, злоумышленник может удаленно установить полный контроль над системой. В случае успешной атаки злоумышленник сможет устанавливать программы, просматривать, изменять и удалять данные или создавать новые учетные записи с неограниченными полномочиями.
  
**В чем причина уязвимости?** Эта уязвимость вызвана неправильной проверкой специальных заголовков WSD-сообщений с помощью интерфейса WSDAPI, как на клиентах, так и на серверах.
  
**Что такое веб-службы Майкрософт в устройствах?**    
Веб-службы в устройствах позволяет клиенту Windows обнаруживать удаленные устройства и получать к ним доступ. К таким устройствам относятся КПК и периферийные устройства компьютера (включая принтеры и камеры), а также бытовая электроника и связанные с ними сетевые службы. Поддерживается обнаружение, описание устройств и управление ими. Разработчики могут создавать клиентские прокси-серверы WSDAPI и соответствующие заглушки для узлов устройств.
  
Веб-службы в API-интерфейсе устройств (WSDAPI) реализуют профиль устройств для веб-служб (DPWS) в Windows Vista и Windows Server 2008. Профиль DPWS ограничивает спецификации веб-служб таким образом, чтобы облегчить обнаружение устройств клиентами Windows. После обнаружения устройства клиент может получить описание служб, размещенных на данном устройстве, и использовать эти службы.
  
**Включены ли по умолчанию веб-службы для устройств?**    
Веб-службы для устройств — это интерфейс API, который может использоваться различными компонентами системы. На всех уязвимых платформах данный интерфейс API по умолчанию включен. Однако данная уязвимость может использоваться удаленно и без вмешательства пользователя, только когда служба или приложение использует данный интерфейс API с открытым TCP-портом 5357 или 5358 либо выполнятся обнаружение удаленных устройств. Это распространенная ситуация в случае подключения к сети поддерживаемых устройств (таких как принтеры и видеоустройства).
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Воспользовавшись уязвимостью, злоумышленник может получить полный контроль над уязвимой системой Windows. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и удалять данные, а также создавать новые учетные записи с неограниченными правами.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может попытаться воспользоваться этой уязвимостью, отправив специально созданное WSD-сообщение через TCP-порт 5357 или 5358 уязвимой системы. Другой способ: злоумышленник может отправить специально созданный ответ на WSD-сообщение с запросом устройств, инициируемым клиентом Windows. Обратите внимание, что приложения, использующие WSDAPI, могут задействовать порты, отличные от TCP-портов 5357 и 5358 (порты по умолчанию).
  
**Какие системы в первую очередь подвергаются риску?**    
Этой уязвимости подвержены все системы, использующие WSDAPI для связи через сеть с основанными на DPWS устройствами. Рабочие станции подвергаются особому риску, если конфигурация их брандмауэра Windows отличается от "public".
  
**Как действует обновление?**    
Данное обновление устраняет эту уязвимость, исправляя обработку заголовков WSD-сообщений.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

#### Сведения о развертывании
  
**Установка обновления**
  
При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).
  
Обновление для системы безопасности поддерживает следующие параметры командной строки.
  
| Параметр          | Описание                                                                                                                         |  
|-------------------|----------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Вывод справки о поддерживаемых параметрах.                                                                                       |  
| **/quiet**        | Подавление отображения сообщений о состоянии или ошибке.                                                                         |  
| **/norestart**    | При вводе с параметром **/quiet** система не перезагрузится после установки, даже если для ее завершения требуется перезагрузка. |
  
**Примечание**. Дополнительные сведения об установщике wusa.exe см. в [статье 934307 базы знаний Майкрософт](http://support.microsoft.com/kb/934307).
  
**Проверка успешного применения обновления**
  
-   **Анализатор безопасности Microsoft Baseline Security Analyzer**
  
    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.
  
-   **Проверка версий файлов**
  
    Последующие действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.
  
    1.  Нажмите кнопку **Пуск** и затем введите имя файла обновления в поле **Начать поиск**.  
    2.  Когда файл появится в меню **Программы**, щелкните имя файла правой кнопкой мыши и выберите пункт **Свойства**.  
    3.  Сравните размер файла на вкладке **Общие** с данными таблиц сведений о файлах, приведенных в разделе статьи базы знаний этого бюллетеня.  
    4.  Также можно открыть вкладку **Дополнительно** и сравнить версии файлов и дату их изменения с данными, приведенными в таблицах сведений о файлах раздела статьи базы знаний этого бюллетеня.  
    5.  Наконец, также можно открыть вкладку **Предыдущие версии** и сравнить предыдущие версии файлов с данными о новых или обновленных версиях.
