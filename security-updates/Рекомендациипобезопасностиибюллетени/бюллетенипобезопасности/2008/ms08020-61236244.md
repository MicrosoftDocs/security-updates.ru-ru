Бюллетень по безопасности (Майкрософт) MS08-020 - Важное
========================================================

Уязвимость в DNS-клиенте делает возможной подделку (945553)
-----------------------------------------------------------

Дата публикации: 8 апреля 2008 г. | Дата обновления: 11 апреля 2008 г.

**Версия:** 1.2

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет уязвимость, о которой сообщалось в частном порядке. В DNS-клиентах Windows существует уязвимость, допускающая подделку. Она может позволить злоумышленнику отправлять специально созданные ответы на DNS-запросы и таким образом подделывать или перенаправлять данные, передаваемые через Интернет с легальных адресов.

Это обновление для системы безопасности является существенным для системы Windows Vista x64 Edition и всех поддерживаемых выпусков Microsoft Windows 2000, Windows XP и Windows Server 2003. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** этого раздела.

Обновление для системы безопасности устраняет уязвимость, увеличивая степень случайности идентификаторов транзакций DNS. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.** Корпорация Майкрософт рекомендует установить это обновление при первой возможности.

Уязвимость DNS, делающая возможной атаку путем подделки, — CVE-2008-0087  
------------------------------------------------------------------------
  
<span></span>
В DNS-клиентах Windows существует уязвимость, допускающая подделку. Она позволяет злоумышленнику, не прошедшему проверку подлинности, отправлять вредоносные ответы на DNS-запросы, созданные уязвимыми клиентами, и таким образом подделывать или перенаправлять данные, передаваемые через Интернет с легальных адресов.
  
В перечне Common Vulnerabilities and Exposures этой уязвимости присвоен номер [CVE-2008-0087](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0087).
  
#### Факторы, снижающие опасность уязвимости DNS, делающей возможной атаку путем подделки, — CVE-2008-0087
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости.
  
-   Корпорацией Майкрософт не найдены факторы, снижающие опасность этой уязвимости.
  
#### Методы обхода уязвимости DNS, делающей возможной атаку путем подделки, — CVE-2008-0087
  
К методам обхода уязвимости относятся параметр или изменение конфигурации, которые не приводят к устранению уязвимости, но могут блокировать известные направления атак, прежде чем обновление будет установлено.
  
-   Корпорацией Майкрософт не найдены методы обхода этой уязвимости.
  
#### Вопросы и ответы об уязвимости DNS, делающей возможной атаку путем подделки, — CVE-2008-0087
  
**Какова область воздействия данной уязвимости? **  
В DNS-клиентах Windows существует уязвимость, допускающая подделку. Воспользовавшись уязвимостью, злоумышленник может подменить легальный адрес.
  
**В чем причина уязвимости? **  
Служба DNS-клиента Windows не обеспечивает достаточной энтропии при задании степени случайности значений транзакций при отправке DNS-запросов.
  
**Что может сделать злоумышленник, воспользовавшись этой уязвимостью? **  
Злоумышленник может получить сведения об идентификаторах транзакций DNS-клиентов, что позволит ему отправлять вредоносные ответы на DNS-запросы.
  
**Каким образом злоумышленник может воспользоваться уязвимостью? **  
Воспользовавшийся уязвимостью злоумышленник может отправлять на DNS-запросы ложные или вводящие в заблуждение ответы и таким образом перенаправлять данные, передаваемые через Интернет с легальных адресов, на нужный адрес.
  
**Что такое служба доменных имен (DNS)?**  
Служба доменных имен (DNS) является одним из наборов протоколов, признанных отраслевым стандартом, в который входят протоколы TCP/IP. Служба DNS реализована с использованием двух программных компонентов: DNS-сервера и DNS-клиента (или распознавателя). Оба компонента работают в качестве фоновых приложений служб. Сетевые ресурсы обозначаются числовыми IP-адресами, но такие адреса сложны для запоминания пользователями сети. В базе данных службы DNS содержатся записи, которые сопоставляют понятные пользователю буквенно-цифровые имена сетевых ресурсов с IP-адресами, используемыми этими ресурсами для связи друг с другом. Таким образом, служба DNS выполняет роль символического устройства, позволяющего пользователям легко запоминать имена сетевых ресурсов. Дополнительные сведения и логические схемы взаимодействия службы DNS с другими технологиями Windows см. в статье [Принцип работы DNS](http://technet2.microsoft.com/windowsserver/en/library/ff937311-03ce-4d04-b72c-b39c4d51cb361033.mspx) (на английском языке).
  
**Можно ли воспользоваться данной уязвимостью через Интернет? **  
Да, если отправить специально созданные ответы в клиентскую систему, подключенную к Интернету и выполняющую DNS-запросы.
  
**Какие системы в первую очередь подвергаются риску? **  
Все системы Windows, подключенные к сети или к Интернету. Этой уязвимости не подвержены системы Windows Vista с пакетом обновления 1 (SP1) и Windows Server 2008.
  
**Как действует обновление? **  
Обновление устраняет уязвимость, увеличивая степень случайности идентификаторов транзакций при обмене данными между DNS-клиентами.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности? **  
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками? **  
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

**Проверка успешного применения обновления**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе "Руководство и средства по диагностике и развертыванию" данного бюллетеня.

-   **Проверка версий файлов**

    Эти действия зависят от установленной версии системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание.** В зависимости от версии операционной системы или установленных программ некоторые из файлов, перечисленных в таблице сведений о файлах, могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, устанавливаемого на компьютере, сравнив ее с версией, приведенной в соответствующей таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

#### Сведения о развертывании

**Установка обновления**

При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.

Если один из файлов был обновлен ранее другим исправлением, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае будут скопированы файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

-   **Проверка версий файлов**

    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
