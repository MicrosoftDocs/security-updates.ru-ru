Сведения об уязвимости
----------------------

<span></span>
#### Уязвимость повреждения памяти SNMP — CVE-2006-5583

В службе SNMP имеется уязвимость удаленного выполнения кода, которая может позволить воспользовавшемуся ею злоумышленнику получить полный контроль над системой.

#### Факторы, снижающие опасность уязвимости повреждения памяти SNMP — CVE-2006-5583

-   По умолчанию служба SNMP не установлена
-   Если необходимо использовать уязвимый компонент, стандартные параметры конфигурации брандмауэра позволяют защитить сеть от атак из-за периметра организации. На непосредственно подключенных к Интернету системах рекомендуется держать открытыми минимально необходимое количество портов.

#### Методы обхода уязвимости повреждения памяти SNMP — CVE-2006-5583

Корпорация Майкрософт проверила представленные ниже методы обхода проблемы. Хотя данные меры не устраняют саму уязвимость, они позволяют блокировать известные направления атак. Если методы обхода снижают функциональность, в следующем разделе это отмечено особо.

-   **Ограничьте количество IP-адресов, имеющих доступ к данному компьютеру.**
    1.  Нажмите кнопку **Пуск** и выберите команду **Выполнить**.
    2.  В поле **Открыть**, введите **services.msc** и нажмите кнопку **ОК**.
    3.  Щелкните **SNMP Service** и выберите пункт **Свойства**.
    4.  Откройте вкладку **Безопасность** и выберите параметр **Принимать пакеты SNMP только от этих узлов**.
    5.  Добавьте IP-адрес надежной станции управления. Для этого нажмите кнопку **Добавить**, введите IP-адрес или имя сервера и нажмите кнопку **Добавить**.
-   **Заблокируйте на брандмауэре:**

    -   UDP-порт 161

    Этот порт используется для установки соединения с уязвимым компонентом. Его блокировка на брандмауэре позволяет защитить системы, защищенные брандмауэром, от попыток воспользоваться данной уязвимостью. Кроме того, заблокируйте все специально настроенные порты SNMP в удаленной системе. Для предотвращения возможных атак через другие порты корпорация Майкрософт рекомендует блокировать весь непредусмотренный входящий трафик из Интернета.

-   **Для защиты от попыток воспользоваться уязвимостью из сети используйте персональные брандмауэры, например брандмауэр Windows, входящий в состав системы Windows XP.**

    По умолчанию брандмауэр Windows, входящий в состав Windows XP, обеспечивает защиту подключения к Интернету, блокируя непредусмотренный входящий трафик. Рекомендуется блокировать все непредусмотренные входящие соединения из Интернета.

    Чтобы включить брандмауэр Windows с помощью мастера настройки сети, выполните следующие действия.

    -   Нажмите кнопку **Пуск** и выберите **Панель управления**.
    -   Дважды щелкните значок **Сетевые подключения** и щелкните ссылку **Изменить параметры брандмауэра Windows**.
    -   Убедитесь, что на вкладке **Общие** выбрано значение **Включить** (рекомендуется). Это действие включит брандмауэр Windows.
    -   После включения брандмауэра Windows установите флажок **Не разрешать исключения**, чтобы блокировать весь входящий трафик.

    Если необходимо разрешить некоторым программам и службам взаимодействовать через брандмауэр, снимите флажок **Не разрешать исключения** и откройте вкладку «Исключения». На вкладке «Исключения» выберите программы, протоколы и службы, которые необходимо активировать.

-   Для защиты от попыток воспользоваться уязвимостью из сети заблокируйте уязвимые порты систем с помощью IPsec.

    Используйте безопасный IP-протокол (IPSec) для защиты работы в сети. Для получения подробных сведения о IPSec и использовании фильтров обратитесь к [статьям 313190](http://support.microsoft.com/kb/313190) и [813878 базы знаний Майкрософт](http://support.microsoft.com/kb/813878).

-   **Отключите службу SNMP**

    Отключение службы SNMP поможет защитить систему от попыток воспользоваться уязвимостью. Чтобы отключить службу SNMP, выполните следующие действия.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Панель управления** .Кроме того, можно последовательно выбрать пункты **Настройка** и **Панель управления**.
    2.  Дважды щелкните **Администрирование**.
    3.  Дважды щелкните **Службы**.
    4.  Дважды щелкните **SNMP Service**.
    5.  В списке **Тип запуска** щелкните **Отменено**.
    6.  Щелкните **Остановка**, а затем щелкните **OK**.

    Службу SNMP можно остановить и отключить, введя в командной строке следующую команду:

    `sc stop SNMP & sc config SNMP start= disabled`

-   **Побочные эффекты.** При отключенной службе SNMP локальная или удаленная печать невозможна. Поэтому рекомендуется использовать этот метод обхода уязвимости только в тех системах, где не требуется печать.

#### Вопросы и ответы об уязвимости повреждения памяти SNMP — CVE-2006-5583

**Какова область воздействия данной уязвимости?**

В службе SNMP имеется уязвимость удаленного выполнения кода, которая может позволить воспользовавшемуся ею злоумышленнику получить полный контроль над системой.

**В чем причина уязвимости?**

Причиной уязвимости является неконтролируемый буфер в службе SNMP.

**Что такое SNMP?**

[Протокол SNMP](http://search.msdn.microsoft.com/search/redirect.aspx?title=snmp+functions+%5bsnmp%5d&url=http://msdn.microsoft.com/library/en-us/snmp/snmp/snmp_functions.asp) (протокол Simple Network Management Protocol) позволяет администраторам удаленно управлять сетевыми устройствами, например, серверами, рабочими станциями, маршрутизаторами, мостами, брандмауэрами и т. д. SNMP является стандартным протоколом, позволяющим управлять устройствам различных производителей.

**Что такое служба SNMP?**

Служба SNMP позволяет обрабатывать входящие запросы протокола Simple Network Management Protocol на локальном компьютере. Протокол SNMP включает в себя средства наблюдения за работой сетевых устройств и отправки отчетов на рабочую станцию сетевой консоли.

**Как определить, установлена ли в системе служба SNMP?**

Наличие в системе службы SNMP можно определить следующим способом.

1.  Нажмите кнопку **Пуск** и выберите команду **Выполнить**.
2.  В поле **Открыть**, введите **services.msc** и нажмите кнопку **ОК**.
3.  В списке служб найдите пункт **SNMP Service**.

Если этот пункт есть в списке, значит служба SNMP установлена.

**Каким образом злоумышленник может воспользоваться уязвимостью?**

Злоумышленник может воспользоваться данной уязвимостью для создания специально сконструированного сообщения и отправки его в уязвимую систему. Такое сообщение может привести к выполнению кода в уязвимой системе.

**Какие системы в первую очередь подвергаются риску?**

В первую очередь этой уязвимости подвержены системы Microsoft Windows

**Можно ли воспользоваться данной уязвимостью через Интернет?**

Да. Злоумышленник может попытаться воспользоваться данной уязвимостью через Интернет. Стандартные параметры конфигурации брандмауэра позволяют защитить от атак из Интернета. Корпорация Майкрософт предоставляет сведения, помогающие защитить персональный компьютер. Конечные пользователи могут обратиться на [веб-узел Защитите свой компьютер](http://go.microsoft.com/fwlink/?linkid=21169). ИТ-профессионалы могут обратиться на [веб-узел Security Guidance Center](http://go.microsoft.com/fwlink/?linkid=21171).

**Как действует обновление?**

Обновление устраняет уязвимость путем изменения способа, которым служба SNMP проверяет длину сообщения перед тем как передать его в выделенный буфер.

**Было ли объявлено об этой уязвимости до выпуска этого бюллетеня безопасности?**

Нет. Корпорация Майкрософт получила информацию об этой уязвимости из достоверных источников.

**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня безопасности какие-либо сообщения о том, что эта уязвимость была использована злоумышленниками?**

Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

**Сведения о развертывании**

Чтобы установить обновление безопасности без вмешательства пользователя на компьютере под управлением Microsoft Windows XP, введите следующую команду:

**Windowsxp-kb926247-x86-enu /quiet**

**Примечание.** Использование параметра **/quiet** приведет к отключению всех сообщений. Сообщения об ошибках также не будут выводиться. При использовании параметра **/quiet** администратор должен проверить успешность установки при помощи какого-либо из поддерживаемых методов. Кроме того, при использовании этого параметра администратор должен проверить файл KB926247.log на наличие сообщений о сбое.

Чтобы установить обновление безопасности на компьютере под управлением Windows XP без последующей перезагрузки, введите следующую команду:

**Windowsxp-kb926247-x86-enu /norestart**

-   **Программа Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). См. пункт «Можно ли определить необходимость установки этого обновления с помощью средства Microsoft Baseline Security Analyzer (MBSA)?» в разделе «Вопросы и ответы о данном обновлении безопасности» данного бюллетеня.

-   **Проверка версий файлов**

    **Примечание.** На каждом конкретном компьютере эти действия определяются установленной версией операционной системы Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  На панели **Результаты поиска** в области **Помощник по поиску** выберите пункт **Файлы и папки**.
    3.  В области **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы по файлам и нажмите кнопку **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.

    **Примечание.** В зависимости от версии операционной системы или установленных программ некоторые из файлов, перечисленных в таблице сведений о файлах, могут быть не установлены.

-   На вкладке **Версия** определите версию файла, устанавливаемого на компьютере, сравнив ее с версией, приведенной в соответствующей таблице сведений о файлах.

    **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.
