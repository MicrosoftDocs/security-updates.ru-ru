Security Bulletin

Бюллетень по безопасности (Майкрософт) MS09-069 - Важное
========================================================

Уязвимость в службе LSASS делает возможным отказ в обслуживании (974392)
------------------------------------------------------------------------

Дата публикации: 8 декабря 2009 г.

**Версия:** 1.0

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость в Microsoft Windows. Эта уязвимость делает возможной атаку типа "отказ в обслуживании", если удаленный злоумышленник, прошедший проверку подлинности, посылает особым образом созданное сообщение ISAKMP, обмениваясь данными по протоколу IPSEC (Internet Protocol security) со службой LSASS на пораженной системе.

Это обновление для системы безопасности является существенным для всех поддерживаемых выпусков Microsoft Windows 2000, Windows XP, and Windows Server 2003. Подробнее см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Оно устраняет уязвимость, изменяя способ обработки службой LSASS особым образом созданных запросов. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которым необходимо установить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

Уязвимость службы LSASS, связанная с нехваткой системных ресурсов (CVE-2009-3675)
---------------------------------------------------------------------------------

<span></span>
В Microsoft Windows существует уязвимость, которая может привести к атаке типа "отказ в обслуживании"; эта уязвимость вызвана неправильной обработкой службой LSASS особым образом созданных сообщений ISAKMP во время обмена данными по протоколу IPSec.

Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2009-3675](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-3675).

#### Смягчающие факторы для уязвимости службы LSASS, связанной с нехваткой системных ресурсов (CVE-2009-3675)

Корпорацией Майкрософт не найдено смягчающих факторов для этой уязвимости.

#### Временные решения для уязвимости службы LSASS, связанной с нехваткой системных ресурсов (CVE-2009-3675)

К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.

-   Отключение службы IPSec

    Если служба IPSec не используется, ее можно отключить.

    **Примечание.** Неправильное использование редактора реестра может привести к серьезным проблемам, для устранения которых, возможно, потребуется переустановить операционную систему. Корпорация Майкрософт не гарантирует решение проблем, возникших в результате неправильной работы с редактором реестра. Ответственность за использование редактора реестра несет пользователь. Сведения о порядке редактирования реестра см. в разделе справки "Изменение разделов и параметров" редактора реестра (Regedit.exe) или в разделах справки программы Regedt32.exe, посвященных добавлению и удалению данных реестра, а также редактированию данных реестра.

    -   **Использование интерактивного метода**
        1.  В меню **Пуск** выберите пункт **Выполнить**, в поле **Открыть** введите команду **regedit** и нажмите кнопку **ОК**.
        2.  Разверните раздел **HKEY\_LOCAL\_MACHINE**.
        3.  Раскройте ветвь **System**, затем **CurrentControlSet** и **Services**.
        4.  Щелкните раздел **PolicyAgent**.
        5.  Создайте резервную копию данного раздела реестра с помощью описанных ниже действий.
            В меню "Файл" выберите команду **Экспорт**.
            В диалоговом окне экспорта файла реестра введите **PolicyAgent\_configuration\_backup.reg**, а затем нажмите кнопку **Сохранить**.
        6.  В разделе **PolicyAgent** дважды щелкните параметр **Start**.
        7.  Измените значение поля **Значение параметра** на **4**.
        8.  Нажмите **ОК**.
        9.  Откройте окно командной строки с правами администратора и выполните следующую команду:
            net stop PolicyAgent
    -   **Применение сценария управляемого развертывания**

            В меню "Файл" выберите команду **Экспорт**.

            Regedit /s Disable\_PolicyAgent.reg
        7.  Измените значение поля **Значение параметра** на **4**.
        8.  Нажмите **ОК**.
        9.  В целевой системе откройте окно командной строки с правами администратора и выполните следующую команду:
            net stop PolicyAgent

#### Часто задаваемые вопросы об уязвимости службы LSASS, связанной с нехваткой системных ресурсов (CVE-2009-3675)

**Какова область воздействия данной уязвимости?**  
Эта уязвимость позволяет провести атаку типа "отказ в обслуживании". Обмениваясь данными по протоколу IPSec, удаленный злоумышленник, прошедший проверку подлинности, может отправить серверу особым образом созданное сообщение ISAKMP, которое вызывает потребление системных ресурсов процессом LSASS.exe, приводящее к отказу в обслуживании. Такое состояние может сохраняться до перезагрузки сервера.

**В чем причина этой уязвимости?** Существование данной уязвимости связано с неправильной обработкой службой LSASS особым образом созданных сообщений ISAKMP с неверно сформированными пакетами.

**Что такое IPsec?**
IPsec (Internet Protocol security) — это инфраструктура открытых стандартов для защиты обмена личными данными по IP-сетям с использованием служб шифрования. IPSec поддерживает проверку подлинности узлов на сетевом уровне, проверку подлинности источника данных, проверку целостности данных, их конфиденциальность (шифрование) и защиту повторения. Реализация IPSec корпорацией Майкрософт основана на стандартах, разработанных рабочей группой IPsec IETF (Internet Engineering Task Force).

**Что такое LSASS?**  
Служба LSASS предоставляет интерфейс для управления локальной безопасностью, доменной проверкой подлинности и процессами службы Active Directory. Служба LSASS управляет проверкой подлинности клиента и сервера. Служба LSASS также включает функции для поддержки служебных программ Active Directory.

**Что такое IKE?**  
Протокол IKE — это стандарт Интернета, определенный в статье RFC 2409, в которой описывается механизм сопоставлений безопасности IPSec. Сопоставление безопасности представляет собой комбинацию взаимно согласованные политику и ключи, определяющие службы и механизмы безопасности для защиты обмена данными между равноправными IPSec-узлами. В частности, IKE комбинирует протоколы ISAKMP (Internet Security Association and Key Management Protocol), описанный в RFC 2408, и Oakley Key Determination Protocol, описанный в RFC 2412
В IKE протокол Oakley используется для создания материала секретных ключей для защиты обмена данными. В IKE протокол ISAKMP используется для согласования сопоставлений безопасности.

**Что такое сообщение ISAKMP?**  
ISAKMP — это основа защищенного обмена данными с согласованием, независящая от конкретных протоколов обмена ключами, алгоритмов шифрования и обеспечения целостности и методов проверки подлинности. ISAKMP включает средства для идентификации и проверки подлинности равноправных узлов, управления сопоставлениями безопасности и обмена материалом ключей.

**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**  
Воспользовавшийся ею злоумышленник может вызвать в уязвимой системе такое потребление ресурсов центрального процессора, что система перестает отвечать до перезагрузки.

**Каким образом злоумышленник может воспользоваться этой уязвимостью?**  
Злоумышленник может воспользоваться ею, отправив особым образом созданное сообщение ISAKMP при подключении и проверки подлинности на сервере LSASS по протоколу IPSEC.

**Какие системы в первую очередь подвергаются риску?**  
В первую очередь подвержены риску серверы, запрашивающие или требующие обмена данными с клиентами по протоколу IPSec.

**Как действует обновление?**  
Данное обновление изменяет способ обработки службой LSASS особым образом созданных запросов ISAKMP.

**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**  
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.

**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**  
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что эта уязвимость была открыто использована для атак на пользователей, и ей не были представлены никакие доказательства публикации концептуального кода до первоначального выпуска этого бюллетеня безопасности.

**Установка обновления**

При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.

Если один из файлов ранее уже был обновлен с помощью исправления, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае копируются файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Последующие действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  В окне **Результаты поиска** в области **Помощник по поиску** выберите **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.

    Эти разделы реестра могут не содержать полного списка установленных файлов. Кроме того, эти разделы реестра могут быть созданы неправильно, если администратор или сборщик систем интегрируют или добавляют данное обновление для системы безопасности в исходные установочные файлы Windows.

#### Сведения о развертывании

**Установка обновления**

При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.

Если один из файлов ранее уже был обновлен с помощью исправления, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае копируются файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).

Дополнительные сведения об установщике см. на [веб-сайте Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).

**Установка обновления**
  
При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.
  
Если один из файлов ранее уже был обновлен с помощью исправления, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае копируются файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).
  
Дополнительные сведения об установщике см. на [веб-сайте Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).

### Прочие сведения

#### Программа Microsoft Active Protections Program (MAPP)
