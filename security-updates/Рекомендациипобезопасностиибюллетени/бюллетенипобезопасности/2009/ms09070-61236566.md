Security Bulletin

Бюллетень по безопасности (Майкрософт) MS09-070 - Важное
========================================================

Уязвимости служб федерации Active Directory делают возможным удаленное выполнение кода (971726)
-----------------------------------------------------------------------------------------------

Дата публикации: 14 декабря 2009 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет две обнаруженных пользователями уязвимости в Microsoft Windows. Наиболее опасная из этих уязвимостей позволяет злоумышленнику удаленно выполнять код в том случае, если злоумышленник отправил специально созданный запрос HTTP на веб-сервер с поддержкой ADFS. Чтобы воспользоваться любой из этих уязвимостей, злоумышленник должен быть пользователем, прошедшим проверку подлинности.

Уровень важности этого обновления определен как "существенный" для уязвимых выпусков Windows Server 2003, Windows Server 2003 x64 Edition, Windows Server 2008 и Windows Server 2008 x64 Edition. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** далее в этом разделе.

Это обновление для системы безопасности устраняет уязвимости путем исправления способа проверки, применяемого веб-серверами с поддержкой служб федерации Active Directory к заголовкам запросов, отправленных веб-клиентом. Дополнительные сведения об этих уязвимостях см. в подразделе ответов на часто задаваемые вопросы для каждой уязвимости, упомянутой в разделе **Сведения об уязвимостях**.

**Рекомендация.**  У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которым необходимо установить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

**Известные проблемы.** В [статье 971726 базы знаний Майкрософт](http://support.microsoft.com/kb/971726) описаны известные на данный момент проблемы, с которыми пользователи могут столкнуться при установке этого обновления для системы безопасности. **Почему это обновление устраняет несколько уязвимостей системы безопасности, о которых уже сообщалось?**  
Это обновление разработано для нескольких уязвимостей, поскольку для устранения их причин требуется выполнить изменения в связанных файлах. Вместо установки нескольких почти одинаковых обновлений пользователям необходимо установить только данное обновление.

**Что такое службы федерации Active Directory?**  
Службы федерации Active Directory основаны на развивающейся и поддерживаемой в отрасли архитектуре веб-служб, которые определяются спецификациями WS-\*. Службы федерации Active Directory позволяют использовать единый вход для проверки подлинности пользователей в нескольких связанных веб-приложениях в рамках одного рабочего сеанса. Это реализуется службами федерации Active Directory путем безопасного общего использования цифрового удостоверения и прав поверх границ безопасности и предприятия.

**Как я могу проверить, что на моем сервере установлены службы федерации Active Directory? **
Чтобы убедиться, что службы федерации Active Directory установлены на сервере, нажмите кнопку **Пуск**, щелкните пункт **Панель управления**, щелкните **Установка и удаление программ**, дважды щелкните пункт **Компоненты Windows** и затем найдите в списке установленных компонентов **Службы федерации**.

**Что такое Windows Server 2003 R2?**  
Windows Server 2003 R2 расширяет возможности операционной системы Windows Server 2003, предоставляя более эффективный способ управления доступом к локальным и удаленным ресурсам и упрощая интеграцию в существующую среду Windows Server 2003. Windows Server 2003 R2 предоставляет масштабируемую веб-платформу с усиленной безопасностью, обеспечивает эффективное взаимодействие с системами на основе UNIX, а также позволяет использовать новые сценарии, включая упрощенное управление серверами подразделений, улучшенное управление удостоверениями и доступом, а также более эффективное управление хранением данных. В Windows Server 2003 R2 Datacenter Edition и Windows Server 2003 R2 Enterprise Edition также предоставляется новое динамичное лицензирование, позволяющее пользователям получить еще большую отдачу от виртуализации серверов.

Дополнительные сведения о Windows Server 2003 R2 см. на веб-сайте [Windows Server 2003 R2](http://www.microsoft.com/windowsserver2003/default.mspx).

**Что такое спецификации веб-служб WS-\*?**

Это спецификации для архитектуры веб-службы, основанной отраслевых стандартах (таких как протокол SOAP, XML, язык WSDL и UDDI). WS-\* служит основой для предоставления законченных взаимосовместимых бизнес-решений для расширенного предприятия, включая возможность управления федеративным удостоверением и безопасностью.

Модель веб-служб основана на предположении, что корпоративные системы написаны на разных языках с использованием различных моделей программирования, причем для их работы и доступа к ним используются различные типы устройств. Веб-службы являются средством построения распределенных систем, которые могут легко и эффективно связываться и взаимодействовать между собой через Интернет, независимо от языка, на котором они написаны, или платформы, на которой они выполняются.

Уязвимость служб федерации Active Directory, связанная со спуфингом единого входа (CVE-2009-2508)  
-------------------------------------------------------------------------------------------------
  
<span></span>
Уязвимость служб федерации Active Directory, связанная со спуфингом, дает злоумышленнику возможность выдать себя за прошедшего проверку подлинности пользователя, если у злоумышленника есть доступ к рабочей станции и веб-браузеру, недавно использованных целевым пользователем для доступа к веб-сайту, предлагающему единый вход.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508).
  
#### Смягчающие факторы для уязвимости служб федерации Active Directory, связанной со спуфингом единого входа (CVE-2009-2508)
  
К факторам, снижающим степень уязвимости, относятся параметры, стандартные конфигурации и общие рекомендации, реализованные в режиме по умолчанию и позволяющие снизить опасность использования уязвимости злоумышленниками. Следующие смягчающие факторы могут оказаться полезными в данной ситуации:
  
-   Злоумышленник не может выбрать, какого пользователя он будет олицетворять. Злоумышленник может выдать себя только за того пользователя, чей маркер проверки подлинности удалось получить (например, из компьютера в интернет-кафе).  
-   Атака возможна только до истечения времени ожидания сеанса на сервере. По умолчанию сеансы остаются открытыми на сервере 600 минут. Если злоумышленник попытается повторно воспользоваться маркером проверки подлинности жертвы после истечения времени ожидания, атака будет неудачной.  
-   У злоумышленника должен быть доступ к маркеру проверки подлинности, использованному прежним пользователем на данном терминале. В случае сервера с поддержкой HTTPS маркер проверки подлинности будет защищен при передаче, поэтому у злоумышленника должен быть доступ к компьютеру жертвы (например, в интернет-кафе), чтобы воспользоваться данной уязвимостью. В интернет-кафе пользователь может зарегистрироваться в веб-приложении с единым входом на основе служб федерации Active Directory, а затем выполнить выход из приложения. Затем злоумышленник может воспользоваться этим компьютером для доступа к веб-приложению, использованному в качестве прежнего пользователя, даже если этот пользователь уже выполнил выход из приложения.

Корпорацией Майкрософт не найдено временных решений для устранения этой уязвимости.
  
#### Часто задаваемые вопросы об уязвимости служб федерации Active Directory, связанной со спуфингом единого входа (CVE-2009-2508)
  
**Какова область воздействия данной уязвимости?**    
Это уязвимость, которая делает возможным спуфинг. Злоумышленник, успешно воспользовавшийся этой уязвимостью, может выдать себя за другого пользователя, получив права доступа целевого пользователя к веб-сайту, поддерживающему функцию единого входа служб федерации Active Directory.
  
**В чем причина этой уязвимости?** Эта уязвимость возникает потому, что в службах федерации Active Directory нет достаточной проверки управления сеансами.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
В системах с общим доступом (таких как интернет-кафе) воспользовавшийся этой уязвимостью злоумышленник может получить доступ к ресурсам прежнего пользователя, прошедшего проверку подлинности. Такая атака возможна только в пределах тайм-аута сеанса, настроенного администратором служб федерации Active Directory (значение по умолчанию: 600 минут).
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может воспользоваться этой уязвимостью, если он получит доступ к определенным данным в кэше браузера и воспользуется ими для прохождения проверки подлинности в веб-приложении с функцией единого входа на основе служб федерации Active Directory. По истечении времени ожидания сеанса на сервере службы федерации Active Directory отключают проверенный сеанс на стороне сервера, исключая дальнейшую возможность использования этих учетных данных злоумышленником.
  
**Какие системы в первую очередь подвергаются риску?**    
Этой уязвимости подвержены веб-серверы, использующие службы федерации Active Directory для реализации функции единого входа. Эта уязвимость не затрагивает напрямую клиентские системы.
  
**Как действует обновление?**    
Это обновление устраняет эту уязвимость, реализуя правильный сброс учетных данных проверки подлинности после выхода пользователя из сети.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.
  
Уязвимость служб федерации Active Directory, делающая возможным удаленное выполнение кода (CVE-2009-2509)  
---------------------------------------------------------------------------------------------------------
  
<span></span>
В реализациях служб федерации Active Directory существует уязвимость, которая делает возможным удаленное выполнение кода. Эта уязвимость вызвана неправильной проверкой заголовков запросов, когда прошедший проверку подлинности пользователь подключается к веб-серверу с поддержкой служб федерации Active Directory. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509).

#### Часто задаваемые вопросы об уязвимости, служб федерации Active Directory, делающей возможным удаленное выполнение кода (CVE-2009-2509)
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость делает возможным удаленное выполнение кода. Злоумышленник, успешно воспользовавшийся этой уязвимостью, сможет затем выполнять действия на сервере IIS с теми же правами, что и рабочий процесс WPI, по умолчанию имеющий права учетной записи "Network Service".
  
**В чем причина этой уязвимости?** Эта уязвимость возникает потому, что службы федерации Active Directory неправильно проверяют заголовки запросов, отправленных прошедшими проверку подлинности клиентами.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник, воспользовавшийся этой уязвимостью, сможет выполнять любые действия на сервере IIS с теми же правами, что и процесс WPI. Рабочий процесс WPI по умолчанию имеет права учетной записи "Network Service". Серверы IIS с ASP-страницами, для которых пулы приложений настроены с помощью процесса WPI, использующего учетную запись с правами администратора, подвержены более серьезному риску, чем те серверы IIS, для пула приложений которых установлены настройки WPI по умолчанию.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может отправить особым образом созданный HTTP-запрос на веб-сервер с поддержкой служб федерации Active Directory. Чтобы воспользоваться этой уязвимостью, злоумышленник должен быть пользователем, прошедшим проверку подлинности.
  
**Какие системы в первую очередь подвергаются риску?**    
Риску подвергаются веб-серверы с поддержкой служб федерации Active Directory. Клиентские системы не подвержены этой уязвимости.
  
**Как действует обновление?**    
Данное обновление исправляет способ проверки, применяемый веб-серверами с поддержкой служб федерации Active Directory к заголовкам запросов, отправленных веб-клиентами.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников.

Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

#### Сведения о развертывании
  
**Установка обновления**
  
При установке данного обновления для системы безопасности программа-установщик проверяет, не выполнялось ли ранее обновление одного или нескольких обновляемых файлов с помощью исправления Майкрософт.
  
Если один из файлов ранее уже был обновлен с помощью исправления, установщик копирует на компьютер файлы RTMQFE, SP1QFE и SP2QFE. В противном случае копируются файлы RTMGDR, SP1GDR или SP2GDR. Обновления для системы безопасности могут не содержать всех этих вариантов. Дополнительную информацию о поведении такого рода см. в [статье 824994 базы знаний Майкрософт](http://support.microsoft.com/kb/824994).
  
Дополнительные сведения об установщике см. на [веб-сайте Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).

**Проверка успешного применения обновления**

-   **Анализатор безопасности Microsoft Baseline Security Analyzer**

    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.

-   **Проверка версий файлов**

    Последующие действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.

    1.  Нажмите кнопку **Пуск** и выберите пункт **Найти**.
    2.  В окне **Результаты поиска** в области **Помощник по поиску** выберите **Файлы и папки**.
    3.  В поле **Часть имени или имя файла целиком** введите имя файла из соответствующей справочной таблицы и нажмите **Найти**.
    4.  В списке файлов щелкните правой кнопкой мыши имя файла из соответствующей таблицы сведений о файлах и выберите **Свойства**.
        **Примечание**. В зависимости от выпуска операционной системы или имеющихся на компьютере программ некоторые файлы из таблицы сведений о файлах могут быть не установлены.
    5.  На вкладке **Версия** определите версию файла, установленного на компьютере, сравнив ее с версией, приведенной в таблице сведений о файлах.
        **Примечание**. Все атрибуты, кроме версии файла, в ходе установки могут изменяться. Проверка того, что обновление было применено, путем сравнения других атрибутов файла с данными из таблицы сведений о файлах не поддерживается. Кроме того, при определенных условиях файлы в ходе установки могут быть переименованы. Если сведения о файле или версии отсутствуют, обратитесь к другим доступным методам проверки установки обновления.

-   **Проверка параметров системного реестра**

    Файлы, установленные этим обновлением для системы безопасности, также можно проверить в разделах реестра, перечисленных в **справочной таблице** данного раздела.
