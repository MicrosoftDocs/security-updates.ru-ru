Security Bulletin

Бюллетень по безопасности (Майкрософт) MS09-030 - Важное
========================================================

Уязвимость приложения Microsoft Office Publisher делает возможным удаленное выполнение кода (969516)
----------------------------------------------------------------------------------------------------

Дата публикации: 14 июля 2009 г. | Дата обновления: 15 июля 2009 г.

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет обнаруженную пользователями уязвимость приложения Microsoft Office Publisher, которая делает возможным удаленное выполнение кода, если пользователь открывает специально созданный файл Publisher. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.

Это обновление для системы безопасности имеет уровень "существенный" для Microsoft Office Publisher 2007 с пакетом обновления 1 (SP1). Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** данного раздела.

Это обновление устраняет уязвимость, изменяя способ, который используется приложением Microsoft Office Publisher для открывания файлов Publisher. Дополнительные сведения об этой уязвимости см. в подразделе "Вопросы и ответы" раздела **Сведения об уязвимости**.

**Рекомендация.** Корпорация Майкрософт рекомендует установить это обновление при первой возможности.

**Известные проблемы.** В [статье 969693 базы знаний Майкрософт](http://support.microsoft.com/kb/969693) описаны известные на данный момент проблемы, с которыми пользователи могут столкнуться при установке этого обновления для системы безопасности.

Вопросы и ответы о данном обновлении безопасности  
-------------------------------------------------
  
<span></span>
**Почему этому обновлению присвоен уровень опасности "существенный"?**    
В приложении Microsoft Office Publisher 2002 и более поздних версиях имеется встроенная функция, которая перед открытием документа выводит запрос с вариантами выбора "Открыть", "Сохранить" и "Отмена". В связи с этим уровень опасности уязвимости, делающей возможным удаленное выполнение кода, оценивается не как "критический", а как "существенный", поскольку для проведения атаки с ее использованием необходимо выполнение пользователем более одного действия.
  
**С какими известными проблемами могут столкнуться пользователи при установке этого обновления для системы безопасности?**    
В [статье 969693 базы знаний Майкрософт](http://support.microsoft.com/kb/969693) приведены известные на данный момент проблемы, с которыми можно столкнуться при установке этого обновления для системы безопасности. В этой статье также приводятся решения, рекомендованные для устранения этих проблем.
  
**Содержит ли это обновление дополнительные изменения, связанные либо не связанные с безопасностью?**    
Да, в порядке обслуживания выпуска 2007 системы Microsoft Office после установки данного обновления выпуск 2007 системы Microsoft Office с пакетом обновления 1 (SP1) будет поддерживать функции безопасности, изначально входившие в состав [пакета обновления 2 (SP1) для выпуска 2007 системы Microsoft Office](http://www.microsoft.com/downloads/details.aspx?familyid=b444bf18-79ea-46c6-8a81-9db49b4ab6e5&displaylang=en). Все пакеты обновления, выпущенные после февраля 2009 года, содержат обновления, перечисленные в [статье 953195 базы знаний Майкрософт](http://support.microsoft.com/kb/953195) "Обзор улучшений, реализованных в пакете обновления 2 (SP2)" (на английском языке). Уязвимость разыменования указателя — CVE-2009-0566  
--------------------------------------------------
  
<span></span>
Существует уязвимость, делающая возможным удаленное выполнение кода, в способе, который используется приложением Microsoft Office Publisher для открытия, импорта и преобразования файлов, созданных в более старых версиях, чем Microsoft Office Publisher 2007. Злоумышленник может воспользоваться этой уязвимостью, создав особый файл Publisher, который может быть вложен в сообщение электронной почты или размещен на вредоносном или опасном веб-сайте.
  
Если пользователь вошел в систему с правами администратора, злоумышленник, которому удалось воспользоваться данной уязвимостью, сможет установить полный контроль над уязвимой системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2009-0566](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0566).
  
#### Факторы, снижающие опасность уязвимости разыменования указателя, — CVE-2009-0566
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Перед открытием документа в Microsoft Office Publisher 2002 и более поздних версиях пользователю предлагаются варианты выбора "Открыть", "Сохранить" или "Отмена". Это снижает опасность этой уязвимости, так как для проведения атаки с ее использованием от пользователя требуется выполнение нескольких действий.  
-   Злоумышленник, успешно воспользовавшийся данной уязвимостью, может получить те же самые права, которые имеет локальный пользователь. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.  
-   В случае атаки через Интернет злоумышленник может разместить в сети веб-сайт, содержащий веб-страницу, предназначенную для использования данной уязвимости. Кроме того, веб-сайты, которые подверглись атаке злоумышленников, и веб-сайты, которые принимают или размещают сведения или рекламу, могут иметь специальное содержимое, использующее данную уязвимость. В любом случае злоумышленник не может заставить пользователей посетить эти веб-сайты. Он старается склонить их к этому, убеждая, как правило, щелкнуть ссылку, ведущую на веб-сайт, в сообщении электронной почты или программы обмена мгновенными сообщениями.  
-   Уязвимость не может быть использована автоматически посредством электронной почты. Пользователь должен открыть вложение, отправленное в сообщении электронной почты, чтобы атака злоумышленника посредством электронной почты прошла успешно.
  
#### Способы временного устранения уязвимости разыменования указателя, — CVE-2009-0566
  
К временным решениям относятся изменения параметров или конфигурации, не приводящие к полному устранению уязвимости, но позволяющие блокировать известные направления атак до того, как будет установлено обновление. Корпорация Майкрософт проверила представленные ниже временные решения и сообщила, снижают ли они функциональные возможности.
  
-   **Отключить библиотеку DLL Publisher Converter**
  
    В Windows XP выполните в командной строке следующую команду:
  
    `cacls “c:\program files\microsoft office\office12\pubconv.dll” /E /P everyone:N`
  
    В 64-разрядных выпусках Windows XP выполните в командной строке следующую команду:
  
    `cacls <64-РАЗРЯДНЫЙ_ПУТЬ_И_ИМЯ_ФАЙЛА> /E /P everyone:N`
  
    **Примечание.** Используйте пути Windows, соответствующие текущей системе, например "%ProgramFiles(86)%" или "%windir%\\SysWOW64".
  
    В Windows Vista и Windows Server 2008 выполните в командной строке с повышенными привилегиями следующую команду:
  
    `takeown /f "c:\program files\microsoft office\office12\pubconv.dll"icacls "c:\program files\microsoft office\office12\pubconv.dll" /save %TEMP%\ PUBCONV ACL.TXTicacls "c:\program files\microsoft office\office12\pubconv.dll" /deny everyone(F)`
  
    В 64-разрядных выпусках Windows Vista и Windows Server 2008 выполните в командной строке с повышенными привилегиями следующую команду:
  
    `takeown /f <64-РАЗРЯДНЫЙ_ПУТЬ_И_ИМЯ_ФАЙЛА>icacls <64-РАЗРЯДНЫЙ_ПУТЬ_И_ИМЯ_ФАЙЛА> /save %TEMP%\ ИМЯ_ФАЙЛА _ACL.TXTicacls <64-РАЗРЯДНЫЙ_ПУТЬ_И_ИМЯ_ФАЙЛА> /deny everyone (F)`
  
    **Примечание.** Используйте пути Windows, соответствующие текущей системе, например "%ProgramFiles(86)%" или "%windir%\\SysWOW64".
  
**Побочные эффекты** Пользователи, отключившие библиотеку DLL Publisher Converter не смогут открывать файлы Microsoft Office Publisher, созданные в более ранних версиях, чем Publisher 2007.
  
**Отмена изменений**
  
В Windows XP выполните в командной строке следующую команду:
  
`cacls “c:\program files\microsoft office\office12\pubconv.dll” /E /R everyone`
  
В 64-разрядных выпусках Windows XP выполните в командной строке следующую команду:
  
`cacls <64-РАЗРЯДНЫЙ_ПУТЬ_И_ИМЯ_ФАЙЛА> /E /R everyone`
  
**Примечание.** Используйте пути Windows, соответствующие текущей системе, например "%ProgramFiles(86)%" или "%windir%\\SysWOW64".
  
В Windows Vista и Windows Server 2008 выполните в командной строке с повышенными привилегиями следующую команду:
  
`icacls “c:\program files\microsoft office\office12\” /restore %TEMP%\PUBCONV_ACL.TXT`
  
В 64-разрядных выпусках Windows Vista и Windows Server 2008 выполните в командной строке с повышенными привилегиями следующую команду:
  
`icacls <ТОЛЬКО_64-РАЗРЯДНЫЙ_ПУТЬ> /restore %TEMP%\ИМЯФАЙЛА_ACL.TXT`
  
**Примечание.** Используйте пути Windows, соответствующие текущей системе, например "%ProgramFiles(86)%" или "%windir%\\SysWOW64".
  
#### Часто задаваемые вопросы об уязвимости разыменования указателя — CVE-2009-0566
  
**Какова область воздействия данной уязвимости?**    
Эта уязвимость связана с запуском программного кода в удаленном режиме. Воспользовавшись этой уязвимостью, злоумышленник может установить полный контроль над системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами.
  
**В чем причина уязвимости?** Она возникает из-за ошибки в вычислении данных обработчика объектов при открытии специально созданного файла Publisher. При этом может произойти такое повреждение системной памяти, которое позволит злоумышленнику выполнить произвольный код.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Воспользовавшись ею, злоумышленник может выполнить произвольный код под учетной записью вошедшего в систему пользователя. Если пользователь вошел в систему с правами администратора, злоумышленник может получить полный контроль над уязвимой системой. После этого злоумышленник сможет устанавливать программы, просматривать, изменять и уничтожать данные или создавать новые учетные записи с неограниченными правами. Риск для пользователей, учетные записи которых имеют ограниченные права, меньше, чем для пользователей, работающих с правами администратора.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Злоумышленник может воспользоваться уязвимостью, если пользователь откроет специально созданный файл Publisher с помощью программы Microsoft Office Publisher.
  
При атаке через электронную почту злоумышленник может отправить такой файл и убедить пользователя открыть его.
  
В случае атаки через Интернет на веб-сайте злоумышленника должен находиться файл Publisher, предназначенный для использования этой уязвимости. Кроме того, веб-сайты, которые подверглись атаке злоумышленника и веб-сайты, которые принимают или размещают сведения пользователей, могут иметь специальное содержимое, использующее данную уязвимость. Злоумышленник не может заставить пользователей посетить специальный веб-сайт. Однако он может склонить их к этому, убедив, как правило, щелкнуть ссылку в сообщении электронной почты или в запросе программы обмена мгновенными сообщениями.
  
**Какие системы в первую очередь подвергаются риску?**    
Наиболее подвержены риску использования уязвимости те системы, в которых используется приложение Microsoft Office Publisher (включая рабочие станции и серверы терминалов). Серверы могут быть подвержены большему риску, если администраторы позволяют пользователям входить в систему на сервере и запускать программы. Однако общепринятой практикой настоятельно рекомендуется не предоставлять пользователям таких полномочий.
  
**Как действует обновление?**    
Обновление устраняет уязвимость, изменяя способ, которым приложение Microsoft Office Publisher открывает особым образом созданные файлы Publisher.
  
**Было ли объявлено о данной уязвимости до выпуска этого бюллетеня по безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из достоверных источников. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня безопасности было открыто объявлено о данной уязвимости.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. На момент первоначального выпуска этого бюллетеня корпорация Майкрософт не получала сведений об открытом использовании этой уязвимости для атак на пользователей, а также доказательств публикации концептуального кода.
