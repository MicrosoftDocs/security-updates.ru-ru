Security Bulletin

Бюллетень по безопасности (Майкрософт) MS12-032 - Важное
========================================================

Уязвимость в TCP/IP делает возможным несанкционированное получение прав (2688338)
---------------------------------------------------------------------------------

Дата публикации: 8 мая 2012 г. | Дата обновления: 9 мая 2012 г.

**Версия:** 1.1

### Общие сведения

#### Аннотация

Это обновление для системы безопасности устраняет одну уязвимость в Microsoft Windows, о которой сообщалось в открытых источниках, и одну уязвимость, которой сообщалось в частном порядке. Наиболее серьезные из этих уязвимостей делают возможным несанкционированное получение прав в том случае, если злоумышленник войдет в систему пользователя и запустит особым образом созданное приложение.

Это обновление для системы безопасности имеет уровень серьезности "существенный" для всех поддерживаемых выпусков Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** этого раздела.

Данное обновление безопасности устраняет эти уязвимости путем изменения способа, которым Windows Firewall обрабатывает исходящие пакеты и путем изменения способа, которым стек Windows TCP/IP обрабатывает привязку адреса IPv6 к локальному интерфейсу. Дополнительные сведения об этих уязвимостях см. в подразделе ответов на часто задаваемые вопросы для каждой уязвимости, упомянутой в разделе **Сведения об уязвимостях**.

**Рекомендация.** У большинства клиентов включено автоматическое обновление, поэтому нет необходимости предпринимать какие-либо действия: данное обновление для системы безопасности загрузится и установится автоматически. Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

Администраторам корпоративных установок и конечным пользователям, которые планируют установить это обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это при первой возможности, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

**Известные проблемы.** Нет

Уязвимость, связанная с обходом функций Windows Firewall - CVE-2012-0174
------------------------------------------------------------------------

<span></span>
В системе Windows существует уязвимость, связанная с обходом функций безопасности, причиной которой является способ обработки исходящих пакетов приложением Windows Firewall. Злоумышленник, успешно воспользовавшийся данной уязвимостью, может обойти механизм многоуровневой защиты Windows Firewall, чтобы облегчить использование других уязвимостей.

Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2012-0174](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0174).

#### Смягчающие факторы для уязвимости Windows Firewall, связанной с обходом функций безопасности, - CVE-2012-0174

Корпорация Майкрософт не обнаружила смягчающих факторов для данной уязвимости.

#### Временные решения для уязвимости Windows Firewall, связанной с обходом функций безопасности, - CVE-2012-0174

Корпорацией Майкрософт не найдено временных решений для устранения этой уязвимости.

#### Часто задаваемые вопросы об уязвимости Windows Firewall, связанной с обходом функций безопасности, - CVE-2012-0174

**Какова область воздействия этой уязвимости?**
Это уязвимость, связанная с обходом функций безопасности. Злоумышленник, успешно воспользовавшийся этой уязвимостью, может обойти функции безопасности Windows Firewall и облегчить использование других уязвимостей.

**В чем причина уязвимости?**
Уязвимость связана с тем, что Windows Firewall неправильно фильтрует исходящие пакеты при применении правила для исходящих соединений.

**Какой** **компонент подвержен этой уязвимости?**
Этой уязвимости подвержен стек TCP/IP (tcpip.sys).

**Что такое TCP/IP?**
TCP/IP — это набор протоколов, широко используемых в Интернете. TCP/IP обеспечивает коммуникацию между подключенными друг к другу сетями компьютеров с разными аппаратными архитектурами, работающих под управлением разных операционных систем. TCP/IP включает в себя стандарты взаимодействия между компьютерами и соглашения о подключении сетей и маршрутизации сетевого трафика. Дополнительные сведения о TCP/IP см. в статье TechNet, посвященной [обзору сетевых подключений и стека TCP/IP](http://technet.microsoft.com/en-us/library/cc739443(ws.10).aspx).

**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**
Злоумышленник, успешно воспользовавшийся этой уязвимостью, может обойти функции безопасности Windows Firewall. Впоследствии злоумышленник может воспользоваться другой уязвимостью для получения информации о целевой системе или выполнения кода в целевой системе.

**Каким образом злоумышленник может воспользоваться этой** **уязвимостью?**
Чтобы воспользоваться этой уязвимостью, злоумышленнику необходимо сначала получить доступ в локальную подсеть, к которой подключен целевой компьютер. Впоследствии злоумышленник может воспользоваться другой уязвимостью для получения информации о целевой системе или выполнения кода в целевой системе.

**Какие системы в первую очередь подвержены риску?**
В первую очередь данной уязвимости подвержены рабочие станции и серверы.

**Как действует обновление?**
Данное обновление системы безопасности устраняет эту уязвимость посредством изменения способа обработки исходящих пакетов приложением Windows Firewall.

**Какая политика для исходящих соединений применяется по умолчанию?**
Политика, применяемая по умолчанию, разрешает весь исходящий трафик с хоста. Для блокировки нежелательного исходящего трафика пользователям необходимо создать соответствующие правила брандмауэра. Дополнительные сведения о создании правил исходящего трафика в брандмауэре см. в статье TechNet [Создание правил, блокирующих нежелательный исходящий сетевой трафик](http://technet.microsoft.com/en-us/library/cc732306(v=ws.10).aspx).

**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.

**На момент выпуска этого бюллетеня получала ли корпорация Майкрософт сообщения об использовании этой уязвимости?**
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

Уязвимость, связанная с двойным освобождением TCP/IP - CVE-2012-0179
--------------------------------------------------------------------

<span></span>
В стеке Windows TCP/IP существует уязвимость, делающая возможным несанкционированное получение прав. Этой уязвимости подвержены системы Windows 7 и Windows Server 2008 R2. Все выпуски Microsoft Windows XP, Windows Server 2003, Windows Vista и Window Server 2008 не подвержены этой уязвимости. Уязвимость возникает по причине того, что стек Windows TCP/IP неправильно обрабатывает привязку адресов IPv6.

Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2012-0179](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0179).

#### Смягчающие факторы для уязвимости, связанной с двойным освобождением TCP/IP - CVE-2012-0179

К смягчающим факторам относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие смягчающие факторы могут снизить опасность использования уязвимости.

-   Чтобы воспользоваться этой уязвимостью, злоумышленник должен обладать действительными учетными данными и возможностью локального входа в систему.

#### Временные решения для уязвимости, связанной с двойным освобождением TCP/IP - CVE-2012-0179

Корпорацией Майкрософт не найдено временных решений для устранения этой уязвимости.

#### Часто задаваемые вопросы об уязвимости, связанной с двойным освобождением TCP/IP - CVE-2012-0179

**Какова область воздействия этой уязвимости?**
Эта уязвимость может приводить к повышению уровня прав.

**В чем причина уязвимости?**
Уязвимость возникает, когда стек Windows TCP/IP неправильно обрабатывает привязку адреса IPv6 к локальному интерфейсу.

**Какой компонент подвержен этой уязвимости?**
Этой уязвимости подвержен стек TCP/IP (tcpip.sys).

**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**
Воспользовавшийся ею злоумышленник сможет запустить произвольный код в контексте другого процесса. Если данный процесс запущен с правами администратора, то злоумышленник сможет устанавливать программы, просматривать, изменять, удалять данные или создавать новые учетные записи с полными правами пользователя.

**Каким образом злоумышленник может воспользоваться этой уязвимостью?**
Для использования этой уязвимости злоумышленник должен войти в систему. Затем он может запустить специальное приложение, способное воспользоваться уязвимостью, и таким образом установить контроль над уязвимой системой.

**Какие системы в первую очередь подвержены риску?**
Данной уязвимости подвержены рабочие станции и серверы.

**Как действует обновление?**
Это обновление безопасности устраняет уязвимость посредством изменения способа, которым стек Windows TCP/IP обрабатывает привязку адреса IPv6 к локальному интерфейсу.

**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**
Да. О данной уязвимости сообщалось в открытых источниках. В списке "Common Vulnerabilities and Exposures" данной уязвимости присвоен номер [CVE-2012-0179](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0179).

**На момент выпуска этого бюллетеня получала ли корпорация Майкрософт сообщения об использовании этой уязвимости?**
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.
