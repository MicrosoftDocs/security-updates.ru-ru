Security Bulletin

Бюллетень по безопасности (Майкрософт) MS10-085 - Важное
========================================================

Уязвимость в протоколе SChannel делает возможной атаку типа "отказ в обслуживании" (2207566)
--------------------------------------------------------------------------------------------

Дата публикации: 12 октября 2010 г. | Дата обновления: 18 октября 2010 г.

**Версия:** 2.0

### Общие сведения

#### Аннотация

Данное обновление для системы безопасности устраняет обнаруженную пользователями уязвимость пакета безопасности безопасного канала (Schannel) в системе Windows. Эта уязвимость делает возможной атаку типа "отказ в обслуживании" в том случае, если пораженная система получила особым образом созданное пакетное сообщение по протоколу SSL. По умолчанию все поддерживаемые выпуски Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2 не настроены для получения сетевого трафика по протоколу SSL.

Это обновление для системы безопасности имеет уровень серьезности "существенный" для всех поддерживаемых выпусков Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2. Дополнительные сведения см. в подразделе **Подвержены и не подвержены уязвимости** этого раздела.

Это обновление безопасности устраняет уязвимость путем изменения способа, используемого в SChannel для обработки запросов протокола при обработке входящих SSL-соединений. Дополнительные сведения об этой уязвимости см. в подразделе "Часто задаваемые вопросы" для соответствующей уязвимости в следующем ниже разделе **Сведения об уязвимости**.

Клиентам, у которых не включено автоматическое обновление, необходимо проверить наличие обновлений и установить это обновление вручную. Дополнительные сведения об особых параметрах конфигурации автоматического обновления см. в [статье 294871 базы знаний Майкрософт](http://support.microsoft.com/kb/294871).

При установке на предприятиях, а также администраторам или конечным пользователям, которым необходимо установить данное обновление для системы безопасности вручную, корпорация Майкрософт рекомендует сделать это немедленно, используя программное обеспечение для управления обновлениями или проверив наличие обновлений через [службу обновлений Майкрософт](http://go.microsoft.com/fwlink/?linkid=40747).

См. также раздел **Руководство и средства по диагностике и развертыванию** этого бюллетеня.

Уязвимость в протоколе TLSv1, приводящая к отказу в обслуживании (CVE-2010-3229)  
--------------------------------------------------------------------------------
  
<span></span>
Имеется уязвимость, делающая возможной атаку типа "отказ в обслуживании", причина которой заключается в способе, используемом в SChannel для обработки запросов протокола при обработке входящих SSL-соединений в Windows Server 2008, Windows Vista, Windows Server 2008 R2 и Windows 7. Удаленный анонимный злоумышленник может отправить в уязвимую систему особым образом созданный сетевой пакет, что может привести к тому, что служба LSASS перестанет отвечать на запросы, а система будет перезагружаться. Системы подвержены уязвимости только в том случае, если служба или приложение настроены для получения сетевого трафика SSL.
  
Чтобы просмотреть стандартную запись об этой уязвимости в списке "Common Vulnerabilities and Exposures", см. [CVE-2010-3229](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3229).
  
#### Смягчающие факторы для уязвимости в протоколе TLSv1, приводящей к отказу в обслуживании (CVE-2010-3229)
  
К факторам, снижающим опасность уязвимости, относятся параметр, стандартная конфигурация или общие рекомендации, которые при использовании по умолчанию могут снизить опасность использования уязвимости. Следующие факторы могут снизить опасность использования уязвимости.
  
-   Уязвимость можно использовать только в системах, где приложение или служба настроена для приема входящих SSL-соединений. В системах, в которых разрешены только исходящие SSL-соединения, использовать эту уязвимость невозможно.  
-   По умолчанию все поддерживаемые выпуски Windows Vista, Windows Server 2008, Windows 7 и Windows Server 2008 R2 не настроены для получения сетевого трафика по протоколу SSL.
  
#### Временные решения для уязвимости в протоколе TLSv1, приводящей к отказу в обслуживании (CVE-2010-3229)
  
Корпорацией Майкрософт не найдено временных решений для устранения этой уязвимости.
  
#### Часто задаваемые вопросы об уязвимости в протоколе TLSv1, приводящей к отказу в обслуживании (CVE-2010-3229)
  
**Какова область воздействия этой уязвимости?**    
Эта уязвимость позволяет провести атаку типа «отказ в обслуживании». Злоумышленник, воспользовавшийся этой уязвимостью, может вызвать "зависание" службы LSASS и перезагрузку компьютера, что приводит к отказу в обслуживании до тех пор, пока система не будет успешно перезагружена. После чего злоумышленник сможет снова воспользоваться этой уязвимостью.
  
**В чем причина уязвимости?**    
Она вызвана тем, что протоколу SChannel не удается проверить логическое состояние перед обработкой запроса протокола во время подтверждения SSL/TLS-соединения.
  
**Что такое пакет безопасности SChannel?**    
Пакет безопасности безопасного канала Schannel — это поставщик поддержки безопасности (SSP), реализующий стандартные интернет-протоколы проверки подлинности SSL и TLS. Эти компоненты используются для безопасного обмена данными при поддержке некоторых общих интернет-приложений и сетевых приложений, например веб-браузеров. Безопасный канал Schannel является частью пакета безопасности, предоставляющего службу проверки подлинности для обеспечения безопасного обмена данными между клиентом и сервером.
  
Дополнительные сведения см. в статье MSDN [Secure Channel](http://msdn.microsoft.com/en-us/library/aa380123(vs.85).aspx) (Безопасный канал).
  
**Что такое TLS и SSL?**    
Протокол подтверждения соединения TLS отвечает за проверку подлинности и обмен ключами, необходимые для установления или возобновления безопасных соединений. SSL (Secure Sockets Layer) — это предшественник протокола TLS (Transport Layer Security). Оба протокола выполняют аналогичные функции и поддерживают защищенные сетевые взаимодействия, используя технологию создания открытого и секретного ключей.
  
Дополнительные сведения см. в статье [Как работает TLS/SSL](http://technet.microsoft.com/en-us/library/cc783349(ws.10).aspx) на веб-сайте Technet.
  
**Что такое подтверждение TLS- или SSL-соединения?**    
При установлении безопасного соединения с помощью TLS или SSL протокол подтверждения соединения обеспечивает управление следующими процессами:
  
-   согласование комплекта шифров;  
-   проверка подлинности сервера и (необязательно) клиента;  
-   обмен сведениями о ключах сеанса.
  
Дополнительные сведения см. в статье [Как работает TLS/SSL](http://technet.microsoft.com/en-us/library/cc783349(ws.10).aspx) на веб-сайте Technet.
  
**Что может сделать злоумышленник, который воспользуется этой уязвимостью?**    
Злоумышленник, успешно использовавший эту уязвимость, может заставить сервер не отвечать на запросы и автоматически перезагрузиться.
  
**Каким образом злоумышленник может воспользоваться этой уязвимостью?**    
Удаленный анонимный злоумышленник может отправить особым образом созданное пакетное сообщение в службу или приложение с включенным протоколом SSL, работающее на уязвимой системе. Уязвимость может быть использована, только если служба или приложение использует SChannel для обработки SSL-трафика. К таким службам и приложениям с включенным протоколом SSL относятся, в частности, службы IIS (Internet Information Services), протоколы RDP (Remote Desktop Protocol) и SSTP (Secure Socket Tunneling Protocol), а также другие реализации туннелирования (такие как SMTP через SSL или LDAP через SSL). Во всех случаях злоумышленник должен отправить особым образом созданный пакет в рамках SSL-подтверждения. Успешное использование этой уязвимости приведет к тому, что LSASS перестанет отвечать и будет инициирована перезагрузка системы, что приведет к временному отказу в обслуживании. После успешной перезагрузки системы злоумышленник может снова использовать эту уязвимость.
  
**Какие системы в первую очередь подвержены риску?**    
Использование этой уязвимости возможно на компьютерах, на которых установлена уязвимая версия Microsoft Windows, в которой приложение или служба настроена для приема входящих SSL-соединений. К числу уязвимых систем относятся, в частности, системы, настроенные в качестве веб-серверов, принимающих SSL-соединения, VPN-серверы, принимающие входящие SSTP-соединения, а также серверы, имеющие роль "Службы удаленных рабочих столов". Риску также подвержены рабочие станции, на которых установлена уязвимая версия Microsoft Windows, в которой включен протокол RDP (Remote Desktop Protocol).
  
Меньшему риску подвержены системы, которые инициируют, но не принимают SSL-соединения. К числу систем, менее подверженных риску, относятся рабочие станции, с которых пользователи выходят в Интернет, или ноутбуки, с которых пользователи подключаются как клиенты к VPN-сети. Обратите внимание, что на таких системах могут работать приложения или службы, которые принимают входящие SSL-соединения, и в этом случае такие системы все равно подвержены риску.
  
**Как действует обновление?**    
Это обновление устраняет уязвимость путем изменения способа, используемого в SChannel для обработки запросов протокола при обработке входящих SSL-соединений.
  
**Была ли эта уязвимость опубликована до выпуска этого бюллетеня безопасности?**    
Нет. Корпорация Майкрософт получила сведения об этой уязвимости из надежных источников.
  
**Получала ли корпорация Майкрософт к моменту выпуска этого бюллетеня по безопасности какие-либо сведения о том, что уязвимость была использована злоумышленниками?**    
Нет. Корпорация Майкрософт не получала никакой информации, указывающей на то, что до выпуска этого бюллетеня по безопасности данная уязвимость была открыто использована для организации атак на пользовательские компьютеры.

#### Сведения о развертывании
  
**Установка обновления**
  
При установке этого обновления для системы безопасности проверяется, не выполнялось ли обновление этого файла (файлов) ранее с помощью исправления Microsoft.
  
Подробнее о терминологии, встречающейся в этом бюллетене, например о том, что такое *исправление*, см. в [статье 824684 базы знаний Майкрософт](http://support.microsoft.com/kb/824684).
  
Обновление для системы безопасности поддерживает следующие параметры установки.
  
| Параметр                                       | Описание                                                                                                                         |  
|------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help**                              | Вывод справки о поддерживаемых параметрах.                                                                                       |  
| **/quiet**                                     | Подавление отображения сообщений о состоянии или ошибке.                                                                         |  
| **/norestart**                                 | При вводе с параметром **/quiet** система не перезагрузится после установки, даже если для ее завершения требуется перезагрузка. |  
| **/warnrestart:&lt;секунды&gt;**               | При использовании вместе с параметром **/quiet** установщик предупреждает пользователя перед запуском перезагрузки.              |  
| **/promptrestart**                             | При использовании вместе с параметром **/quiet** установщик выдает запрос перед запуском перезагрузки.                           |  
| **/forcerestart**                              | При использовании вместе с параметром **/quiet** установщик принудительно закрывает приложения и запускает перезагрузку.         |  
| **/log:&lt;имя файла&gt;**                     | Включается ведение журнала в указанном файле.                                                                                    |  
| **/extract:&lt;назначение&gt;**                | Содержимое пакетов извлекается в указанную папку назначения.                                                                     |  
| **/uninstall /kb:&lt;номер в базе знаний&gt;** | Удаление обновления для системы безопасности.                                                                                    |
  
**Примечание.** Дополнительные сведения об установщике wusa.exe см. в разделе "Автономный установщик обновлений Windows" статьи TechNet [Другие изменения в Windows 7](http://technet.microsoft.com/en-us/library/dd871148(ws.10).aspx).
  
**Проверка успешного применения обновления**
  
-   **Анализатор безопасности Microsoft Baseline Security Analyzer**
  
    Для проверки успешного применения обновления безопасности можно воспользоваться средством Microsoft Baseline Security Analyzer (MBSA). Дополнительные сведения см. выше в разделе **"Руководство и средства по диагностике и развертыванию"** данного бюллетеня.
  
-   **Проверка версий файлов**
  
    Эти действия зависят от установленного выпуска Microsoft Windows. В случае затруднений для выполнения процедуры обратитесь к документации по установленной версии продукта.
  
    1.  Нажмите кнопку **Пуск** и затем введите имя файла обновления в поле **Начать поиск**.  
    2.  Когда файл появится в меню **Программы**, щелкните имя файла правой кнопкой мыши и выберите пункт **Свойства**.  
    3.  Сравните размер файла на вкладке **Общие** с данными таблиц сведений о файлах, приведенных в разделе статьи базы знаний этого бюллетеня.  
    4.  Также можно открыть вкладку **Дополнительно** и сравнить версии файлов и дату их изменения с данными, приведенными в таблицах сведений о файлах раздела статьи базы знаний этого бюллетеня.  
    5.  Наконец, также можно открыть вкладку **Предыдущие версии** и сравнить предыдущие версии файлов с данными о новых или обновленных версиях.
